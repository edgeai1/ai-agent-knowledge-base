# AI Agent 各方向研究递进脉络

> 本文梳理知识库中全部 110 篇论文的研究演进逻辑，揭示各方向的问题发现、方法迭代和范式转换。

## 一、奠基论文：从推理到行动的范式建立 (2022-2023)

### 递进脉络

```
问题：LLM 只能生成文本，无法推理也无法行动
                │
                ▼
┌─────────────────────────────────────────────────────────────┐
│ 第一阶段：让 LLM 学会推理                                    │
│                                                             │
│  Chain-of-Thought (2022.01)                                 │
│  "在提示中加入中间步骤，LLM 就能做多步推理"                    │
│  发现：推理是规模涌现能力（~100B+ 才有效）                     │
│       │                                                     │
│       ├──→ Tree of Thoughts (2023.05)                       │
│       │    "线性推理不够，需要分支探索和回溯"                   │
│       │    从 System 1 → System 2 思考                       │
│       │                                                     │
│       └──→ Self-Consistency (2022)                          │
│            "采样多条链，投票选最佳"                            │
└─────────────────────────────────────────────────────────────┘
                │
                ▼
┌─────────────────────────────────────────────────────────────┐
│ 第二阶段：让 LLM 学会使用工具                                 │
│                                                             │
│  MRKL (2022.05)                                             │
│  "LLM 不应什么都做，应路由到专家模块"                          │
│  概念先驱：LLM 作为路由器 + 专家系统                          │
│       │                                                     │
│       └──→ Toolformer (2023.02)                             │
│            "工具使用可以被学习，而不仅是被提示"                  │
│            关键突破：6.7B + 工具 > 175B 裸模型                 │
└─────────────────────────────────────────────────────────────┘
                │
                ▼
┌─────────────────────────────────────────────────────────────┐
│ 第三阶段：推理 + 行动 = 智能体                                │
│                                                             │
│  ReAct (2022.10)  ← 融合 CoT + MRKL                        │
│  "交替进行思考和行动，形成 Thought-Action-Observation 循环"    │
│  → 几乎所有现代 Agent 框架的基础                              │
│       │                                                     │
│       ├──→ Reflexion (2023.03) — "从失败中学习"              │
│       ├──→ HuggingGPT (2023.03) — "LLM 编排多模型"          │
│       ├──→ AutoGPT (2023.03) — "完全自主循环"                │
│       ├──→ Generative Agents (2023.04) — "记忆 + 社会行为"   │
│       └──→ Voyager (2023.05) — "技能库 + 终身学习"           │
└─────────────────────────────────────────────────────────────┘
```

### 关键递进逻辑

| 阶段 | 解决的问题 | 遗留的问题 |
|------|-----------|-----------|
| CoT | LLM 无法多步推理 | 推理无法获取外部信息 |
| MRKL/Toolformer | LLM 无法使用工具 | 推理和工具使用是割裂的 |
| ReAct | 推理与行动如何统一 | 无法从失败中学习、无记忆、不自主 |
| Reflexion/Voyager/AutoGPT | 各自解决记忆/学习/自主性 | 每个方案都有局限，需要系统化 |

## 二、Deep Research：从搜索到自主研究的演进 (2024-2026)

### 总体演进路线

```
传统搜索引擎 (Google)
    │  问题：只能返回链接列表，用户需自行阅读和综合
    ▼
RAG (2023-2024)
    │  "检索+生成"，但单轮、被动、无规划能力
    ▼
Deep Research Agent (2024-2026)
    │  多轮主动搜索 + 推理 + 综合 + 报告生成
    ▼
三条技术路线并行发展：
    ├── A. 提示工程路线（无需训练）
    ├── B. 强化学习路线（端到端训练）
    └── C. 科研自动化路线（全流程自动）
```

### A. 提示工程路线：从手工流水线到自进化

```
STORM (2024, NAACL)
│ "多视角提问 + 检索 → 长文写作"
│ 洞察：好的研究需要多角度思考，而非单一查询
│ 局限：流水线是手工设计的，无法自我改进
│
├──→ MindSearch (2024)
│    "模拟人类认知：分解 → 并行搜索 → 综合"
│    进步：图结构搜索，处理 300+ 网页
│    局限：仍依赖提示，性能受限于 LLM 基座
│
├──→ Search-o1 (2025, EMNLP)
│    "将搜索嵌入推理链内部"
│    进步：Reason-in-Documents 避免噪声注入
│    局限：推理模型本身不为搜索优化
│
├──→ WebThinker (2026, WWW Oral)
│    "LRM + Deep Web Explorer + Think-Search-Draft"
│    进步：推理模型主动触发搜索，而非被动接收
│    局限：仍需人工设计的 DPO 训练
│
└──→ EvoFSM (2026.01)
     "用有限状态机实现可控自进化"
     进步：FSM 显式建模搜索流程，可进化
     突破：Flow + Skill 解耦，自进化记忆
```

**脉络逻辑**：从固定流水线（STORM）→ 图结构搜索（MindSearch）→ 推理内嵌搜索（Search-o1）→ 推理模型主动搜索（WebThinker）→ 可控自进化（EvoFSM）。核心趋势是**从人工设计走向自主进化**。

### B. 强化学习路线：从 RAG-RL 到真实环境端到端

```
R1-Searcher (2025.03)
│ "两阶段 RL：先学格式，再学准确性"
│ 首次用 RL 训练搜索能力，但在 RAG 环境中
│ 局限：RAG 环境与真实搜索有差距
│
├──→ Search-R1 (2025.03, COLM)
│    "关键创新：检索 token 掩码"
│    进步：掩码外部 token 的损失，稳定训练
│    +26% (7B), +21% (3B)
│
├──→ ZeroSearch (2025.05)
│    "不需要真实搜索引擎也能训练搜索能力"
│    突破性洞察：用 LLM 模拟搜索结果
│    课程学习：逐步降低文档质量
│    成本：$70 vs $587（降 88%）
│
├──→ R1-Searcher++ (2025.05)
│    "自适应选择内部 vs 外部知识"
│    进步：不是所有问题都需要搜索
│    减少 42.9% 检索调用
│
├──→ DeepResearcher (2025, EMNLP)
│    "首个在真实网络环境中端到端 RL 训练"
│    里程碑：从 RAG 模拟 → 真实搜索引擎
│    涌现行为：规划、交叉验证、自我反思
│    +28.9 over 提示基线
│
├──→ PANGU DeepDiver (2025.05, NeurIPS)
│    "搜索强度缩放（SIS）是涌现能力"
│    发现：RL 训练后模型自动学会对难题搜索更多
│    7B 匹敌 671B DeepSeek-R1
│
├──→ PokeeResearch (2025.10)
│    "RLAIF：不需要人工标注的 RL"
│    进步：LLM 自动评估事实性、引用忠实度
│    7B 规模 SOTA
│
├──→ CoSearch (2026.04)
│    "推理和排序联合训练"
│    洞察：检索质量是瓶颈，应一起优化
│    +26.8% F1
│
├──→ T²PO (2026.05, ICML Spotlight)
│    "训练崩溃的根因是低效探索"
│    双粒度控制：token 级思考干预 + turn 级动态采样
│    WebShop 81.64%, 训练全程无崩溃
│
└──→ LiteResearcher (2026.04)
     "虚拟世界模拟 + 零成本 RL 训练"
     4B 模型：GAIA 71.3%, Xbench 78.0%
     证明：可扩展 RL > 模型规模
```

**脉络逻辑**：RAG 环境 RL（R1-Searcher）→ 稳定训练技巧（Search-R1 掩码）→ 模拟环境降成本（ZeroSearch）→ 真实环境端到端（DeepResearcher）→ 涌现能力发现（PANGU SIS）→ 无人工标注（PokeeResearch RLAIF）→ 解决训练崩溃（T²PO）→ 可扩展零成本（LiteResearcher）。核心趋势是**从受控环境走向真实环境，从昂贵训练走向零成本训练**。

### C. 科研自动化路线：从辅助到全自主

```
ResearchAgent (2024, NAACL)
│ "迭代生成研究想法 + 评审反馈"
│ 首次自动化：问题定义 → 方法提出 → 实验设计
│ 局限：仅生成想法，不执行实验
│
├──→ AI Scientist v1 (2024→Nature 2026)
│    "全流程：想法 → 代码 → 实验 → 论文 → 评审"
│    里程碑：首个通过同行评审的 AI 论文（6.33/10）
│    成本：$15/篇论文
│    局限：依赖人工模板
│
├──→ AI Scientist v2 (2025)
│    "去除人工模板 + 智能体树搜索"
│    进步：跨 ML 领域泛化
│    局限：仍限于 ML 领域
│
├──→ Agent Laboratory (2025, EMNLP)
│    "人机协作：文献→实验→报告"
│    进步：人类可在每阶段提供反馈
│    成本降 84%
│
├──→ AgentRxiv (2025)
│    "多 Agent 协作研究平台"
│    进步：Agent 间知识共享，协作超越单 Agent
│    +13.7% on MATH-500
│
├──→ DeepScientist (2025.09)
│    "贝叶斯优化 + 三阶段验证"
│    进步：超越人类 SOTA 达 183.7%
│    从 ML 扩展到物理、化学
│
├──→ SPARK (2026, Nature Medicine)
│    "癌症病理自主发现"
│    里程碑：自然语言作为通用接口
│    18 个队列、5 种癌症、5400+ 患者
│
└──→ SciResearcher (2026.05)
     "前沿科学推理的自动数据构造"
     8B 模型在 HLE 上 SOTA
     扩展到生物、化学等前沿学科
```

**脉络逻辑**：想法生成（ResearchAgent）→ 全流程但依赖模板（AI Scientist v1）→ 去模板化（v2）→ 人机协作（Agent Lab）→ 多 Agent 协作（AgentRxiv）→ 超人类表现（DeepScientist）→ 临床应用（SPARK）→ 前沿学科（SciResearcher）。核心趋势是**从 ML 自娱自乐走向真实科学发现**。

## 三、多智能体系统：从对话到工程化 (2023-2024)

```
CAMEL (2023, NeurIPS)
│ "两个 Agent 角色扮演，通过 inception prompting 自主对话"
│ 奠基：确立了多 Agent 通信的基本范式
│ 局限：只有两个 Agent，无结构化输出
│
├──→ MetaGPT (2023, ICLR 2024 Oral)
│    "用 SOP 编码人类工作流程"
│    关键转变：从自由对话 → 结构化工件通信
│    洞察：Agent 应通过文档（PRD、设计文档）通信，而非闲聊
│    发布-订阅机制减少噪声
│
├──→ ChatDev (2023, ACL 2024)
│    "虚拟软件公司，Chat Chain 机制"
│    进步：将瀑布模型分解为顺序双 Agent 对话
│    交互去幻觉机制提升代码质量
│    86.66% 可执行率，<7 分钟 <$1/项目
│
└──→ AutoGen (2023, COLM 2024)
     "通用可对话 Agent 抽象"
     进步：灵活的对话模式（双Agent、顺序、群聊）
     人机协作三种模式
     进化：v0.4 事件驱动架构，AG2 社区分叉
     Magentic-One：基于账本的动态编排
```

**递进逻辑**：自由角色扮演（CAMEL）→ SOP 结构化（MetaGPT）→ 任务链分解（ChatDev）→ 通用框架（AutoGen）。核心转变是**从"让 Agent 自由聊天"到"用工程化流程约束 Agent 行为"**。

## 四、记忆与规划：从无状态到认知架构 (2023-2026)

```
问题起点：LLM 是无状态的，每次对话从零开始

Generative Agents (2023, UIST Best Paper)
│ "记忆流 + 检索 + 反思 = 可信行为"
│ 三因子检索：近期性 × 重要性 × 相关性
│ 首次展示 Agent 可以有"人格"和"社会行为"
│
├──→ Reflexion (2023, NeurIPS)
│    "从失败中学习，不更新权重"
│    语言强化学习：将反思存入情景记忆
│    HumanEval 91% (vs GPT-4 67%)
│    关键发现：自然语言本身可以作为学习媒介
│
├──→ MemGPT (2023, ICLR 2024 Spotlight)
│    "操作系统启发的记忆管理"
│    主上下文 = RAM，外部存储 = 磁盘
│    Agent 自己决定何时换入/换出记忆
│    关键转变：从系统管理记忆 → Agent 自管理记忆
│
├──→ CoALA (2024, TMLR)
│    "统一认知架构框架"
│    将所有现有 Agent 映射到同一框架
│    4 种记忆类型 × 参数化 vs 非参数化
│    从工程经验 → 理论化系统设计
│
└──→ Memory Survey 2026
     "写入-管理-读取循环正式化"
     五大机制族 + 三维分类体系
     新方向：可学习遗忘、因果检索、多模态记忆
```

**递进逻辑**：记忆流（Generative Agents）→ 反思式学习（Reflexion）→ 自主内存管理（MemGPT）→ 理论统一（CoALA）→ 系统综述（2026 Survey）。核心趋势是**从简单存储到自主管理，从工程实践到理论体系**。

## 七、Agentic RAG：从被动检索到主动搜索 (2024-2026)

```
Naive RAG (2023)
│ "检索+生成"，单轮、固定查询、无判断
│
├──→ Self-RAG (2024, ICLR Oral)
│    "模型自己决定是否检索、评估检索质量"
│    反思 token：Retrieve / ISREL / ISSUP / ISUSE
│    关键转变：从系统控制检索 → 模型自主控制
│
├──→ CRAG / Corrective RAG
│    "检索后评估 + 纠错路由"
│    低质量结果 → 触发网络搜索补充
│
└──→ Agentic RAG (2025-2026)
     "完全由 Agent 驱动的检索-推理循环"
     多轮自适应检索 + 反思 + 多源综合
     实质上就是 Deep Research 的核心范式
```

**递进逻辑**：被动单轮检索 → 自判断检索（Self-RAG）→ 纠错路由（CRAG）→ Agent 驱动的主动多轮检索（Agentic RAG ≈ Deep Research）。**Agentic RAG 最终演化为 Deep Research 的基础范式**。

## 八、跨方向融合趋势 (2026)

### 趋势 1：RL 成为 Agent 训练的主流范式

```
提示工程 (2023-2024) → SFT (2024) → RL (2025-2026)

关键转折：DeepSeek-R1 (2025.01) 证明 RL 可以激励推理能力
         → R1-Searcher/Search-R1 将此迁移到搜索
         → DeepResearcher 证明真实环境 RL 可行
         → T²PO 解决了训练稳定性
         → ZeroSearch/LiteResearcher 解决了训练成本
```

### 趋势 2：小模型通过 RL 追赶大模型

```
PANGU DeepDiver: 7B ≈ 671B (DeepSeek-R1)
LiteResearcher: 4B ≈ Claude-4.5-Sonnet
Marco DeepResearch: 8B 在 BrowseComp-ZH 上超越 30B Tongyi
PokeeResearch: 7B 在 10 个基准上 SOTA

核心洞察：可扩展 RL 训练 > 模型规模
```

### 趋势 3：从单一搜索到多模态研究

```
文本搜索 → 多模态搜索 (OpenSearch-VL)
         → 科学仪器数据 (SPARK)
         → 代码+实验 (AI Scientist)
```

### 趋势 4：协议标准化推动 Agent 互操作

```
MCP (2024.11) → A2A (2025.04) → Linux Foundation AAIF (2025.12)
   │                │
   │                └── Agent 间通信标准
   └── Agent 与工具通信标准

```
