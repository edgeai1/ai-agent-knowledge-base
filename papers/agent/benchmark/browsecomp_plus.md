# BrowseComp-Plus: 面向深度研究智能体的更公平透明的评估基准

## 简要总结

BrowseComp-Plus 是在 OpenAI BrowseComp 基础上改进的深度研究智能体评估基准，被 ACL 2026 主会录用。该基准的核心创新在于引入了一个固定的、人工验证的约 100,195 篇文档语料库，替代原始 BrowseComp 依赖的动态黑箱网络搜索 API。基准包含 830 条经过人工验证的查询，每条查询均配有明确标注的支持文档和困难负例文档。这一设计使研究者能够**解耦**检索器（retriever）和大语言模型智能体（LLM agent）的贡献，从而实现公平、透明且可复现的深度研究系统比较。实验结果表明，检索器质量对端到端性能有决定性影响：使用 BM25 检索器时，开源模型 Search-R1 仅达 3.86% 准确率，而 GPT-5 达到 55.9%；搭配 Qwen3-Embedding-8B 检索器后，GPT-5 准确率进一步提升至 70.1%。

## 动机与问题

### 现有基准的根本缺陷

当前深度研究评估基准（如 OpenAI 原始 BrowseComp）存在三大关键问题：

1. **公平性缺失**：现有基准依赖动态、不透明的网络搜索 API（如 Bing、Google），不同时间点和不同 API 的搜索结果差异巨大，导致不同系统之间的比较不公平
2. **不可复现**：由于网络内容和搜索引擎排名不断变化，实验结果难以复现，违背了科学评估的基本原则
3. **组件不可解耦**：缺乏固定文档语料库使得无法区分检索器性能和 LLM 推理能力各自的贡献，端到端的评估模糊了各组件的实际效果

### 实际意义

在深度研究系统中，检索器负责从海量文档中找到相关信息，LLM 智能体负责基于检索到的信息进行推理和回答。若不能分离评估这两个组件，研究者无法判断性能瓶颈究竟在检索端还是推理端，从而无法有针对性地优化系统。

### 与原始 BrowseComp 的关系

OpenAI 在 2025 年发布的 BrowseComp 基准包含 1,005 道需要复杂网络搜索和推理才能回答的困难问题。该基准虽然在问题设计上质量很高，但其评估完全依赖实时网络搜索 API，使得不同实验室、不同时间的评估结果难以直接比较。BrowseComp-Plus 继承了 BrowseComp 的高质量问题设计，同时通过固定语料库解决了上述根本性问题。

## 方法

### 基准构建流程

BrowseComp-Plus 的构建经历了严格的多阶段流程：

1. **查询来源**：从 OpenAI BrowseComp 中获取推理密集型查询，这些查询需要复杂的多步信息检索和推理才能回答
2. **语料库构建**：策划了一个包含约 100,195 篇文档的固定语料库，涵盖回答查询所需的信息源
3. **文档标注**：
   - **支持文档（Supportive Documents）**：人工标注并验证的、包含回答查询所需信息的文档
   - **困难负例文档（Hard Negative Documents）**：通过挖掘获得的、表面上与查询相关但实际不包含正确答案的文档，用于增加检索挑战
4. **人工验证**：对原始 BrowseComp 的 1,005 个问答对进行人工验证，最终保留 830 个通过验证的高质量查询
5. **语料库封闭性验证**：确保每个查询均可在固定语料库内找到充分的支持文档来得出正确答案

### 评估框架设计

BrowseComp-Plus 提供了组件级评估能力：

- **检索器独立评估**：在固定语料库上直接评估不同检索方法的文档召回率和排序质量
- **LLM 智能体独立评估**：给定相同的检索结果，对比不同 LLM 的推理和综合能力
- **端到端评估**：将检索器与 LLM 自由组合，评估完整系统的准确率
- **LLM-as-Judge 评估**：采用 LLM 作为评判者的方式进行自动化评估

### 实验配置

实验涉及多种检索器和 LLM 的组合：

- **检索器**：BM25（稀疏检索）、Qwen3-Embedding-8B（稠密检索）等
- **LLM 智能体**：Search-R1（开源）、GPT-5、o3 等前沿模型

### 与现有基准的对比

| 特征 | BrowseComp | BrowseComp-Plus |
|------|-----------|-----------------|
| 搜索方式 | 实时网络 API | 固定语料库 |
| 文档数量 | 无限（网络） | 100,195 篇 |
| 可复现性 | 低 | 高 |
| 组件解耦 | 不支持 | 支持 |
| 评估成本 | 高（API 费用） | 低 |
| 查询数量 | 1,005 | 830 |
| 困难负例 | 无显式标注 | 有显式标注 |

## 关键创新

1. **首个解耦评估基准**：首次在深度研究领域实现了检索器与 LLM 智能体效果的系统性解耦，使研究者可以精准定位性能瓶颈
2. **固定语料库设计**：100,195 篇文档的静态语料库彻底消除了动态网络搜索带来的不公平和不可复现问题
3. **困难负例挖掘**：精心构建的困难负例文档保持了检索任务的挑战性，避免了因语料库固定而导致任务过于简单
4. **大幅降低评估成本**：消除了对商业搜索 API 的依赖，显著降低了评估的经济成本和时间成本
5. **组件级洞察能力**：揭示了稠密推理专用检索器（如 Qwen3-8B）相比稀疏 BM25 检索器带来的巨大性能提升
6. **人工验证的高标准**：从原始 1,005 个查询中通过人工验证筛选出 830 个，确保每个查询都可在固定语料库中找到充分证据
7. **可扩展的评估范式**：固定语料库 + 显式文档标注的范式可推广至其他搜索密集型 AI 系统的评估

## 实验结果

### 核心性能数据

| 配置 | 准确率 |
|------|--------|
| Search-R1 + BM25 | 3.86% |
| GPT-5 + BM25 | 55.9% |
| GPT-5 + Qwen3-Embedding-8B | 70.1% |

### 关键发现

1. **检索器质量至关重要**：稠密推理专用检索器（如 Qwen3-Embedding-8B）相比稀疏 BM25 检索器，在检索召回率和端到端准确率上均有显著提升
2. **LLM 能力差异巨大**：在相同的 BM25 检索器下，Search-R1（3.86%）与 GPT-5（55.9%）之间存在 14 倍以上的性能差距
3. **检索器-LLM 协同效应**：优质检索器与前沿 LLM 的组合不仅提高准确率，还减少了搜索调用次数
4. **组件解耦的价值**：通过分离评估，研究者可以明确看到性能提升是来自更好的检索器还是更强的 LLM
5. **查询难度分布**：830 个查询涵盖了从单步检索到复杂多步推理的多种难度级别，困难负例的存在使得即使是简单查询也需要精确的检索才能避免错误答案

### 检索器效果的深入分析

实验揭示了一个重要的现象——检索器类型对不同 LLM 的增益幅度不同：

- 对于前沿 LLM（如 GPT-5、o3），从 BM25 切换到稠密推理检索器带来了显著但适度的提升，说明这些模型自身已具备较强的信息整合和推理能力
- 对于开源模型（如 Search-R1），检索器质量的提升并不能弥补 LLM 推理能力的根本差距
- 稠密推理专用检索器不仅提高了召回率，还减少了搜索调用次数，表明更精准的检索可以降低系统的整体计算开销

## 局限性

1. **语料库覆盖范围有限**：固定的 100K 文档语料库无法覆盖所有可能的网络信息，某些类型的查询可能因此被排除
2. **查询类型受限**：继承自 BrowseComp 的查询主要聚焦于事实性推理，未必能代表深度研究的所有场景（如开放式探索性研究）
3. **时效性问题**：固定语料库无法反映网络信息的持续更新，可能不适合评估需要最新信息的任务
4. **评估指标单一**：主要采用准确率作为评估指标，缺乏对报告质量、信息完整性等维度的多元评估
5. **语料库静态性**：深度研究的真实场景中，信息源是动态变化的，固定语料库可能过于简化了这一挑战
6. **仅覆盖问答任务**：未涉及长文本报告生成等深度研究的核心任务形式
7. **困难负例的代表性**：挖掘生成的困难负例可能无法完全模拟真实检索场景中出现的所有误导性文档类型

## 核心要点

- BrowseComp-Plus 通过固定语料库和组件解耦设计解决了现有深度研究基准的公平性和可复现性问题
- 830 条人工验证查询 + 100,195 篇文档 + 支持/困难负例文档构成了完整的评估体系
- 检索器质量对深度研究系统的端到端性能影响巨大，甚至可能超过 LLM 本身的差异
- 稠密推理检索器（Qwen3-Embedding-8B）相比稀疏检索器（BM25）可带来超过 14 个百分点的准确率提升
- 该工作为深度研究系统的模块化优化提供了方法论基础，使研究者可以有针对性地改进检索或推理组件
- 代码和数据公开于 GitHub（texttron/BrowseComp-Plus）和 Hugging Face（Tevatron/browsecomp-plus）
- 与 IDRBench、DeepResearch Bench II 等同期工作互补：BrowseComp-Plus 侧重解耦评估和检索质量分析，后者侧重交互和报告质量评估
- 该工作被 ACL 2026 主会录用，证明了学术界对公平可复现的深度研究评估的重视
- 从研究方法论角度看，BrowseComp-Plus 建立了深度研究评估的"可控实验"范式——通过控制变量（固定语料库）来隔离和研究各组件的独立贡献
- 该基准对深度研究系统的工程实践也有重要指导意义：开发者应优先投入检索器优化，因为检索器质量是决定系统上限的关键因素
