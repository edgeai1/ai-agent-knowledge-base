# DeepResearch Bench II: 基于专家报告评分规则诊断深度研究智能体

## 简要总结

DeepResearch Bench II（DRB2）是一个用于诊断深度研究系统（Deep Research Systems, DRS）生成报告质量的综合基准。该基准包含 132 个有据可查的研究任务，横跨 22 个学科领域，共计 9,430 条细粒度二值评分规则（binary rubrics）。这些评分规则从精心挑选的专家撰写的调查性文章中通过四阶段 LLM + 人工流水线提取和精炼而来，涵盖三个评估维度：信息召回（Information Recall）、分析（Analysis）和呈现（Presentation）。在榜单上，Zhipu Deep Research 以 57.06 分排名第一。即使是最强的模型也仅满足不到 50% 的评分规则，揭示了当前深度研究系统与人类专家之间存在的巨大差距。

## 动机与问题

### 现有评估方法的不足

现有的深度研究评估方法存在以下关键缺陷：

1. **评估粒度过粗**：大多数评估采用整体打分或少量维度的 Likert 量表，无法精确定位系统在哪些具体方面存在不足
2. **评分规则缺乏根基**：评估标准往往由研究者主观设定，缺乏与领域专家知识的对齐
3. **幻觉检测困难**：通用评估方法难以识别看似正确但实际错误的"精细幻觉"（hallucination），因为评分规则本身不够具体
4. **缺乏诊断能力**：现有基准只能给出系统的总体得分，但无法具体告诉研究者系统在哪些能力维度（检索、分析、表达）上存在瓶颈

### 为什么选择专家报告作为评分基准？

人类专家撰写的调查报告代表了深度研究的"黄金标准"。将专家报告分解为原子性的、可验证的评分规则，可以：
- 确保评估标准的权威性和领域相关性
- 使 LLM 评判者能够识别并拒绝精细幻觉
- 提供无偏的、可验证的评估，与人类专家判断对齐

## 方法

### 任务构建

1. **专家文章收集**：从声誉良好的开放获取平台收集高质量、专家撰写的调查性报告
2. **任务设计**：基于这些报告的主题和领域，构建对应的研究型任务
3. **质量过滤**：通过一系列质量检查筛选任务，确保任务的难度适中且有意义
4. **领域覆盖**：最终保留 132 个任务，覆盖 22 个不同的学科领域

### 四阶段评分规则构建流水线

这是 DRB2 的核心方法论创新：

**第一阶段：LLM 提取（LLM Extraction）**
- 使用 LLM 从专家撰写的文章中自动提取候选评分规则
- 每条规则编码一个具体的事实性或推理性要求

**第二阶段：自评估过滤（Self-Evaluation Filtering）**
- 如果从某篇文章提取的评分规则集在关键维度上的准确率低于 90%，则重新执行提取
- 通过反复迭代确保规则的事实有效性

**第三阶段：人工清洗（Manual Cleaning）**
- 经过培训的标注员审核所有评分规则
- 操作包括：拆分复合陈述、消除歧义、去除冗余、强制要求数值精确验证
- 确保每条规则是原子性的（仅包含一个可验证的判断点）

**第四阶段：领域专家精炼（Domain Expert Refinement）**
- 领域专家投入超过 400 小时的人工审核
- 验证所有规则是否代表了本质性的、可验证的、领域适当的标准
- 最终产出 9,430 条高质量二值评分规则

### 三维度评估体系

**信息召回（Information Recall）**
- 评估模型能否利用规划和推理能力，借助搜索工具从互联网准确、全面地检索相关信息
- 考察检索的广度（覆盖了多少相关信息源）和深度（信息的细节程度）

**分析（Analysis）**
- 评估模型能否综合所有收集到的信息，提取出新的、隐藏的洞见
- 包括趋势识别、范式发现、因果推理、数据交叉分析等高阶认知能力

**呈现（Presentation）**
- 评估模型能否以恰当的方式呈现研究发现
- 确保用户能够信任和验证信息：包括引用标注、数据来源追溯、结构化组织等

### LLM 评判机制

- 使用 LLM 作为评判者（LLM-as-Judge）对每条二值评分规则进行逐一判定
- 由于规则源自专家报告且具有原子性，LLM 评判者能够更准确地拒绝幻觉内容
- 每条规则的判定为二值：满足（1）或不满足（0）

## 关键创新

1. **专家报告驱动的评分规则**：首次系统性地从人类专家报告中提取评分规则，确保评估标准的权威性
2. **四阶段混合流水线**：结合 LLM 自动化与 400+ 小时人工审核，在效率和质量之间取得平衡
3. **原子性二值规则**：9,430 条细粒度规则每条仅判断一个具体要求，避免了主观评分的模糊性
4. **三维度诊断能力**：在信息召回、分析、呈现三个维度上提供精确的性能诊断，使研究者可针对性优化
5. **抗幻觉评估**：基于专家报告的具体事实要求，使评估系统能识别和拒绝精细幻觉
6. **大规模覆盖**：132 个任务覆盖 22 个领域，确保评估的广泛代表性

### 关键发现

1. **人机差距显著**：即使是排名第一的 Zhipu Deep Research（57.06 分），也仅满足约一半的专家评分规则，表明当前系统与人类专家之间仍存在巨大差距
2. **最强模型也不足 50%**：最优秀的模型在三个维度上的综合满足率均低于 50%，说明深度研究仍是一个极具挑战性的任务
3. **维度间差异明显**：
   - OpenAI GPT-o3 Deep Research 在信息召回维度和整体聚合指标上表现最强
   - Gemini-3-Pro / Gemini-2.5-Pro Deep Research 在分析维度上表现突出，具有更偏向分析的能力画像
4. **各系统能力结构不同**：不同系统在三个评估维度上呈现出不同的强弱分布，反映了系统设计理念的差异
5. **长文本综合能力不足**：评估暴露了当前系统在长上下文综合、结构化推理和任务对齐报告生成方面的显著不足

### NVIDIA AI-Q 的表现

NVIDIA AI-Q 在 DeepResearch Bench I 和 II 上均取得了领先成绩，展示了其在深度研究任务上的工程优化能力。

## 局限性

1. **评分规则的完备性**：9,430 条规则可能无法完全覆盖所有重要的评估方面，某些隐性知识可能难以用规则表达
2. **英文为主**：基准主要基于英文专家报告，跨语言和跨文化的覆盖不足
3. **静态评估**：评分规则一旦确定即固定，无法适应领域知识的快速更新
4. **人工成本高昂**：400+ 小时的专家审核使得基准的扩展和更新代价较大
5. **二值判定的局限**：某些评估要求可能存在程度差异（部分满足），二值判定可能过于粗糙
6. **LLM 评判者偏差**：使用 LLM 进行评判可能引入系统性偏差，特别是在评估需要深度领域知识的规则时
7. **领域不均衡**：22 个领域的任务和规则分布可能不均匀，某些领域的评估可能不够充分

## 核心要点

- DRB2 通过 9,430 条从专家报告提取的细粒度二值规则实现了前所未有的评估精度和诊断能力
- 四阶段流水线（LLM 提取 -> 自评估过滤 -> 人工清洗 -> 专家精炼）是高质量规则构建的方法论贡献
- 信息召回/分析/呈现三维度评估体系使研究者能精准定位系统瓶颈
- 当前最强系统仅满足约 50% 的专家规则，深度研究仍有巨大提升空间
- 不同系统在三个维度上呈现差异化的能力画像，反映了设计理念的不同
- 基于专家报告的规则可有效识别精细幻觉，提升评估的可靠性
