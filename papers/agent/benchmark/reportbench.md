# ReportBench: 通过学术综述任务评估深度研究智能体

## 简要总结

ReportBench 是由字节跳动 BandAI 团队开发的深度研究智能体评估基准，以学术综述论文（survey paper）为任务形式，系统性地评估生成报告的内容质量。与其他基准不同，ReportBench 聚焦于两个学术研究中至关重要的维度：（1）引用文献的质量与相关性；（2）报告中陈述的忠实度与真实性。方法论上，ReportBench 以 arXiv 上已发表的高质量综述论文作为黄金标准参考，通过逆向提示工程（reverse prompt engineering）生成任务提示词，并构建了一个基于智能体的自动评估框架。实验表明，OpenAI 和 Google 等商业深度研究系统生成的报告显著优于单独使用搜索增强的 LLM，但在覆盖广度和事实一致性方面仍有较大改进空间。

## 动机与问题

### 学术综述生成的独特挑战

学术综述论文是深度研究的典型任务形式，对生成系统提出了极高的要求：

1. **文献覆盖的全面性**：综述需要广泛覆盖相关领域的重要工作，遗漏关键文献将严重影响综述质量
2. **引用的准确性**：每一个引用的论文和观点必须真实存在且正确引用，虚构引用（hallucinated citations）是一个严重问题
3. **陈述的忠实度**：引用某篇论文后对其内容的描述必须与原文一致，不能曲解或编造
4. **事实的可验证性**：即使不直接引用文献的陈述也必须事实正确，不能包含虚假信息

### 现有评估的盲区

当前的深度研究评估基准主要关注信息的全面性和分析质量，但对以下关键方面关注不足：

- **引用质量的系统性评估**：缺乏对生成报告中引用文献的全面验证机制
- **引用忠实度的验证**：即使引用了真实文献，对文献内容的描述是否准确也缺乏检验
- **非引用陈述的事实检查**：报告中大量未明确引用来源的陈述的真实性难以验证
- **与领域标准的对齐**：缺乏以专业学术综述为标准的比较基线

## 方法

### 基于综述论文的任务构建

ReportBench 的任务构建流程如下：

1. **黄金标准选取**：从 arXiv 上精选已发表的高质量综述论文作为参考标准，每篇参考综述平均包含约 153 篇参考文献
2. **逆向提示工程**：根据参考综述的主题和范围，通过逆向工程生成对应的领域特定研究提示词（domain-specific prompts）
3. **评估语料库构建**：建立包含参考综述、提示词和评估标准的综合评估语料库

### 基于智能体的自动评估框架

ReportBench 开发了一个完整的基于智能体的评估流水线：

**引用提取与分析**：
- 自动从生成的报告中提取所有引用的文献信息
- 将提取的引用列表与参考综述的引用列表进行比对
- 计算引用重叠率（Citation Overlap Ratio）作为报告整体质量的指标

**引用陈述的忠实度验证**：
- 对报告中明确引用了某文献的陈述，通过语义匹配（Semantic Matching）验证陈述内容与原始文献的一致性
- 检查生成的描述是否准确反映了被引用论文的实际内容

**非引用陈述的事实验证**：
- 对报告中未明确引用来源的陈述，采用多模型投票机制（Multi-model Voting）进行事实验证
- 利用多个网络连接模型（web-connected models）独立验证每条陈述
- 通过投票机制减少单一模型的判断偏差

**整体质量评估**：
- 综合引用覆盖率、引用忠实度和非引用陈述准确率三个维度给出综合评价

### 评估指标体系

1. **引用匹配率（Citation Match Rate）**：生成报告引用的文献与参考综述引用的重叠比例
2. **引用陈述忠实度（Cited Statement Faithfulness）**：引用内容与原始来源的语义一致性
3. **非引用陈述事实正确率（Non-cited Statement Factual Accuracy）**：通过多模型投票验证的事实准确率
4. **引用与非引用陈述数量**：报告中引用和非引用陈述的总量

## 关键创新

1. **学术综述作为评估任务**：首次系统性地以学术综述论文生成作为深度研究评估的核心任务，该任务形式天然包含了对引用质量和忠实度的高要求
2. **逆向提示工程**：从已有高质量综述逆向生成提示词的方法确保了任务的合理性和可评估性
3. **双维度评估范式**：同时评估引用文献质量和陈述忠实度，覆盖了学术写作中最关键的两个方面
4. **多模型投票事实检查**：利用多个模型独立验证非引用陈述，通过集成减少单模型偏差
5. **语义匹配忠实度验证**：不仅检查是否引用了正确的论文，还验证对论文内容的描述是否忠实
6. **arXiv 综述作为黄金标准**：利用已发表综述论文作为参考，提供了客观、权威的评估基线
7. **全自动化评估流水线**：从引用提取到事实验证的全流程自动化，使大规模评估成为可能

## 实验结果

### 商业系统 vs. 搜索增强 LLM

| 指标 | OpenAI Deep Research | 搜索增强 LLM |
|------|---------------------|-------------|
| 平均引用陈述数 | 88.2 | 16.16 |
| 平均非引用陈述数 | 38.9 | 11.51 |
| 引用匹配率 | 78.87% | 31.43% |
| 事实正确率 | 95.83% | 82.22% |

### 核心发现

1. **商业系统显著领先**：OpenAI 和 Google 等商业深度研究智能体在报告的全面性和可靠性上显著优于单独使用搜索或浏览工具增强的 LLM
2. **引用数量差距悬殊**：商业系统生成的引用陈述数量（88.2）远高于搜索增强 LLM（16.16），表明前者的文献覆盖能力远超后者
3. **引用匹配率差异巨大**：商业系统的引用匹配率（78.87%）是搜索增强 LLM（31.43%）的 2.5 倍
4. **事实正确率差异较小但仍存在**：商业系统（95.83%）虽然也不完美，但事实正确率明显高于搜索增强 LLM（82.22%）
5. **仍有改进空间**：即使是最好的商业系统，在研究覆盖的广度和深度以及事实一致性方面仍有较大改进余地

### 参考综述统计

- 参考综述平均包含约 153 篇参考文献
- 许多引用支持了相同或重叠的陈述，反映了学术综述中引用的冗余性和互证性

### 引用质量的深层分析

实验进一步揭示了引用质量的几个细微但重要的方面：

- **虚构引用问题**：搜索增强 LLM 更容易生成虚构的论文引用（即引用不存在的论文），而商业深度研究系统由于具备真实的网络搜索和论文检索能力，此类问题较少
- **引用深度不足**：即使引用了正确的论文，生成的描述往往停留在论文的表面贡献，缺乏对方法细节和实验发现的深入讨论
- **引用覆盖的不均衡**：生成报告倾向于引用高影响力论文，而忽略了领域内的重要但引用量较低的工作

## 局限性

1. **学术领域偏向**：以 arXiv 综述论文为基准，主要覆盖计算机科学和相关领域，对人文社科等领域的适用性未验证
2. **综述形式单一**：仅评估学术综述这一种报告形式，未覆盖其他类型的深度研究输出（如行业分析、政策建议等）
3. **参考标准的局限**：以单篇综述论文的引用列表作为参考，可能无法覆盖所有合理的引用选择
4. **语义匹配的精度**：当前的语义匹配方法可能无法捕捉所有类型的不忠实描述（如微妙的曲解或过度简化）
5. **多模型投票的可靠性**：多模型投票机制的准确性受限于参与投票的模型本身的能力和知识
6. **时效性约束**：基于已发表综述的任务可能无法评估系统处理最新研究进展的能力
7. **缺乏结构性评估**：未评估报告的逻辑结构、论证连贯性等方面的质量

## 核心要点

- ReportBench 将学术综述论文生成作为深度研究的核心评估任务，填补了引用质量和忠实度评估的空白
- 双维度评估范式（引用质量 + 陈述忠实度）抓住了学术研究报告质量的关键方面
- 商业深度研究系统在引用覆盖和事实准确性上显著优于搜索增强 LLM，但仍远未达到专家水平
- OpenAI Deep Research 的引用匹配率（78.87%）和引用陈述数量（88.2）远超搜索增强 LLM
- 多模型投票加上语义匹配的评估流水线提供了一种可扩展的自动化评估方案
- 逆向提示工程为从已有高质量文档构建评估任务提供了有效的方法论
- 代码公开于 GitHub（ByteDance-BandAI/ReportBench）
- 与 DeepResearch Bench II（基于专家报告规则）和 DeepResearchEval（自适应维度评估）互补，ReportBench 专注于引用质量和忠实度这一独特且关键的维度
- 该工作对学术写作辅助工具和 AI 研究助手的开发具有直接的指导意义
