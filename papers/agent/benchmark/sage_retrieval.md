# SAGE: 深度研究智能体检索基准测试与改进

## 简要总结

SAGE 是一个面向深度研究智能体的科学文献检索基准，包含 1,200 个查询、覆盖四个科学领域，并配备一个包含 200,000 篇论文的检索语料库。该基准评估了六个深度研究智能体系统，揭示了一个核心发现：所有系统在推理密集型检索任务上均表现不佳。最引人注目的发现是，传统的稀疏检索方法 BM25 在短问题上以约 30% 的优势显著超越了基于 LLM 的检索器（如 ReasonIR 和 gte-Qwen2-7B-instruct）。论文进一步提出了一种语料库级别的测试时扩展（corpus-level test-time scaling）框架，通过 LLM 对文档进行元数据和关键词增强，使通用检索器获得了额外 8% 和 2% 的性能增益。

## 动机与问题

### 深度研究中检索的核心地位

深度研究智能体（Deep Research Agent）的核心工作流程是：接收复杂的研究问题 -> 分解为子查询 -> 通过检索获取相关文献 -> 综合生成研究报告。在这一流程中，检索质量直接决定了最终报告的信息覆盖率和准确性。然而，现有评估主要关注端到端的报告质量，而忽略了对检索环节本身的深入分析。

### 现有检索评估的不足

1. **缺乏针对性基准**：大多数深度研究评估关注最终报告质量，没有专门针对检索环节的标准化测试
2. **检索器选择缺乏实证**：深度研究系统通常默认使用基于 LLM 的语义检索器，但缺乏与传统检索方法的系统比较
3. **子查询特性被忽视**：智能体生成的子查询与用户原始查询在形式和语义上存在显著差异，但检索器的训练和评估未考虑这一特性
4. **语料库适配问题**：检索器在通用数据集上训练，但科学文献语料库具有独特的词汇和结构特征，通用检索器可能不适配

### 查询-检索器不匹配问题（Query-Retriever Mismatch）

论文发现了一个关键的系统性问题：尽管 LLM 检索器在自然语言查询上进行训练，但深度研究智能体倾向于生成关键词风格的子查询（keyword-like sub-queries），这与检索器的训练分布严重不匹配。这种不匹配导致 LLM 检索器无法充分发挥其语义理解能力，反而被简单的词汇匹配方法所超越。

## 方法

### 基准构建

**四个科学领域**：SAGE 的查询和语料库覆盖四个不同的科学研究领域，确保评估的跨领域代表性。每个领域的查询都经过精心设计，涵盖从简单事实查找到复杂推理检索的不同难度层次。

**1,200 个查询**：查询被划分为两种类型：
- **短问题（Short-form questions）**：需要精确答案的事实性问题，使用精确匹配（Exact Match, EM）进行评估
- **开放性问题（Open-ended questions）**：需要综合多篇文献进行推理和综合的复杂问题，使用召回率（Recall）进行评估

**200,000 篇论文语料库**：包含来自四个领域的科学论文全文，用于模拟真实的大规模科学文献检索场景。

### 智能体评估

评估了六个深度研究智能体系统，使用 DR Tulu 作为所有语料库搜索实验的骨架智能体（backbone agent），搭配不同的检索器进行对比：

- **BM25**：经典的基于 TF-IDF 的稀疏检索方法
- **gte-Qwen2-7B-instruct**：基于 Qwen2 架构的通用 LLM 嵌入检索器
- **ReasonIR**：专门针对推理密集型检索任务训练的 LLM 检索器

### 语料库级别测试时扩展框架

这是论文的方法论核心创新。该框架在检索之前使用 LLM 对语料库文档进行离线增强：

1. **元数据增强**：利用 LLM 为每篇论文生成结构化元数据标注，包括研究领域分类、方法论标签、关键发现摘要等
2. **关键词扩展**：LLM 自动为文档生成扩展关键词集合，涵盖同义词、相关概念和潜在的查询意图匹配词
3. **文档表示增强**：将 LLM 生成的元数据和关键词融合到文档的索引表示中，使检索器能够匹配更广泛的查询模式

这种方法的核心思想是：与其在查询侧改进（即改进子查询生成），不如在文档侧增强（即让文档更容易被找到）。这本质上是一种将计算资源投入到预处理阶段的策略，通过离线的语料库增强来提升在线检索效率。

## 关键创新

1. **首个专注于检索环节的深度研究基准**：SAGE 是第一个将评估焦点从端到端报告质量转移到检索组件的基准，填补了深度研究评估的关键空白
2. **查询-检索器不匹配假说的实证验证**：系统性地证明了智能体子查询风格与 LLM 检索器训练分布之间的不匹配，为检索器选择提供了实证依据
3. **BM25 优势的反直觉发现**：在语义检索被广泛认为优于词汇匹配的背景下，证明了 BM25 在特定应用场景下仍具有显著优势（约 30% 的性能领先）
4. **语料库级别测试时扩展**：提出了一种新颖的文档侧增强策略，无需修改检索器或智能体架构即可提升检索性能
5. **大规模科学文献检索场景**：200K 论文的语料库规模贴近真实应用需求，相比小规模评估集具有更高的实用参考价值

## 实验结果

### BM25 vs LLM 检索器

| 检索器 | 短问题 EM | 开放性问题 Recall |
|--------|-----------|-------------------|
| BM25 | ~81.2% | ~30.7% |
| gte-Qwen2-7B-instruct | ~63% | ~31% |
| ReasonIR | 最低 | 最低 |

**关键发现**：

1. **短问题上 BM25 大幅领先**：在短问题上，BM25 以约 30 个百分点的优势显著超越 LLM 检索器。这表明稀疏词汇匹配更适合多约束证据检索（multi-constraint evidence retrieval）场景
2. **开放性问题上差距缩小**：在开放性问题上，BM25 和 gte-Qwen2-7B-instruct 的性能基本持平，语义理解在此类问题上的优势开始显现
3. **ReasonIR 排名垫底**：尽管 ReasonIR 专门为推理检索设计，但在两种问题类型上均表现最差，说明其训练分布与智能体子查询模式不匹配
4. **gte-Qwen2 的不对称表现**：gte-Qwen2-7B-instruct 在开放性问题上与 BM25 持平甚至略好，但在短问题上落后约 18 个百分点，反映了不同查询风格对检索器的差异化影响

### 语料库级别扩展效果

通过语料库级别的测试时扩展框架：
- **短问题**：获得约 **8%** 的性能提升
- **开放性问题**：获得约 **2%** 的性能提升

这一结果表明文档侧增强策略确实有效，尤其在需要精确匹配的短问题场景中效果更为显著。

### 六个深度研究智能体的共性问题

所有评估的六个深度研究系统都在推理密集型检索任务上存在困难，表现为：
- 子查询生成质量不稳定：智能体在分解复杂研究问题时，生成的子查询往往过于笼统或过于狭窄
- 检索结果筛选和重排能力不足：从大量检索结果中选择最相关的文档仍是瓶颈
- 多轮检索策略调整缺乏自适应性：当初始检索未能获得满意结果时，智能体难以有效地调整搜索策略
- 跨文档综合推理薄弱：在需要从多篇论文中提取和整合信息时，准确性显著下降

### 查询-检索器不匹配的深层分析

论文深入分析了为什么 LLM 检索器在该场景下反而不如 BM25：

1. **训练分布差异**：LLM 检索器通常在自然语言形式的查询-文档对上训练，但智能体生成的子查询更接近布尔搜索或关键词组合的形式
2. **语义过度泛化**：LLM 检索器的语义匹配能力在关键词查询场景下反而成为劣势——它可能将语义相近但实际不相关的文档排在前列
3. **精确约束的丢失**：当查询包含多个精确约束条件（如特定年份、特定方法、特定数据集）时，BM25 的词汇精确匹配优势尤为明显
4. **推理型检索器的困境**：ReasonIR 虽然针对推理任务训练，但其训练场景与深度研究智能体的实际使用模式仍存在差距

### 对未来检索器设计的启示

SAGE 的发现对检索器设计提出了重要启示：
- 检索器训练需要考虑下游智能体的实际查询分布，而非仅关注人类撰写的查询
- 混合检索策略（结合稀疏和稠密检索）可能是更优的选择
- 文档侧增强与查询侧优化应该协同进行，而非相互替代
- 科学文献检索可能需要领域适应的检索器，通用检索器的迁移效果有限

## 局限性

1. **科学领域覆盖有限**：仅涵盖四个科学领域，可能无法代表更广泛的学科差异（如人文社科、工程技术等）
2. **英文为主**：基准主要基于英文科学文献，未考虑多语言检索场景
3. **静态语料库**：200K 论文的语料库是固定的，无法反映真实场景中持续更新的文献库
4. **单一骨架智能体**：所有实验使用 DR Tulu 作为骨架，结论可能受到该特定智能体的子查询风格影响
5. **语料库增强的计算成本**：LLM 对 200K 文档进行离线增强需要大量计算资源，实际部署成本未充分讨论
6. **检索器种类有限**：仅评估了三种检索器，未包含如 ColBERT、SPLADE 等中间形态的检索方法
7. **子查询风格的代表性**：智能体子查询风格可能随模型版本迭代而变化，当前发现的不匹配问题可能是暂时性的
8. **缺少端到端验证**：未充分验证检索质量的提升是否必然带来最终报告质量的相应改善

## 核心要点

- SAGE 是首个专注于深度研究智能体检索环节的基准，包含 1,200 查询和 200K 论文语料库
- 核心发现：BM25 在短问题上以约 30% 优势超越 LLM 检索器，原因是智能体生成的关键词式子查询与 LLM 检索器的训练分布不匹配（Query-Retriever Mismatch）
- 开放性问题上 BM25 和 gte-Qwen2-7B-instruct 持平，ReasonIR 在所有任务上表现最差
- 语料库级别测试时扩展通过 LLM 增强文档元数据和关键词，在短问题和开放性问题上分别获得 8% 和 2% 的提升
- 文档侧增强（而非查询侧改进）是一种成本效益更高的检索优化策略
- 所有六个深度研究系统在推理密集型检索上均表现不佳，检索仍是制约深度研究质量的关键瓶颈
- 代码公开于 GitHub（HughieHu/Sage），可用于复现实验和评估新的检索方法
