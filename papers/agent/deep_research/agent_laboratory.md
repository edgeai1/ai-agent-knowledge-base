## 摘要

Agent Laboratory 是一个端到端的自主研究框架，接受人类提供的研究想法，经过三个阶段——文献综述、实验和报告撰写——使用专门的大语言模型智能体。与先前方法（AI Scientist）相比，它实现了 84% 的成本降低，o1-preview 产出最佳结果。带有人类反馈的协同驾驶模式可将论文质量提升 +0.58 个 NeurIPS 评分点。

## 动机与问题

现有的自主研究系统面临若干挑战：

1. **高成本**：AI Scientist 每篇论文约花费 $15，限制了可及性。
2. **无人类指导**：完全自主的系统错失了人类专业知识在关键决策点改善质量的机会。
3. **刚性流程**：先前系统缺乏模块化的阶段分离，难以识别和修复故障点。
4. **质量差距**：AI 生成的论文远低于顶级会议的接收阈值（NeurIPS 平均接收分数：5.85/10）。

Agent Laboratory 旨在创建一个具有成本效益的模块化研究流程，支持完全自主和人类指导（协同驾驶）两种操作模式。

### 阶段 1：文献综述

- **智能体**：博士生
- **工具**：arXiv API，支持命令：`summary`、`full text`、`add paper`
- **过程**：迭代查询检索前 20 篇相关论文，评估相关性，优化直至收集 N 篇论文
- **瓶颈**：60-80% 成功率（所有阶段中最低）；模型在指令遵循方面困难，有时会重复 `summarize` 直到达到最大步数

### 阶段 2：实验

#### 2a. 计划制定（博士生 + 博士后对话）
- 输出：要实现的机器学习模型、数据集、高级步骤

#### 2b. 数据准备（机器学习工程师）
- HuggingFace 数据集搜索 + Python 代码生成
- 迭代编译直到无 bug

#### 2c. 运行实验（mle-solver -- 核心创新）

```
算法：mle-solver
-------------------------------------------------
输入：research_plan, initial_code
输出：best_program, results

        top_programs.replace_lowest(best_variant)

    // E. 自我反思
```

#### 2d. 结果解读（博士生 + 博士后对话）
- 从实验结果中得出有意义的见解

### 阶段 3：报告撰写

#### 3a. 框架生成（paper-solver）
- 博士生 + 教授生成包含 8 个章节的 LaTeX：
  摘要、引言、背景、相关工作、方法、实验设置、结果、讨论

#### 3b-3c. ArXiv 研究 + 编辑
- 逐行 LaTeX 修改，带编译验证

#### 3d-3e. 审查 + 修订
- 3 个审稿人智能体生成 NeurIPS 风格的审查意见
- 评价标准：原创性、质量、清晰度、重要性、严谨性、表达（各 0-4 分），总评（0-10 分）
- 审查准确率：65% vs 人类 66%；F1：0.57 vs 人类 0.49
- 如果分数足够高：定稿。如果较低：回到早期阶段。

### 操作模式

| 模式 | 人类角色 | 优势 |
|------|-----------|---------|
| 自主模式 | 仅提供初始想法 | 完全自动化 |
| 协同驾驶模式 | 在每个子任务检查点审查 + 反馈 | 提升 +0.58 个 NeurIPS 评分点 |

## 关键创新

1. **模块化三阶段流程**：清晰的分离使得可以进行针对性改进
2. **mle-solver**：基于池的进化式代码优化，配合大语言模型评分
3. **双操作模式**：自主 + 协同驾驶，带人工检查点
4. **成本效率**：$2.33/篇（gpt-4o）vs. ~$15（AI Scientist）= 84% 成本降低
5. **NeurIPS 风格自动审查**：3 个审稿人智能体配备校准的评价标准
6. **批并行化探索**：每步多个同时修改

## 实验设置

- **测试的后端**：gpt-4o、o1-mini、o1-preview
- **测试的研究主题**：5 个问题（认知偏差、图像 Transformer、性别效应、医学问答、词序敏感性）
- **MLE-Bench**：10 个机器学习挑战，用于 mle-solver 评估
- **人工评估**：由外部审稿人进行 NeurIPS 风格的审查
- **协同驾驶研究**：参与者使用预选和自定义主题

## 局限性

1. **固定论文结构**：8 个必需章节阻止了创造性组织。
2. **图表限制**：每篇论文最多 2 个图表。
3. **质量差距**：论文平均 3.8/10 vs. NeurIPS 接收阈值 5.85/10。
4. **文献综述瓶颈**：60-80% 成功率；模型循环执行 `summarize`。
5. **幻觉**：gpt-4o 捏造实验细节（批大小、轮次），这些实际上从未在代码执行中发生。
6. **token 溢出**：检索的论文和打印输出可能超过 token 限制。
7. **自我评估偏差**：大语言模型审稿人高估质量 2.3 分。
8. **mle-solver 失败**：约 20% 的实验产生 0% 准确率且无法恢复；生成 `exit()` 和 `subprocess.run()` 命令需要手动干预。
9. **代码编辑偏差**：倾向于修改第 0 行，偏好 `REPLACE` 而非 `EDIT`。
10. **伦理问题**：存在用低质量提交淹没同行评审的风险。

## 核心要点

1. **成本效率是可实现的**：84% 的成本降低表明自主研究可以变得更加可及（使用 gpt-4o 每篇 $2.33）。
2. **人类参与至关重要**：协同驾驶模式有意义地提升质量（+0.58 分），75% 的用户继续使用该工具。
3. **o1-preview 产出最佳研究**：尽管最昂贵，但产出最高的整体质量（4.0/10）和最佳实验质量。
4. **报告撰写主导成本**：gpt-4o 成本的 74% 和 o1-preview 成本的 73% 来自报告撰写阶段。
5. **自动审查不可靠**：2.3 分的高估差距意味着自动审查无法替代人工对论文质量的判断。
6. **mle-solver 具有竞争力**：在 MLE-Bench 上以 100% 提交率超越 AIDE 和 OpenHands，表明进化式方法具有鲁棒性。
7. **流程模块化支持诊断**：阶段级成功率清楚地识别出文献综述是最薄弱环节（60-80%）。
8. **距离会议质量仍有差距**：与 NeurIPS 接收阈值相差 1.85 分，表明在新颖性、深度和方法论严谨性方面需要根本性改进，才能让自主论文在顶级会议上具有竞争力。
