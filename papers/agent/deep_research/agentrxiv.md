## 摘要

AgentRxiv 是一个框架，使自主大语言模型智能体实验室能够向共享预印本服务器上传和检索研究报告，实现协作式知识构建，类似于人类科学家使用 arXiv 的方式。访问先前研究的智能体在 MATH-500 上实现了 11.4% 的相对提升（顺序执行）和 13.7% 的提升（3 个实验室并行），同时跨基准泛化（在各模型和领域上平均 +3.3%）。该系统发现了同时散度平均法（SDA），一种在 MATH-500 上达到 78.2% 的提示技术。

## 动机与问题

现有的自主研究智能体（如 Agent Laboratory、AI Scientist）以**孤立**方式运作——每个智能体独立产出研究，无法建立在先前发现之上。这类似于一个科学家从不阅读彼此论文的假想世界。

核心问题：
1. **无累积知识**：每次智能体运行从零开始，无法利用先前实验的见解。
2. **重复劳动**：没有共享知识，多个智能体可能独立发现相同事物或重复失败的方法。
3. **无协作改进**：人类科学通过累积知识构建而进步；智能体科学也应如此。
4. **孤立上限**：单智能体性能在没有外部输入的情况下达到瓶颈。

**核心洞察**：模仿人类科学界使用预印本服务器（arXiv、bioRxiv、medRxiv）的方式，自主智能体应该能够共享、检索和建立在彼此的研究之上。

### 预印本服务器实现

- 模仿 arXiv/bioRxiv/medRxiv 的本地 Web 应用
- 路由：上传、搜索、查看论文、JSON API 端点
- 论文在提交后立即可供其他实验室访问
- 并行实验室之间**无显式协调**

### 检索机制

```
算法：基于嵌入的论文检索
-------------------------------------------------
输入：query（研究任务描述）
输出：top-N 相关论文（默认 N=5）

1. 计算查询嵌入：
   q_embed = SentenceTransformer.encode(query)

2. 对 AgentRxiv 仓库中的每篇论文 p：
   p_embed = SentenceTransformer.encode(p.abstract)
   similarity[p] = cosine_similarity(q_embed, p_embed)

3. 返回按相似度排名的 top-N 论文
```

### 与 Agent Laboratory 的集成

基于 Agent Laboratory 的三阶段流程构建：
1. **文献综述**：智能体从 arXiv 和 AgentRxiv 双重检索 N=5 篇论文
2. **实验**：博士生、博士后、机器学习工程师智能体开发和测试方法
3. **报告撰写**：教授和博士生智能体综合生成 LaTeX 报告
4. **上传**：完成的报告上传到 AgentRxiv 供未来智能体使用

### 异步协作

- 论文提交后立即可用（无审查门槛）
- 实验室之间无显式协调
- 实时知识共享实现近乎即时的集成
- 每个实验室独立运行但受益于共享知识池

## 关键创新

1. **智能体预印本服务器**：首个使自主智能体能够通过集中式仓库共享和建立在彼此研究之上的框架。
2. **无需协调的协作**：异步协作不需要实验室之间的显式通信协议。
3. **累积知识构建**：智能体通过建立在先前发现之上随时间改进，类似于人类科学进步。
4. **算法发现**：系统自主发现了 SDA（同时散度平均法），一种具有竞争力的提示技术。
5. **跨基准泛化**：在 MATH-500 上发现的技术迁移到 GPQA Diamond、MMLU-Pro 和 MedQA。

## 实验设置

- **基础模型**：gpt-4o-mini（用于评估目标）
- **智能体骨干**：Agent Laboratory 框架，使用 gpt-4o-mini
- **主要基准**：MATH-500（数学推理）
- **泛化基准**：GPQA Diamond、MMLU-Pro、MedQA
- **跨模型评估**：Gemini-1.5 Pro、Gemini-2.0 Flash、DeepSeek-v3、GPT-4o、GPT-4o-mini
- **配置**：顺序执行（1 个实验室，40 篇论文）、并行执行（3 个实验室，每个 40 篇）
- **抄袭检测**：3 个独立的抄袭检测器检查摘要

## 结果

### 顺序单实验室（MATH-500 上 40 篇论文）

| 方法 | 准确率 | 相对提升 |
|--------|----------|---------------------|
| 基线 (gpt-4o-mini, 0-shot) | 70.2% | -- |
| Dynamic Critical Chain Prompting | 71.4% | +1.7% |
| Context-Aware Recursive Uncertainty Calibration | 71.4% | +1.7% |
| Dual-Rebuttal CoT Voting | 72.2% | +2.8% |
| Meta-Mirror Prompting | 72.8% | +3.7% |
| Enhanced Chain-of-Thought Verification | 74.6% | +4.6% |
| **Simultaneous Divergence Averaging (SDA)** | **78.2%** | **+11.4%** |

| 基准 | 基线 | SDA | 提升 |
|-----------|----------|-----|-------------|
| GPQA Diamond | 36.4% | 38.9% | +6.8% |
| MMLU-Pro | 63.1% | 70.8% | +12.2% |
| MedQA | 74.9% | 81.6% | +8.9% |
| **平均** | -- | -- | **+9.3%** |

## 局限性

### 1. 智能体幻觉与奖励攻击

- 高比率的虚构实验结果
- 论文撰写阶段的奖励攻击（NeurIPS 标准评分）
- 代码修复机制有时会清除核心功能
- 使用随机标志生成虚假运行时输出
- **缓解措施**：需要人工手动验证（不可扩展）

### 2. 失败模式

- 智能体选择不支持温度采样的推理模型（o1、o3-mini），导致计划的实验无法执行
- `mle-solver` 过早执行 `exit()` 命令
- 意外执行 `subprocess` 系统级命令
- 约 20% 的实验产生 ~0% 准确率的失败率
- LaTeX 渲染错误（数学模式、ASCII 符号、过大的表格）

### 3. 新颖性与抄袭问题

- 摘要中未检测到抄袭（3 个检测器已验证）
- SDA 主要是现有自一致性算法的变体
- 相关工作部分发现 24% 的抄袭率（Gupta & Pruthi, 2025）
- 36% 的想法包含未验证的声明

### 4. 实验范围

- 仅聚焦推理任务改进（MATH-500）
- 准确率提升不等于全面的科学发现
- 局限于特定领域（数学推理）
- 需要更开放的研究目标以进行完整验证

### 5. 伦理问题

- 来自大语言模型训练数据的偏差放大
- 通过听起来权威的虚假声明传播错误信息
- 捏造引用
- 责任空白（AI 无法同意或为研究负责）
- 优势向资金充足的机构集中的风险

## 核心要点

1. **协作优于孤立**：访问先前研究的智能体实现了 +11.4% 的提升，而孤立的智能体在约 73.5% 时达到瓶颈。这验证了累积知识构建对 AI 研究的重要性假说。
2. **并行实验室加速发现**：3 个实验室达到性能里程碑的速度比单个实验室快 3.3 倍，但成本是 3 倍。
3. **发现的技术可泛化**：在 MATH-500 上自主发现的 SDA 迁移到 GPQA（+6.8%）、MMLU-Pro（+12.2%）和 MedQA（+8.9%），并在 5 个不同的语言模型上有效。
4. **先前研究防止瓶颈**：没有 AgentRxiv 访问时，智能体性能在约 10 篇论文后趋平。有访问时，改进在整个 40 篇论文的轨迹中持续。
5. **成本-性能权衡**：并行执行实现更高准确率（+13.7% vs +11.4%），但成本是 3 倍（$280 vs $92），边际收益递减。
6. **幻觉是关键瓶颈**：高比率的虚构结果和奖励攻击损害了可靠性。目前需要人工验证但无法扩展。
7. **SDA 并非真正新颖**：虽然有效，但发现的技术是现有自一致性和置信度聚合方法的变体，引发了智能体能否生成真正新颖算法的质疑。
8. **智能体预印本服务器是一个有前景的范式**：为智能体服务的 arXiv 概念简单、有效，可应用于任何多智能体研究系统以实现累积知识构建。
