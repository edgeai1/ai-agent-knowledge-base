## 摘要

AI Scientist-v2 是一个端到端的智能体系统，能够自主生成假设、编写代码、运行实验、创建可视化并撰写科学论文。它引入了由实验管理器智能体管理的渐进式智能体树搜索，消除了对人工编写模板的依赖，并产出了首篇通过同行评审的完全由 AI 生成的论文（在 ICLR 研讨会上获得 6.33/10 分，位于提交论文的前 45%）。

## 动机与问题

AI Scientist-v1 展示了自动化科学工作流程的可行性，但存在关键局限：

1. **模板依赖**：每个新课题都需要人工编写的代码模板，限制了跨领域的自主性和可部署性。
2. **线性实验**：严格的顺序假设测试阻碍了对复杂研究问题的深入探索。
3. **无视觉反馈**：缺乏 VLM 集成用于图表质量评估。
4. **范围狭窄**：无法在不手动设置的情况下泛化到多样的机器学习领域。

v2 的目标是创建一个完全自主的系统，能够在没有任何人工编写代码模板的情况下产出达到同行评审研讨会论文水平的研究。

## 方法

### 架构概览（ASCII 图）

```
  [想法生成] --> [实验管理器]
        |                     |
        v                     v
  [Semantic Scholar]    [四阶段流程]
  [文献综述]                  |
                     +-------+-------+-------+
                     |       |       |       |
                   阶段1   阶段2   阶段3   阶段4
                   初步     超参数   研究    消融
                   调查     调优     议程    研究
                     |       |       |       |
                     +---+---+---+---+---+---+
                         |
                    [最佳节点选择]
                         |
                    [论文撰写]
                         |
                    [VLM 审查循环]
                         |
                    [最终论文]
```

### 1. 通用化想法生成（第 3.1 节）

与 v1 的模板条件方法不同，v2 从更高层次的抽象开始：
- 提示进行开放式思考，探索研究方向
- 集成 Semantic Scholar 在构思过程中进行实时文献搜索
- 在投入实现之前评估新颖性和可行性
- 每个提示配置生成约 20 个候选想法

### 2. 实验进度管理器（第 3.2.1 节）

一个专门的智能体协调四个明确定义的阶段：

| 阶段 | 目的 | 停止标准 |
|-------|---------|-------------------|
| 阶段 1：初步调查 | 通过最小原型建立可行性 | 原型成功执行 |
| 阶段 2：超参数调优 | 优化关键超参数 | 训练曲线收敛；在 2+ 个数据集上成功 |
| 阶段 3：研究议程执行 | 实施核心研究议程 | 计算预算耗尽 |
| 阶段 4：消融研究 | 评估组件重要性 | 计算预算耗尽 |

每个阶段结束后，管理器：
- 通过大语言模型评估器选择最佳表现节点
- 将其推进到下一阶段作为种子
- 启动多个副本以实现统计鲁棒性（均值/标准差）

### 3. 并行化智能体树搜索（第 3.2.2 节）

```
算法：渐进式智能体树搜索
-------------------------------------------------
输入：research_idea, num_workers, max_nodes_per_stage
输出：best_experiment_node

for stage in [初步调查, 超参数调优, 研究议程, 消融研究]:
    root = 前一阶段的最佳节点（或阶段 1 的想法）
    tree = initialize_tree(root)

    for step in range(max_nodes_per_stage):
        // 选择要扩展的节点
        nodes_to_expand = []
        for w in range(num_workers):
            with probability p_debug:
                选择有 bug 的节点进行调试
            else:
                通过最佳优先搜索选择无 bug 节点
                （大语言模型评估：指标、训练动态、图表质量）
            nodes_to_expand.append(selected_node)

        // 并行扩展
        for node in nodes_to_expand (parallel):
            child = LLM.generate(plan + code, parent=node)
            execute(child.code)
            if error:
                child.status = BUGGY
                child.error_trace = error_msg
            else:
                save_metrics_to_numpy(child)
                figures = generate_plots(child)
                vlm_feedback = VLM.critique(figures)
                if vlm_flags_issues:
                    child.status = BUGGY
                else:
                    child.status = NON_BUGGY

    best_node = LLM_evaluator.select_best(tree)
    launch_replications(best_node, n_seeds)
```

### 节点类型

- **无 bug 节点**：成功执行；可进行优化
- **有 bug 节点**：执行失败或被 VLM 标记；可进行调试
- **超参数节点**（阶段 2）：探索不同的超参数配置
- **消融节点**（阶段 4）：评估组件重要性
- **副本节点**：使用不同随机种子重新运行父实验
- **聚合节点**：将副本结果合并为摘要图表

### 4. VLM 审查者集成（第 3.4 节）

VLM 在两个阶段使用：
- **实验阶段**：对生成的图表进行即时反馈（标签、图例、清晰度）
- **论文撰写阶段**：评估图表 + 标题与引用文本的一致性

质量检查包括：图表-标题对齐、视觉清晰度、缺失图例/标签、正文和附录之间的图表重复检测。

### 5. 论文撰写

替换了 v1 基于 Aider 的迭代方法：
- 一次性生成完整论文
- 由推理模型（o1）驱动的单独反思阶段
- 图表优化的 VLM 反馈循环
- 通过 PDF 编译检查强制执行页数限制

## 关键创新

| 特性 | v1 | v2 |
|---------|----|----|
| 代码库起草 | 特定主题的模板 | 跨领域通用，无需模板 |
| 执行规划 | 线性、顺序 | 树状、并行 |
| 并行实验 | 否 | 是（num_workers 个并发） |
| VLM 审查者 | 否 | 是（实验 + 撰写阶段） |
| 人工评估 | 未提交到会议 | 达到研讨会接收水平 |
| 数据集处理 | 手动设置 | 通过 HuggingFace Hub 自动化 |

### 评估协议

- 3 篇完全自主的论文提交到 ICLR 2025 ICBINB 研讨会
- 包含在总共 43 篇提交中；盲审
- 审稿人被告知可能存在 AI 论文，但不知道是哪些
- 获得不列颠哥伦比亚大学 IRB 批准（H24-02652）

## 结果

### 同行评审分数

| 论文 | R1 | R2 | R3 | 平均 | 结果 |
|-------|----|----|-----|---------|---------|
| 组合正则化 | 6（弱接受） | 7（接受） | 6（弱接受） | **6.33** | **被接收**（前 45%） |
| 标签噪声与校准 | 较低 | 较低 | 较低 | 低于阈值 | 被拒绝 |
| 害虫检测挑战 | 较低 | 较低 | 较低 | 低于阈值 | 被拒绝 |

- 研讨会接收率通常为 60-80%，主会议为 20-30%
- 被接收的论文在评审后按先前与组织者的约定撤回
- 研讨会级别接收的成功率为 1/3
- 内部团队评估与审稿人决定一致

### 被接收论文主题

被接收的论文研究了神经网络的组合正则化：
- 提出了 LSTM 嵌入上的时间一致性正则化器
- 发现正则化并未提升组合泛化能力（阴性结果）
- 审稿人肯定了对方法失败原因的清晰分析

## 局限性

1. **仅研讨会级别**：没有论文达到主会议标准（20-30% 接收率的会议）；研讨会接收率为 60-80%。
2. **引用幻觉**：系统偶尔引入不准确的引用。
3. **方法论深度**：缺乏顶级会议论文所需的详细严谨性和深入分析。
4. **代码质量问题**：存在数据集重叠（被接收论文代码中发现约 57% 的训练/测试重叠），论文中存在术语混淆。
5. **1/3 成功率**：三篇提交中仅一篇通过了同行评审。
6. **仍需人工筛选**：人类从 AI 生成的池中选择初始想法，并从多个种子运行中选择最佳论文。
7. **计算成本**：每篇论文生成需要数小时到 15 小时。
8. **领域覆盖**：仅在机器学习主题上评估；未测试向其他科学领域的泛化能力。
9. **图表准确性**：VLM 审查有助于但无法捕捉所有标题不准确（例如，被接收论文中图 3 对验证损失的误读）。

## 核心要点

1. **首篇 AI 撰写的同行评审论文**：一个真正的里程碑——首篇完全由 AI 生成的论文在公认的机器学习研讨会上通过了同行评审。
2. **无模板泛化**：消除人工模板是迈向能够探索任意领域的真正自主研究智能体的关键步骤。
3. **树搜索 > 线性搜索**：具有 4 个阶段的渐进式智能体树搜索比 v1 的线性方法实现了更深入和系统的探索。
4. **VLM 集成很重要**：视觉反馈循环显著提升了图表质量和论文连贯性。
5. **阴性结果也可发表**：被接收的论文报告了阴性结果，表明 AI 科学家即使假设失败也能为科学做出贡献。
6. **质量差距依然存在**：研讨会级别的接收令人鼓舞，但与主会议标准仍有显著差距。该系统尚无法生成真正新颖、高影响力的假设或深度创新的方法论。
7. **v1 的后续发表在 Nature 上**：AI Scientist v1 随后发表在 Nature 上，确立了该研究项目的信誉。
8. **开源承诺**：完整代码库已发布，以促进社区发展和负责任地推进自动化科学发现。
