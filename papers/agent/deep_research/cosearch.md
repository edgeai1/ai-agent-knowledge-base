# CoSearch: Joint Training of Reasoning and Document Ranking via Reinforcement Learning for Agentic Search

## 简要总结

CoSearch 提出了一个**联合训练推理智能体与生成式文档排序器**的框架，通过 Group Relative Policy Optimization（GRPO）实现端到端的强化学习训练。该工作首先通过 Oracle 实验揭示了一个关键瓶颈：固定检索系统与理想检索之间存在高达 **+26.8% 的相对 F1 差距**，表明检索质量是制约智能搜索性能扩展的核心瓶颈。为解决这一问题，CoSearch 引入了**语义分组策略**和**复合奖励机制**，在 7 个单跳和多跳 QA 基准上实现了一致性提升，7B 推理智能体相比 Search-R1 提升了 +6.6% 相对 F1，3B 智能体提升了 +10.8%。

## 动机与问题

### 现有方法的局限

现有的智能搜索方法（如 Search-R1）将检索系统视为**固定工具**，仅优化推理智能体，而检索组件保持不变。这种设计假设存在根本性问题：

1. **推理智能体**通过强化学习不断改进查询策略
2. **检索系统**保持冻结状态，无法适应推理智能体的进化
3. 两者之间存在**协作鸿沟**，限制了系统整体性能

### Oracle 差距分析

为量化检索系统的瓶颈效应，作者进行了 Oracle 实验：

| 设置 | 相对 F1 差距 |
|------|-------------|
| 7B 推理智能体 + Oracle 检索 vs. 固定检索 | +14.7% |
| 3B 推理智能体 + Oracle 检索 vs. 固定检索 | +26.8% |

这一实验清楚地表明：
- 检索质量是扩展智能搜索性能的**关键瓶颈**
- 较小的推理模型对检索质量更为敏感
- 联合优化推理和检索具有巨大的改进空间

### 研究目标

设计一个框架，实现推理智能体与文档排序器的联合训练，同时解决 GRPO 在排序器训练中面临的技术挑战。

## 方法

### 1. 整体框架

CoSearch 包含两个核心组件：
- **推理智能体（Reasoning Agent）**：负责多步推理和查询生成
- **生成式文档排序器（Generative Document Ranker）**：负责对检索结果进行重排序

两个组件通过 GRPO 联合训练，共享轨迹级别的反馈信号。

### 2. 骨干模型

- **推理智能体**：Qwen2.5-7B-Instruct（主实验）/ Qwen2.5-3B-Instruct（扩展实验）
- **文档排序器**：Qwen2.5-7B-Instruct

### 3. 语义分组策略（Semantic Grouping）

**挑战**：GRPO 要求同一组内的样本具有可比性，但排序器的输入在不同推理轨迹中变化很大（不同子查询对应不同的文档集合）。

**解决方案**：CoSearch 引入语义分组策略，通过 token 级别的相似度对子查询进行聚类，形成有效的优化组：
1. 收集不同推理轨迹中的子查询
2. 计算子查询之间的 token 级相似度
3. 将相似的子查询聚类为优化组
4. 在组内进行 GRPO 优化

**优势**：无需额外的 rollout，即可为排序器构建有效的优化分组。

### 4. 复合奖励（Composite Reward）

CoSearch 设计了结合两种信号的复合奖励：

**即时信号 - 排序质量奖励**：
- 衡量排序器对当前子查询的排序质量
- 提供即时的学习信号

**长期信号 - 轨迹级结果反馈**：
- 基于最终答案的正确性
- 提供长期的学习目标
- 鼓励排序器学习对最终任务有益的排序策略

```
R_composite = α * R_ranking + β * R_trajectory
```

### 5. 联合训练流程

1. 推理智能体生成多步推理轨迹，包含多个子查询
2. 每个子查询送入检索系统获取候选文档
3. 生成式排序器对候选文档进行重排序
4. 排序后的文档返回推理智能体继续推理
5. 最终生成答案并计算奖励
6. 通过 GRPO 同时更新推理智能体和排序器的参数

## 关键创新

1. **首次实现推理与检索的联合 RL 训练**：区别于 Search-R1 等仅训练推理智能体的方法，CoSearch 同时优化了检索组件
2. **语义分组策略**：创造性地解决了 GRPO 在排序器训练中输入异构性的技术挑战
3. **复合奖励设计**：融合即时排序信号和长期轨迹反馈，为排序器提供多粒度的学习信号
4. **Oracle 差距分析**：通过实验量化了检索瓶颈的影响，为联合训练的必要性提供了强有力的证据

## 实验结果

### 评估基准

7 个 QA 基准，涵盖单跳和多跳推理：

**单跳**：PopQA, Natural Questions (NQ), TriviaQA
**多跳**：HotpotQA, 2WikiMultiHopQA, Musique, Bamboogle

### 主实验结果（7B 推理智能体）

| 方法 | 平均 F1 | 相对提升 |
|------|---------|---------|
| Search-R1 | 基准 | - |
| Retrieval Only | 基准 | - |
| Fixed Ranker | 基准 | - |
| CoSearch（联合训练） | **0.568** | **+6.6%** vs Search-R1 |

CoSearch 在所有 7 个基准上均实现了一致性提升。

### 3B 推理智能体结果

| 方法 | 相对提升 |
|------|---------|
| CoSearch（3B Agent + 7B Ranker） | **+10.8%** vs Search-R1 |

较小的推理模型从联合训练中获益更大，与 Oracle 差距分析的结论一致。

### 消融实验

消融实验验证了每个设计选择的贡献：
- **去除语义分组**：性能下降，说明分组策略对排序器 GRPO 训练至关重要
- **去除排序质量奖励**：性能下降，说明即时信号对排序器学习不可或缺
- **去除轨迹反馈**：性能下降，说明长期信号帮助排序器学习全局有益的策略
- **冻结排序器**：退化为固定排序器基线，性能显著下降

## 局限性

1. **计算开销**：联合训练两个大型模型（推理智能体 + 排序器）的计算成本显著高于仅训练推理智能体
2. **排序器与检索器的解耦**：当前工作仅优化重排序阶段，底层检索器（如 BM25 或密集检索）仍保持固定，未来可探索端到端优化
3. **规模限制**：实验主要在 3B 和 7B 模型上验证，对更大规模模型的效果尚不确定
4. **基准覆盖**：主要评估在传统 QA 基准上，对更具挑战性的深度研究场景（如 BrowseComp）的泛化性有待验证
5. **语义分组的粒度**：基于 token 级相似度的分组可能在语义差异较大的子查询上不够精确
6. **部署复杂度**：需要同时部署推理智能体和排序器两个模型，增加了系统部署的复杂性

## 核心要点

1. 检索系统是扩展智能搜索性能的核心瓶颈，Oracle 实验显示固定检索与理想检索之间存在高达 +26.8% 的 F1 差距
2. CoSearch 首次实现了推理智能体与文档排序器的联合 RL 训练，证明了联合优化的可行性和有效性
3. 语义分组策略巧妙解决了 GRPO 在排序器训练中的输入异构性问题，无需额外 rollout
4. 复合奖励结合即时排序信号和长期轨迹反馈，为排序器提供了多粒度的学习信号
5. 在 7 个 QA 基准上，CoSearch 7B 实现 +6.6% 相对 F1 提升，3B 实现 +10.8% 提升
6. 较小模型从联合训练中获益更大，暗示检索优化可以弥补推理能力的不足
7. 该工作为未来智能搜索系统的设计指出了关键方向：推理与检索应协同优化而非独立优化
