## 摘要

一篇全面的综述论文，定义并系统审视了深度研究（DR）智能体——结合动态推理、自适应规划、迭代工具使用和多跳信息检索以生成结构化分析报告的自主 AI 系统。论文提供了架构分类体系（单智能体 vs. 多智能体、静态 vs. 动态工作流），梳理了行业实现（OpenAI DR、Gemini DR、Grok、Perplexity 等），回顾了训练范式（SFT、基于 GRPO 的 RL、基于案例的推理），并识别了检索范围、基准对齐和智能体互操作性方面的开放挑战。

## 动机与问题

一类新的自主 AI 系统已经出现——深度研究智能体——它们超越了传统的 RAG 或工具使用范式。这些系统：

1. 处理复杂的多轮信息研究任务
2. 利用动态（而非静态）推理和规划
3. 跨多种模态使用迭代工具调用
4. 生成带有引用的全面结构化报告

然而，该领域缺乏：
- 对什么构成 DR 智能体的统一定义
- 对架构方法的系统比较
- 对训练范式权衡的理解
- 与实际目标对齐的全面基准测试

本综述通过正式定义、分类体系和路线图填补了这些空白。

## 方法

### 深度研究智能体的正式定义

> "由大语言模型驱动的 AI 智能体，集成动态推理、自适应规划和迭代工具使用来获取、聚合和分析外部信息，最终为完成开放式信息研究任务生成全面的输出。"

### 与传统系统的关键区别

| 方面 | RAG | 工具使用智能体 | **DR 智能体** |
|--------|-----|----------------|---------------|
| 自主性 | 有限 | 预定义 | 高（自适应） |
| 推理 | 事实准确性 | 模式匹配 | 深入、多轮 |
| 规划 | 静态 | 固定工作流 | 动态、自适应 |
| 实时交互 | 有限 | 被动响应 | 主动、实时 |
| 工具适应 | 无 | 手动 | 自主 |

### 分类体系：架构类别

#### 1. 工作流分类

```
工作流
    |
    +-- 静态工作流
    |     - 手动预定义的任务流程
    |     - 顺序执行
    |     - 示例：AI Scientist, Agent Laboratory, AgentRxiv
    |     - 局限：泛化能力有限
    |
    +-- 动态工作流
          - 带反馈的自适应任务规划
          - 实时重配置
          - 对复杂任务有更优的通用性
```

#### 2. 规划策略（用户意图处理）

| 策略 | 描述 | 示例 |
|----------|-------------|---------|
| 仅规划 | 直接从提示规划；无澄清 | Grok, H2O, Manus |
| 意图到规划 | 先澄清意图，再规划 | OpenAI DR |
| 统一意图-规划 | 生成计划 + 请求确认 | Gemini DR |

#### 3. 智能体架构

```
智能体系统
    |
    +-- 单智能体系统
    |     - 统一的认知循环（规划 + 工具 + 执行）
    |     - 优势：可直接进行端到端 RL 优化
    |     - 劣势：对基础模型要求高
    |     - 示例：Search-o1, R1-Searcher, DeepResearcher,
    |                 WebDancer, Kimi-Researcher
    |
    +-- 多智能体系统
          - 带专业化的分层协调
          - 优势：任务专业化，模块化扩展
          - 劣势：协调复杂；难以进行 RL 训练
          - 示例：Manus, OpenManus, OWL, Alita, AWorld
```

### 核心技术组件

#### 3.1 信息获取

```
信息获取
    |
    +-- 基于 API 的检索
    |     - 快速、结构化、可扩展
    |     - Google Search API, arXiv, Semantic Scholar, PubMed
    |     - 局限：无法处理 JS 渲染或交互式内容
    |
    +-- 基于浏览器的检索
    |     - 模拟人类网络交互
    |     - 无头浏览器（Chromium），沙箱环境
    |     - 处理动态内容、认证
    |     - 局限：更高延迟、资源消耗
    |
    +-- 混合方法（推荐）
          - 结合 API 的效率 + 浏览器的全面性
```

#### 3.2 工具使用能力

| 工具类型 | 用途 | 示例 |
|-----------|---------|---------|
| 代码解释器 | 脚本执行、算法验证 | Python, Aider |
| 数据分析 | 统计、可视化、聚合 | SQL, 图表 |
| 多模态处理 | 文本 + 图像 + 音频 + 视频 | OCR, 视觉模型 |
| 模型上下文协议 | 标准化工具接口 | MCP 服务器 |
| 计算机使用 | 实时网络 + 代码 + API | AutoGLM Rumination |

#### 3.3 长上下文优化的记忆机制

| 策略 | 方法 | 权衡 | 示例 |
|----------|--------|-----------|---------|
| 扩展上下文窗口 | 增加到 1M+ token | 高计算成本 | Gemini DR |
| 压缩中间步骤 | 摘要推理 | 信息丢失风险 | AI Scientist, Search-o1 |
| 外部结构化存储 | 向量/知识数据库 | 更高维护成本 | Manus, OWL, AutoAgent |

#### 3.4 训练范式

```
训练方法
    |
    +-- 参数化（权重更新）
    |     |
    |     +-- 监督微调 (SFT)
    |     |     - 优化：查询制定、报告生成、工具使用
    |     |     - 方法：Open-RAG, AUTO-RAG, DeepRAG, CoRAG
    |     |     - 局限：离线、静态流程
    |     |
    |     +-- 强化学习 (RL)
    |           - 从实时奖励直接优化
    |           - 算法：PPO, GRPO（首选）, Reinforce++, DPO
    |           - GRPO > PPO：更宽的奖励分布，更快收敛
    |           - 奖励：基于规则（检索相关性、工具成功）
    |           - 示例：ReSearch, R1-Searcher, Search-R1, Agent-R1
    |
    +-- 非参数化（无权重更新）
          |
          +-- 基于案例的推理 (CBR)
                - 检索 + 调整过去的问题解决轨迹
                - 动态运行时适应
                - 示例：DS-Agent, Agent K, AgentRxiv, Alita
                - 优势：可扩展，数据/计算需求最小
```

## 关键创新

1. **正式的 DR 智能体定义**：首个区分 DR 智能体与 RAG 和工具使用系统的严格定义。
2. **全面的分类体系**：静态/动态工作流 x 单/多智能体 x 3 种规划策略，创建了完整的设计空间。
3. **训练范式比较**：RL 训练中 GRPO > PPO；CBR 实现无需重训练的持续学习。
4. **记忆策略分析**：三种长上下文管理方法及清晰的权衡。
5. **行业全景**：首次对 7 个商业 DR 系统的系统比较。

## 结果（关键研究发现）

1. **检索**：混合 API + 浏览器优于单一模态；动态 > 静态 RAG。
2. **工具集成**：基于 MCP 的多工具生态系统对可扩展性至关重要。
3. **架构**：单智能体更易进行 RL；多智能体提供更好的专业化。
4. **训练**：GRPO 是 RL 的首选；CBR 实现无需重训练的持续学习。
5. **记忆**：扩展上下文需要压缩；外部存储扩展性最好。

## 局限性

1. **综述范围**：主要聚焦基于文本的 DR；对多模态研究智能体（科学成像、视频分析）覆盖有限。
2. **快速发展的领域**：分类体系可能无法捕捉 2025 年 6 月提交日期之后出现的架构。
3. **无实证验证**：综述对现有工作进行分类，但未在各系统之间运行比较实验。
4. **行业偏向**：对商业实现的大量关注可能低估了学术研究原型。
5. **有批评无替代的基准**：识别了基准不对齐问题，但未提出新的评估框架。

## 开放挑战（来自论文）

### 1. 信息获取
- 扩展到传统网络/API 来源之外
- 处理长尾、实时、快速变化的信息
- 可靠地访问用户认证资源

### 2. 执行效率
- 当前系统存在顺序依赖
- 需要：异步并行执行框架
- 多步工作流的延迟优化

### 3. 基准对齐
- 现有基准限制外部知识访问
- 评估指标与实际目标之间的不对齐
- 需要：全面的多模态基准

### 4. 多智能体优化
- 复杂的协调机制
- 跨分布式智能体的端到端 RL 训练
- 分层规划器的可扩展性和鲁棒性

### 5. 可解释性与对齐
- 平衡计算开销与推理深度
- 确保智能体行为与用户意图对齐
- 自适应工作流的透明性

## 核心要点

1. **DR 智能体是一个独特类别**：它们不仅仅是"更好的 RAG"——它们以根本不同于检索增强生成的方式结合了动态推理、自适应规划和迭代工具使用。
2. **没有单一最佳架构**：单智能体系统更适合 RL 训练；多智能体系统更适合专业化。最优选择取决于使用场景。
3. **GRPO 正成为 RL 训练的标准**：更宽的奖励分布和更快的收敛使其在 DR 智能体优化中优于 PPO。
4. **基于案例的推理实现持续学习**：CBR 等非参数化方法允许智能体在不重训练的情况下随时间改进，使其特别适合建立在先前工作基础上的研究任务。
5. **混合检索至关重要**：仅 API 或仅浏览器的方法都不够；两者结合才能提供所需的覆盖范围和效率。
6. **基准不对齐是一个关键问题**：限制外部知识访问的现有基准未能评估 DR 智能体的核心能力——开放式信息综合。
7. **该领域正在快速趋同**：来自 OpenAI、Google、字节跳动、阿里巴巴和月之暗面的行业实现共享相似模式，表明在关键设计决策上正在形成共识。
8. **记忆管理尚未解决**：目前没有方法能令人满意地处理上下文长度、信息保存和计算之间的矛盾。
