# DeepVerifier：通过测试时准则引导验证实现深度研究智能体的推理时扩展

## 简要总结

本文提出了一种通过推理时（test-time）验证扩展来增强深度研究智能体（DRA）性能的新范式。核心贡献包括：（1）构建了 DRA 失败分类法（5 大类、13 个子类），系统性地刻画深度研究智能体的失败模式；（2）提出 DeepVerifier——基于准则的结果奖励验证器，作为即插即用模块在推理时生成结构化反馈，指导智能体迭代自我改进；（3）DeepVerifier 在元评估中比 vanilla 评判器高出 12%-48% 的 F1 分数；（4）在 GAIA 和 XBench-DeepSearch 等挑战性子集上实现 8%-11% 的准确率提升；（5）发布 DeepVerifier-4K 数据集（4,646 条高质量验证样本）。该工作被 ACL 2026 Findings 收录。

## 动机与问题

### 现有方法的局限

1. **训练时优化的瓶颈**：主流方法侧重于通过后训练（SFT、RL）改进策略模型，但受限于训练数据质量、计算成本和泛化能力
2. **推理时验证的缺失**：大多数深度研究智能体在生成答案后缺乏系统性的自我验证和纠错机制
3. **评判器能力不足**：通用的 LLM-as-Judge 和 Agent-as-Judge 方法难以捕捉深度研究任务中的细粒度错误模式
4. **验证不对称性未被利用**：验证一个答案通常比生成一个正确答案更容易——这一不对称性尚未被系统性利用

### 核心洞察

> 与其仅依赖训练时优化策略模型，不如利用验证的不对称性，在推理时通过准则引导的验证迭代改进输出。

## 方法

### 一、DRA 失败分类法（DRA Failure Taxonomy）

通过分析 WebAggregator 数据集上 90 个任务的 2,997 个智能体动作中的 555 个错误点，构建了系统性的失败分类：

#### 五大失败类别及十三个子类

```
1. 信息源查找失败（Finding Sources）
   1.1 使用错误证据源
   1.2 依赖通用搜索而非精确检索
   1.3 忽略关键信息源

2. 推理失败（Reasoning）
   2.1 过早得出结论
   2.2 误解检索信息
   2.3 生成幻觉声明
   2.4 逻辑推理错误

3. 问题理解与分解失败（Problem Understanding & Decomposition）
   3.1 误解用户指令
   3.2 目标漂移（偏离原始研究方向）
   3.3 分解粒度不当

4. 动作执行失败（Action Errors）
   4.1 UI 交互失败
   4.2 格式错误
   4.3 使用错误的交互模态

5. 步数耗尽（Max Step Reached）
   5.1 早期错误级联导致后续步骤无效
```

### 二、DeepVerifier 技术架构

DeepVerifier 采用三模块流水线设计：

#### 模块一：分解模块（Decomposition Module）

```
输入：智能体轨迹（平均 8.2M tokens）、最终答案、用户查询
处理流程：
  1. 将冗长的智能体轨迹压缩为步骤索引摘要
  2. 利用失败分类法标签识别可疑行为
  3. 生成针对薄弱环节的高杠杆跟进问题
输出：结构化跟进问题列表 + 潜在失败模式标注
```

#### 模块二：验证智能体（Verification Agent）

- 使用 CK-Pro Agent 按顺序回答分解模块生成的跟进问题
- 每个跟进问题触发独立的搜索-验证流程
- 收集外部证据以判断原始答案的各个方面是否正确

#### 模块三：评判智能体（Judge Module）

- 对验证智能体的每个回答进行 1-4 等级评分：
  - 1 = 完全错误
  - 2 = 部分正确但存在重大问题
  - 3 = 大体正确但有细节偏差
  - 4 = 完全正确
- 基于评分生成结构化的纠正性反馈

### 三、推理时自我改进循环

```
Self-Evolving Loop:
  for round = 1 to max_rounds:
    1. 策略智能体生成（或重新生成）答案
    2. DeepVerifier 对答案进行准则引导验证
    3. 评判模块生成详细的纠正反馈
    4. 反馈回传给策略智能体进行迭代改进
    5. 若验证通过或达到最大轮次，输出最终答案
```

关键特性：
- **即插即用**：无需重新训练策略模型，直接在推理时使用
- **准则引导**：基于失败分类法生成的结构化准则确保验证的系统性和全面性
- **验证不对称性利用**：将较容易的验证任务作为杠杆来改进较难的生成任务

### 四、DeepVerifier-4K 数据集构建

```
构建流程：
1. 收集 400 条 WebAggregatorQA 基础轨迹
2. 使用 Claude-3.7-Sonnet 驱动 DeepVerifier 对答案进行验证
3. 筛选真阳性和真阴性的验证结果
4. 将平衡的验证轨迹转换为 SFT 训练对
5. 强调反思和自我批评能力
最终规模：4,646 条 prompt-response 对
```

该数据集使开源模型也能习得验证能力，实现 DeepVerifier 的民主化。

## 关键创新

1. **DRA 失败分类法**：首个系统性的深度研究智能体失败模式分类体系，基于大规模经验分析（555 个错误点）构建，为验证提供了理论框架
2. **推理时验证扩展范式**：首次将 test-time scaling 的理念应用于深度研究验证，证明在推理阶段投入更多计算于验证可以显著提升最终答案质量
3. **验证不对称性的系统利用**：明确提出并利用"验证比生成更容易"这一非对称性，将验证作为推理时性能提升的杠杆
4. **即插即用设计**：DeepVerifier 无需修改底层策略模型即可使用，适用于任何现有的深度研究智能体
5. **开源验证数据集**：DeepVerifier-4K 使开源社区也能训练高质量的验证模型

## 实验结果

### 验证有效性（RQ1）

| 方法 | Precision | Recall | F1 |
|------|-----------|--------|-----|
| **DeepVerifier（完整）** | **75.00%** | **71.43%** | **73.17%** |
| 无验证模块 | 100.00% | 14.29% | 25.00% |
| 无分解模块 | 86.96% | 47.62% | 61.54% |
| LLM Judge 基线 | - | - | 低 12%-48% |
| Agent-as-Judge 基线 | - | - | 低 12%-48% |

### 推理时扩展效果（RQ2）

#### GAIA 基准
| 模型 | 基线准确率 | DeepVerifier 后 | 提升 |
|------|-----------|----------------|------|
| Claude-3.7 | 52.22% | 60.12% | +7.90% |
| Claude-3.7 (Web子集) | 51.11% | 63.33% | +12.22% |
| GPT-4.1 | 29.51% | 32.53% | +3.01% |

#### 其他基准
| 基准 | 基线 | DeepVerifier 后 | 提升 |
|------|------|----------------|------|
| XBench-DeepSearch | 41.0% | 47.0% | +6.0% |
| BrowseComp | 5.0% | 10.0% | +5.0% |

### 开源模型增强（RQ3）

- DeepVerifier-8B：26.73% → 32.21%（+5.48%）
- 证明通过 DeepVerifier-4K 微调的小规模开源模型也能有效执行验证

### 迭代轮次分析

- 性能在第 4 轮左右达到峰值后趋于稳定
- 第 1 轮的"错误→正确"转换率最高（18.99%）
- "正确→错误"的退化在各轮次持续存在但强度较弱
- 两种转换的交互作用形成了观察到的性能曲线

## 局限性

1. **收益递减**：验证迭代在第 4-6 轮后出现性能平台期，更多的验证轮次不再带来显著提升
2. **退化风险**：持续存在的"正确→错误"转换限制了扩展的上限，表明验证过程本身也可能引入新错误
3. **计算开销**：每轮验证需要额外的搜索和推理计算，多轮迭代显著增加总体推理时间和成本
4. **模型依赖性**：验证效果与底层模型能力相关，在较弱模型（如 GPT-4.1 vs Claude-3.7）上提升幅度较小
5. **跨任务泛化**：虽然在多个基准上验证了有效性，但对更多样化任务类型和领域的泛化仍需进一步研究

## 核心要点

- DeepVerifier 开创性地将推理时扩展（test-time scaling）的理念从推理任务扩展到深度研究验证领域，证明"在推理时投入更多计算于验证"是提升深度研究智能体性能的有效策略
- DRA 失败分类法（5 大类 13 子类）为理解和改进深度研究智能体提供了系统性框架，其基于 555 个真实错误点的经验分析具有重要参考价值
- 验证不对称性（验证比生成更容易）是一个可被系统性利用的属性，DeepVerifier 通过将困难的生成问题转化为较容易的验证问题来提升整体性能
- 在 GAIA Web 子集上取得的 +12.22% 提升表明，验证扩展对需要复杂信息检索和推理的任务最为有效
- DeepVerifier-4K 数据集的发布为开源社区提供了训练深度研究验证器的基础设施，有望推动该方向的广泛研究
- 性能在第 4 轮达到峰值的发现为实际部署提供了重要指导：在验证计算预算有限时，4 轮迭代是一个合理的默认设置
