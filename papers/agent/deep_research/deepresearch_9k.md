# DeepResearch-9K: A Challenging Benchmark Dataset of Deep-Research Agent

## 简要总结

DeepResearch-9K 是一个面向深度研究智能体的大规模挑战性基准数据集，包含 **9000 个问题**，跨越**三个难度等级**（L1-L3）。数据集通过**实体混淆**（entity obfuscation）和**接力赛链**（relay race chain）机制工程化地控制任务难度，使得即使是 DeepSeek-V3 等先进模型也难以突破 20% 的准确率阈值。该工作同时提出了 **DeepResearch-R1** 开源训练框架，采用两阶段训练方法（SFT 热启动 + RL 优化），支持 PPO 和 GRPO 等多种强化学习算法。令人惊讶的是，经过 PPO 训练的 Llama-3.2-3B 模型达到了 22.50% 的准确率，超过了体量大得多的 DeepSeek-V3（20.18%）。

## 动机与问题

### 评估困境

深度研究智能体的快速发展迫切需要高质量的评估基准，但现有基准存在以下问题：

1. **难度不足**：现有多跳推理数据集对先进模型而言过于简单
2. **数据泄漏风险**：基于公开知识库构建的问题容易被大模型的预训练数据覆盖
3. **缺乏搜索轨迹**：现有数据集不包含高质量的搜索推理轨迹，无法用于训练
4. **难度不可控**：缺乏系统性的难度控制机制

### 研究目标

构建一个具有以下特性的基准数据集：
- 可控的难度等级
- 防数据泄漏的问题设计
- 包含高质量搜索轨迹
- 可用于训练和评估

## 方法

### 1. 数据集构建管线

DeepResearch-9K 基于开源多跳问答数据集，通过低成本自主管线构建，包含：
- **9000 个问题**：跨越三个难度等级
- **高质量搜索轨迹**：来自 Tongyi-DeepResearch-30B-A3B 的推理链
- **可验证答案**：每个问题都有明确的可验证答案

### 2. 实体混淆机制（Entity Obfuscation）

任务难度通过实体混淆的程度进行系统化控制：

**L1 - 轻度混淆（Light Obfuscation）**：
- 使用同义词替换实体
- 示例："the capital of China" 代替 "Beijing"
- 要求基本的语义理解能力

**L2 - 中度混淆（Moderate Obfuscation）**：
- 使用功能角色描述替换实体
- 示例："the first Fields Medalist" 代替具体人名
- 要求更深层的知识推理能力

**L3 - 极度混淆（Extreme Obfuscation）**：
- 使用复杂嵌套关系从句和历史时代标记
- 移除所有专有名词
- 要求多轮深度搜索和推理

### 3. 接力赛链设计（Relay Race Chain）

为 L3 难度问题设计的复杂度增强机制：

```
A → B → C → D → E → Target
```

**核心规则**：
- 构建多实体的推理链
- **任何单一知识源**（如一个 Wikipedia 页面）**不允许包含超过两个连续实体**
- 强制智能体必须执行新的搜索操作
- 确保问题无法通过单次搜索解决

### 4. 四阶段难度工程

数据集通过四个阶段系统性地构建难度梯度：
1. **基础问题生成**：从多跳 QA 数据集中提取种子问题
2. **实体混淆应用**：按 L1/L2/L3 等级进行实体替换
3. **接力赛链构建**：为高难度问题构建多实体推理链
4. **质量验证**：使用 Tongyi-DeepResearch-30B-A3B 生成搜索轨迹并验证

### 5. DeepResearch-R1 训练框架

**两阶段训练方法**：

**阶段一：热启动（Warm Start / SFT）**
- 从 30B 教师模型的 "Thought-Action" 轨迹中学习
- 学习基本的搜索-推理交互模式
- 建立初始的搜索策略

**阶段二：强化学习优化（RL Optimization）**
- 支持多种 RL 算法：
  - **PPO**（Proximal Policy Optimization）
  - **GRPO**（Group Relative Policy Optimization）
- 多轮 Web 交互环境中的在线训练
- 基于最终答案正确性的奖励信号

**训练数据划分**：
- **训练集**：DeepResearch-9K 中教师模型正确回答的样本
- **测试集（DeepResearch-Hard）**：教师模型未能正确回答的困难样本

### 6. 多轮 Web RL（Multi-turn Web RL）

- 智能体在真实 Web 搜索环境中进行多轮交互
- 每轮包含：思考（Thought）→ 行动（Action）→ 观察（Observation）
- 通过 RL 优化搜索策略和推理能力
- 支持长期多步搜索任务

## 关键创新

1. **实体混淆机制**：创新性地使用实体混淆控制问题难度，有效防止数据泄漏并确保评估的公平性
2. **接力赛链设计**：通过限制单一知识源的实体覆盖范围，强制智能体执行多轮搜索
3. **难度可控性**：三级难度体系提供了从简单到极端的连续评估谱
4. **搜索轨迹数据**：提供来自先进模型的高质量搜索推理轨迹，可直接用于 SFT 训练
5. **DeepResearch-R1 框架**：首个支持多轮 Web 交互 RL 训练的开源框架
6. **小模型超越大模型**：证明了在适当训练下，3B 参数的小模型可以超越百倍参数的大模型

## 实验结果

### 难度等级与搜索复杂度

| 难度等级 | 平均搜索工具调用次数 |
|---------|-------------------|
| L1 | 4.30 |
| L2 | ~10（中等） |
| L3 | 20.23 |

从 L1 到 L3，搜索调用频率增长了近 **5 倍**，验证了难度设计的有效性。

### 主要模型对比

| 模型 | 准确率 |
|------|--------|
| DeepSeek-V3 | 20.18% |
| **Llama-3.2-3B + PPO** | **22.50%** |
| Tongyi-DeepResearch-30B-A3B（教师模型） | 更高（作为教师基准） |

关键发现：经过 PPO 训练的 **Llama-3.2-3B 超过了 DeepSeek-V3**，尽管后者的参数量大约是前者的 200 倍。

### 训练方法对比

- SFT 热启动提供了重要的初始化效果
- PPO 在 RL 阶段表现优于 GRPO
- 两阶段训练（SFT + RL）优于单独使用任一阶段

### 教师模型搜索轨迹统计

Tongyi-DeepResearch-30B-A3B 的基准性能：
- BrowseComp: 43.4
- BrowseComp-ZH: 46.7
- HLE: 32.9
- xbench-DeepSearch: 75

## 局限性

1. **搜索环境依赖**：RL 训练依赖在线 Web 搜索环境，训练成本较高且结果可能受搜索引擎变化影响
2. **教师模型偏差**：搜索轨迹来自特定教师模型，SFT 阶段可能引入教师模型的偏差
3. **评估指标单一**：主要使用准确率评估，未考虑搜索效率、信息覆盖度等多维度指标
4. **难度分布**：三个难度等级的分布可能不均匀，L3 的极端难度可能使部分评估结果难以区分模型差异
5. **领域覆盖**：基于多跳 QA 数据集构建，可能在某些专业领域的覆盖不足
6. **数据泄漏风险残余**：尽管实体混淆降低了数据泄漏风险，但底层知识结构仍可能被大模型的训练数据覆盖
7. **可复现性**：Web 搜索环境的动态性可能影响实验的可复现性

## 核心要点

1. DeepResearch-9K 是首个针对深度研究智能体设计的大规模可控难度基准，包含 9000 个问题和三级难度体系
2. 实体混淆机制（L1-L3）通过同义词替换、功能角色描述、嵌套关系从句等方式系统性控制难度
3. 接力赛链设计确保高难度问题无法通过单次搜索解决，搜索调用从 L1 的 4.30 次增至 L3 的 20.23 次
4. DeepResearch-R1 开源训练框架支持 SFT + RL 两阶段训练，兼容 PPO 和 GRPO
5. 经过 PPO 训练的 Llama-3.2-3B（22.50%）超越了 DeepSeek-V3（20.18%），证明了小模型+RL 的巨大潜力
6. 该基准为深度研究智能体的评估提供了标准化工具，其难度设计确保了对当前最先进模型的有效区分
7. 多轮 Web RL 训练范式为构建高效深度研究智能体提供了可行路径
