# EvoFSM：基于有限状态机的深度研究可控自我演化

## 简要总结

EvoFSM 提出了一种基于有限状态机（FSM）的可控自我演化框架，用于增强 LLM 深度研究智能体的适应性。该方法的核心创新在于将优化空间解耦为宏观层面的 Flow（状态转移逻辑）和微观层面的 Skill（状态内行为），通过 Critic 引导的 FSM 精炼和自我演化记忆机制实现有针对性的改进。与无约束的自我演化方法相比，EvoFSM 通过限制仅允许四种原子操作来确保修改的局部性、可解释性和可逆性。在 DeepSearch 基准上达到 58.0% 的准确率，相比 Search-o1 提升 11.0%，在五个多跳问答基准上也一致性地超越现有方法。

## 动机与问题

### 现有方法的困境

1. **固定工作流的局限**：大多数深度研究智能体依赖预定义的固定工作流（如 ReAct 循环），面对真实世界的开放式查询时缺乏适应性
2. **无约束演化的风险**：允许智能体自由重写代码或 prompt 的方法虽然灵活，但经常导致不稳定性、幻觉和指令漂移
3. **适应性与控制的矛盾**：如何在保持行为可控的同时赋予智能体自我改进的能力，是一个根本性的设计挑战

### 核心矛盾

> 自由度过高导致不稳定，自由度过低则无法适应——EvoFSM 通过结构化 FSM 演化在两者之间取得平衡。

## 方法

### 一、FSM 形式化定义

EvoFSM 将深度研究过程建模为四元组 $\mathcal{M} = \langle \mathcal{S}, \mathcal{T}, \mathcal{I}, \mathcal{C} \rangle$：

- $\mathcal{S}$：可扩展的认知状态集合（节点），初始包含基础能力如问题分解、搜索、浏览等
- $\mathcal{T}$：转移函数，将当前状态和运行时上下文映射到下一状态
- $\mathcal{I}$：节点特定的系统 prompt，定义各状态的操作指南
- $\mathcal{C}$：Critic 机制，评估输出并触发演化

初始状态集合包括：
- **Problem Decomposition**：将复杂查询分解为子任务
- **Search**：执行信息检索
- **Browsing**：浏览和提取网页内容
- **Analysis**：分析和综合检索信息

### 二、Flow-Skill 解耦

这是 EvoFSM 的核心设计理念，将优化空间在两个正交维度上分离：

#### 宏观 Flow 操作（$\mathcal{O}_{flow}$）——重配拓扑结构

```
1. ADD_STATE: 插入中间状态（如添加验证节点）
   示例：在 Search 和 Analysis 之间插入 Verification 状态
   
2. DELETE_STATE: 移除冗余状态
   示例：当发现某个中间状态持续被跳过时将其删除
   
3. MODIFY_TRANSITION: 调整状态转移条件
   示例：当检索质量不足时，从 Analysis 回退到 Search
```

#### 微观 Skill 操作（$\mathcal{O}_{skill}$）——优化状态内行为

```
4. REVISE_INSTRUCTION: 更新单个状态的操作指南
   示例：优化 Search 状态的查询生成策略
   注意：不改变全局拓扑结构
```

**设计约束**：仅允许上述四种原子操作，确保每次修改都是"局部的、可解释的、可逆的"。

#### 演化形式化

$$\mathcal{M}_{t+1} = \mathcal{M}_t \oplus op, \quad op \in \mathcal{O}_{flow} \cup \mathcal{O}_{skill}$$

### 三、Critic 引导的 FSM 精炼

Critic 机制在演化过程中扮演主动触发器的角色：

```
Critic 工作流程：
1. 接收：用户查询 q + 当前 FSM 输出 y + FSM 配置 M_t
2. 评估：对比输出与查询需求
3. 诊断：识别具体失败模式
   - 缺乏量化证据
   - 逻辑不一致
   - 信息覆盖不足
   - 推理链断裂
4. 决策：
   - 若输出满意 → 停止演化，返回结果
   - 若存在问题 → 触发演化，指定改进方向
5. 执行：选择合适的原子操作并应用
```

Critic 不仅判断输出质量，还需诊断**为何**输出不佳，并指出**哪种**原子操作最可能改善结果。

### 四、自我演化记忆机制

维护经验池 $\mathcal{E} = \mathcal{E}^+ \cup \mathcal{E}^-$：

#### 正面经验 $\mathcal{E}^+$（成功策略）
- 记录成功的 FSM 配置和执行轨迹
- 作为新查询的初始化先验
- 通过相似查询匹配复用成功模式

#### 负面经验 $\mathcal{E}^-$（失败模式）
- 记录导致失败的 FSM 配置和转移路径
- 作为负面约束，警告系统避免类似的有问题转换
- 防止在相似场景中重复犯相同错误

#### 记忆检索机制

```
对于新查询 q_new：
1. 计算 q_new 与记忆池中历史查询的语义相似度
2. 检索 top-k 条最相关的历史记录
3. 使用正面经验作为 FSM 初始配置的 warm-start
4. 使用负面经验作为演化过程中的约束条件
```

### 五、完整演化流程

```
EvoFSM 执行流程：
Input: 用户查询 q
1. 记忆检索：从 E 中检索 top-k 相关经验
2. FSM 初始化：基于正面经验 warm-start FSM 配置 M_0
3. for t = 0, 1, 2, ...:
   a. 执行：按 M_t 的状态转移逻辑处理查询
   b. 输出：生成当前结果 y_t
   c. 评估：Critic 评判 y_t 的质量
   d. if Critic 认为满意:
        记录 (q, M_t, y_t) 到 E+
        return y_t
   e. else:
        诊断失败原因
        选择原子操作 op ∈ O_flow ∪ O_skill
        应用约束：排除 E- 中标记的有问题操作
        M_{t+1} = M_t ⊕ op
4. 若达到最大迭代次数，记录 (q, M_t, y_t) 到 E-
   return best y_t
```

## 关键创新

1. **FSM 结构化演化**：首次将有限状态机引入深度研究智能体的自我演化，用结构化约束替代自由形式的代码/prompt 重写，在适应性和可控性之间取得平衡
2. **Flow-Skill 正交解耦**：将宏观拓扑优化（添加/删除状态、修改转换）与微观行为优化（修改指令）分离，实现精准定位的改进
3. **四种原子操作约束**：限制演化仅允许 ADD_STATE、DELETE_STATE、MODIFY_TRANSITION、REVISE_INSTRUCTION 四种操作，保证修改的局部性、可解释性和可逆性
4. **双极性记忆系统**：同时维护成功经验（正面先验）和失败经验（负面约束），前者加速收敛，后者防止重蹈覆辙
5. **Critic 驱动的主动演化触发**：Critic 不仅评判质量，还诊断失败原因并指导操作选择，实现有针对性的改进

## 实验结果

### 主要基准测试（Table 1）

#### Claude-4 作为骨干模型

| 基准 | EvoFSM | Search-o1 | 提升 |
|------|--------|-----------|------|
| HotpotQA | 82.2% | 81.2% | +1.0% |
| 2WikiMultihopQA | 91.8% | 91.6% | +0.2% |
| MuSiQue | 57.6% | 51.0% | +6.6% |
| Bamboogle | 91.2% | 87.2% | +4.0% |
| **DeepSearch** | **58.0%** | **47.0%** | **+11.0%** |

#### GPT-4o 作为骨干模型

| 基准 | EvoFSM | Search-o1 | 提升 |
|------|--------|-----------|------|
| DeepSearch | 45.0% | 35.0% | +10.0% |

#### DeepSeek-v3 作为骨干模型

| 基准 | EvoFSM | Search-o1 | 提升 |
|------|--------|-----------|------|
| DeepSearch | 51.0% | 43.0% | +8.0% |

EvoFSM 在全部五个测试的 LLM（GPT-4o、Claude-4、Llama-3-70B、DeepSeek-v3、Qwen3-32B）上均一致性地超越基线。

### 消融研究（Table 2，DeepSeek-v3）

| 配置 | DeepSearch | HotpotQA | 2WIKI | MuSiQue | Bamboogle |
|------|-----------|----------|-------|---------|-----------|
| **完整 EvoFSM** | **51.0%** | **80.4%** | **88.8%** | **54.2%** | **89.6%** |
| 无结构化演化 | 36.0% | 77.8% | 82.4% | 50.8% | 85.6% |
| 无 FSM 拓扑 | 42.0% | 77.0% | 83.8% | 50.2% | 86.4% |
| 无所有组件（ReAct） | 34.0% | 76.6% | 84.0% | 48.4% | 82.4% |

**关键发现**：
- 在 DeepSearch 上静态 FSM 导致 15.0% 的性能下降，表明演化能力对复杂任务至关重要
- 在较简单的多跳 QA 任务（如 2WIKI）上差距较小，说明演化的价值随任务复杂度增加
- 完整系统比纯 ReAct 基线在 DeepSearch 上提升 17.0%

## 局限性

1. **模型依赖性**：EvoFSM 完全依赖现成的 LLM（通过 prompting 而非微调），演化效果受限于底层模型的能力上限
2. **Critic 可靠性**：系统高度依赖 Critic 的准确判断；若 Critic 产生幻觉或误判，将影响整个学习过程的质量
3. **记忆可扩展性**：经验池无限增长而缺乏整合机制，长期运行可能导致记忆检索效率下降和存储开销增大
4. **无微调**：当前框架仅通过 prompting 工作，未探索将演化经验蒸馏到模型参数中的可能性
5. **评估范围**：主要在多跳 QA 和交互式决策任务上验证，对更复杂的开放式研究报告生成场景尚未充分评估

## 核心要点

- EvoFSM 提出了"结构化自我演化"的新范式：通过有限状态机提供演化的骨架，用四种原子操作约束修改空间，有效解决了无约束演化中的不稳定性问题
- Flow-Skill 解耦是一个优雅的设计选择——宏观拓扑结构决定"做什么以及按什么顺序做"，微观技能优化决定"在每一步怎么做"，两者可以独立改进
- 在 DeepSearch 上 +11.0%（Claude-4）的大幅提升表明，自我演化对需要灵活适应的复杂深度研究任务尤其有价值，而在较简单的多跳 QA 任务上收益更温和
- 双极性记忆系统（成功先验 + 失败约束）的设计思想具有普适性，可推广到其他需要经验积累的智能体框架
- 该工作的主要限制在于它完全基于 prompting 而不涉及模型微调——将演化获得的知识蒸馏到模型参数中是一个有前景的扩展方向
- 消融研究清晰地展示了各组件的贡献：结构化演化 > FSM 拓扑 > 纯 ReAct 基线，验证了设计选择的有效性
