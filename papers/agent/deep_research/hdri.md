# HDRI：假设驱动的大模型深度研究 —— 自动化知识发现的结构化方法论

## 摘要

HDRI（Hypothesis-Driven Deep Research with LLMs）将假设的角色从"研究的产出"转变为**"组织研究过程的工具"**。论文提出六大核心原则 + 八阶段流水线，引入**缺口驱动迭代** (gap-driven iteration)、**事实推理框架** (fact reasoning framework)、**主体锁定机制** (subject locking) 三大创新。实验显示：事实密度提升 22.4%，主体匹配准确率 90%，多源验证置信度 0.92，案例研究质量评分 4.46/5.0。这是首个将假设作为通用领域研究脚手架的框架。

## 动机与问题

### 现有深度研究的"被动信息收集"陷阱

当前主流的深度研究系统（OpenAI DR、Gemini DR、Perplexity DR）将研究流程视为：

```
查询 → 搜索 → 阅读 → 综合 → 输出报告
```

这是**被动信息检索**的扩展，本质上仍是"问什么答什么"。问题在于：

1. **方向感缺失**：没有明确的"我要验证什么"，搜索容易跑偏。
2. **质量盲区**：模型不知道答案是否完整、关键反证是否被检索。
3. **实体混淆**：长程研究中"同名异物""异名同物"频发（如"X 公司"指代多个不同企业）。
4. **置信度无依据**：最终报告的"自信"建立在文本流畅度而非证据强度。

### 假设驱动的科学方法启示

科学研究的核心不是"找信息"，而是**"提假设 → 收集证据验证/证伪 → 修正假设"**的循环。HDRI 将这一范式系统化引入 LLM 研究流程。

## 方法：HDRI 六大原则与八阶段流水线

### 六大核心原则

1. **假设优先 (Hypothesis-First)**：研究启动时首先生成可证伪的假设集，而非直接搜索。
2. **结构化分解 (Structured Decomposition)**：将假设分解为子假设的层次树。
3. **证据导向 (Evidence-Directed)**：每次搜索都为验证/证伪特定假设服务。
4. **缺口感知 (Gap-Aware)**：主动识别"哪些假设证据不足"并补查。
5. **可追溯性 (Traceability)**：每个结论都附带完整的事实链与置信度。
6. **主体一致性 (Subject Coherence)**：贯穿研究锁定核心实体，防止漂移。

### 八阶段流水线

```
[1] 任务理解         → 解析用户意图，识别研究域
        ↓
[2] 假设生成         → 生成可证伪假设集（H1, H2, ..., Hn）
        ↓
[3] 假设分解         → 每个 H 分解为子假设树（H1.1, H1.2, ...）
        ↓
[4] 证据规划         → 为每个子假设规划证据需求清单
        ↓
[5] 检索与验证       → 执行搜索，标注证据等级
        ↓
[6] 缺口检测         → ★关键★ 识别信息/逻辑缺口
        ↓
[7] 缺口补充迭代     → 触发针对性补充搜索（回到 [5]）
        ↓
[8] 综合与置信度报告 → 输出带溯源的结论 + 置信度
```

### 缺口驱动迭代 (Gap-Driven Iteration)

闭环质量保证机制，分两类缺口：

**信息缺口 (Informational Gap)**：
```
"假设 H1.2 需要 2023-2024 年的数据，但当前证据均为 2022 年。"
→ 触发：搜索 2023-2024 年最新数据
```

**逻辑缺口 (Logical Gap)**：
```
"H1.1 与 H1.2 之间需要因果链 X→Y，但只有 X 和 Y 各自的证据。"
→ 触发：搜索 X→Y 因果关系的研究
```

实验表明，**缺口驱动迭代单独带来 14% 的完整性提升**。

### 事实推理框架 (Fact Reasoning Framework)

每个事实节点带有：

```
Fact {
  content: "公司 X 在 2024 Q3 营收 50 亿",
  sources: [url1, url2, url3],
  evidence_level: 1-5,  // 来源权威性
  confidence: 0.0-1.0,  // 综合置信度
  dependencies: [Fact_id1, Fact_id2],  // 推理依赖
}
```

置信度沿着推理链**乘性传播**：

```
P(结论) = ∏ P(支撑事实_i) × consistency_factor
```

### 主体锁定机制 (Subject Locking)

在研究开始时**显式注册核心实体**，并在每轮检索结果中验证：

```python
core_subjects = {"Tesla Inc.": "美国电动车厂商,Nasdaq:TSLA", ...}

for evidence in search_results:
  if "Tesla" in evidence and not matches(evidence, core_subjects["Tesla Inc."]):
    flag(evidence, "可能是 Tesla 的子公司或同名实体")
    request_disambiguation()
```

将**主体匹配准确率从约 60% 提升到 90%**。

## 实验结果

| 指标 | HDRI 结果 | 基线 (无假设驱动) |
|------|----------|------------------|
| 事实密度 | +22.4% | 1.0 (基准) |
| 主体匹配准确率 | 90% | ~60% |
| 多源验证置信度 | 0.92 | 0.71 |
| 缺口驱动完整性增益 | +14% | -- |
| 案例研究质量评分 | 4.46/5.0 | 3.2/5.0 |

### 案例研究

研究覆盖**多个领域**：金融分析、技术尽调、医学综述、政策评估。在每个领域，假设驱动方法都显著优于直接搜索方法。

## 为什么有效

### 1. 假设给搜索目标

无方向的搜索会被搜索引擎的"流行度排序"带跑偏（如查"COVID 疫苗效果"会被海量科普文章淹没）。有了具体假设（"H：mRNA 疫苗对 Delta 变种保护率 > 70%"），搜索能聚焦权威研究。

### 2. 缺口检测 = 主动质量控制

被动模式下模型"觉得够了就停"，主动模式下系统**主动识别**："你还缺 2024 年数据，还缺反证文献。" 这是从"提交作业"思维到"科学发现"思维的转变。

### 3. 主体锁定解决长程实体漂移

[[t2po]] 揭示了 token 级别的"犹豫信号"，HDRI 揭示了**实体级别的"漂移信号"**。两者结合可大幅减少长程研究的累积错误。

### 4. 置信度可解释

事实推理框架使得最终答案的置信度有数学依据，而非 LLM 的"我觉得"。审计员可以追溯每个数字的来源等级。

## 贡献

1. **首个通用领域**的假设驱动深度研究框架（之前的工作如 ResearchAgent、AI Scientist 主要针对 ML/科学领域）。
2. **三大机制**：缺口驱动迭代、事实推理框架、主体锁定。
3. **可量化的质量控制**：将传统"读起来不错"的研究产出转化为"可审计的事实链"。
4. **跨领域验证**：金融、技术、医学、政策多领域案例。

## 局限与未解决问题

- 假设生成质量依赖底层 LLM，弱模型会生成低质量假设导致全流程失效。
- 八阶段流水线相比直接搜索成本高 3-5 倍。
- 缺口检测的召回率（是否找全所有缺口）仍未量化。
