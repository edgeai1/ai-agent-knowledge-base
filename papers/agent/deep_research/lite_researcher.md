# LiteResearcher: A Scalable Agentic RL Training Framework for Deep Research Agent

## 简要总结

LiteResearcher 提出了一个可扩展的智能体强化学习训练框架，其核心思路是构建一个"轻量级虚拟世界"来模拟真实网络搜索环境，从而以零边际成本实现持续高效的 RL 训练。该框架解决了当前深度研究智能体训练中的两大耦合挑战：手工合成数据无法激发真正的搜索能力，以及依赖真实搜索引擎进行 RL 训练带来的不稳定性和高昂成本。基于 Qwen3-4B-Thinking-2507 基座模型训练的 LiteResearcher-4B 在 GAIA 上达到 71.3%、在 Xbench 上达到 78.0%，超越了 8 倍参数规模的开源智能体（如 Tongyi DeepResearch 30B 和 WebSailor 30B），甚至匹配或超越了 Claude-4.5-Sonnet 等商业系统。全部代码和模型已开源。

## 动机与问题

### 核心矛盾

深度研究智能体的训练面临两个深度耦合的挑战，形成了一个"不可能三角"：

1. **合成数据的局限性**：手工构造的合成搜索数据在分布上与真实网络搜索存在显著差距。使用这类数据训练的智能体可能学会了"合成搜索"的模式，但无法泛化到真实的网络搜索场景。训练数据的分布偏差（distribution gap）严重限制了智能体的实际搜索能力。

2. **真实搜索的不可扩展性**：在 RL 训练中直接调用真实搜索引擎存在多重问题：
   - **成本高昂**：每次 rollout 都需要大量 API 调用，训练成本随规模线性增长
   - **训练不稳定**：搜索引擎的结果具有时变性和随机性，导致奖励信号不稳定
   - **速率限制**：搜索 API 的调用频率限制严重制约了训练的并行度和速度
   - **可重复性差**：真实搜索结果的非确定性使得实验难以复现

### 根本问题

如何在保持与真实搜索环境的分布一致性的同时，实现零边际成本、完全可控且可扩展的 RL 训练？

## 方法

### 三阶段框架

LiteResearcher 的训练流程包含三个核心阶段，形成一个完整的"数据-环境-训练"闭环：

#### 第一阶段：合成真实搜索数据（Co-construct Training Data and Corpus Pipeline）

构建一个联合的训练数据与语料库生成流水线，同时保证数据分布的多样性和数据数量的可扩展性：

- 从真实网络搜索中采集原始数据，保留搜索查询、返回结果和网页内容的完整结构
- 利用教师模型（如 Tongyi DeepResearch）生成高质量的搜索交互轨迹
- 对采集的数据进行多维度增强，包括查询改写、结果重排和内容摘要

#### 第二阶段：模拟搜索环境（Stable Local Tool Environment）

利用第一阶段积累的网页语料库，构建两个关键组件来模拟真实搜索动态：

- **本地搜索引擎**（Local Search Engine）：基于采集的网页语料库构建索引，实现与真实搜索引擎一致的检索行为，但完全在本地运行，无 API 调用成本
- **本地浏览工具**（Local Browse Tool）：模拟网页浏览行为，能够解析和提取网页内容，支持智能体像在真实环境中一样浏览和理解网页

这个轻量级虚拟世界的核心优势在于：
- 零边际成本：一旦构建完成，所有搜索和浏览操作完全本地化
- 完全确定性：相同查询始终返回相同结果，保证训练的稳定性和可重复性
- 无限并行：不受外部 API 速率限制，可以任意扩展并行 rollout 数量

#### 第三阶段：课程强化学习（Curriculum RL）

采用难度感知的课程 RL 策略，逐步提升任务难度和上下文长度：

- **难度自适应筛选**：在每个训练阶段，只保留当前模型能部分解决的合成任务，丢弃过于简单（模型已完全掌握）和过于困难（模型完全无法解决）的任务
- **渐进式复杂度提升**：随着训练推进，逐步增加任务的推理深度、搜索跳数和上下文长度
- **动态课程调整**：根据模型在当前阶段的表现动态调整下一阶段的任务分布

### 基座模型

实验基于 Qwen3-4B-Thinking-2507 初始化，这是一个具有思维链能力的 4B 参数模型。

## 关键创新

1. **轻量级虚拟世界范式**：首次提出通过构建"Lite Virtual World"来桥接合成数据与真实搜索之间的鸿沟。这个虚拟世界既保持了与真实网络搜索的分布一致性，又实现了完全本地化的零成本运行。

2. **数据-环境联合构建**：将训练数据合成和环境模拟作为一个联合优化过程，第一阶段产出的语料库直接成为第二阶段虚拟环境的基础，确保了数据与环境之间的一致性。

3. **难度感知课程 RL**：通过动态筛选机制实现了"恰到好处"的训练难度控制，避免了传统 RL 中因任务过难或过易导致的训练效率低下问题。

4. **极致的参数效率**：4B 参数模型超越 8 倍规模的开源模型，甚至匹配商业级系统，证明了在正确的训练框架下，模型规模不是决定性因素。

## 实验设置

### 评测基准

1. **GAIA**：一个综合性的智能体评测基准，考察智能体在多步骤推理、工具使用和信息整合方面的综合能力。
2. **Xbench**：专门针对深度研究智能体设计的跨领域评测基准，覆盖多种知识密集型任务。

### 对比系统

| 系统 | 类型 | 参数规模 |
|-----|------|---------|
| Tongyi DeepResearch 30B | 开源 | 30B |
| WebSailor 30B | 开源 | 30B |
| Claude-4.5-Sonnet | 商业 | 未公开 |
| GLM-4.6 | 商业 | 未公开 |

## 实验结果

### 主要性能对比

| 系统 | GAIA | Xbench |
|-----|------|--------|
| LiteResearcher-4B | **71.3%** | **78.0%** |
| Tongyi DeepResearch 30B | 70.9% | 75.0% |
| WebSailor 30B | 53.2% | 53.3% |
| Claude-4.5-Sonnet | 71.2% | 66.0% |
| GLM-4.6 | 71.9% | 70.0% |

### 核心发现

1. **规模效率突破**：LiteResearcher-4B 以仅 4B 参数达到了开源最优（SOTA），超越了 8 倍参数规模的 Tongyi DeepResearch 30B（GAIA 71.3% vs 70.9%，Xbench 78.0% vs 75.0%）。

2. **超越商业系统**：在 Xbench 上大幅超越 Claude-4.5-Sonnet（78.0% vs 66.0%，+12%），在 GAIA 上与其持平（71.3% vs 71.2%）。

3. **课程 RL 的递增收益**：RL 阶段带来了持续且稳定的性能增益，在全部八个子基准上均有一致的提升，证明了课程 RL 策略的有效性。

4. **虚拟世界的泛化性**：在虚拟世界中训练的智能体成功泛化到真实搜索环境，表明轻量级虚拟世界能够充分捕捉真实搜索的关键动态特征。

## 局限性

1. **虚拟世界的覆盖度**：虚拟世界的语料库规模虽然庞大，但仍无法完全覆盖真实互联网的多样性和长尾分布，可能在极端罕见的查询上表现不佳。

2. **时效性限制**：虚拟世界中的网页内容是静态快照，无法反映互联网内容的实时更新，对于时效性强的问题可能存在不足。

3. **语言和区域偏差**：训练数据和语料库可能存在语言和地区分布偏差，在非主流语言的搜索任务上表现可能下降。

4. **单一基座模型验证**：目前仅在 Qwen3-4B-Thinking 上进行了验证，框架在不同架构和规模的基座模型上的普适性有待进一步探索。

5. **复杂工具链支持**：当前框架主要支持搜索和浏览两种工具，对于需要代码执行、数据库查询等更复杂工具链的研究任务支持有限。

## 核心要点

1. LiteResearcher 提出的"轻量级虚拟世界"范式为深度研究智能体的 RL 训练开辟了一条全新路径——既保持了真实性，又实现了完全的可控性和可扩展性。这可能成为该领域的主流训练方法。

2. 零边际成本训练意味着研究者可以进行更大规模、更长时间的 RL 训练，而不受外部 API 成本和速率限制的制约，这对于推动该领域的民主化发展具有重要意义。

3. 4B 模型超越 30B 模型的结果再次证明：对于智能体能力的提升，训练方法论（how to train）可能比模型规模（how big）更为重要。

4. 课程 RL 的成功应用表明，智能体训练中的任务难度管理是一个被低估但至关重要的因素。

5. 全面开源（代码、模型、数据）为社区提供了一个可复现的基线，有望加速深度研究智能体领域的发展。
