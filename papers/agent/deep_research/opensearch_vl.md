# OpenSearch-VL: Training Frontier Multimodal Deep Search Agents with Agentic Reinforcement Learning

## 简要总结

OpenSearch-VL 提出了一套完全开放的多模态深度搜索智能体训练方案，涵盖高质量数据策管、多样化视觉/搜索工具环境和致命感知的智能体强化学习（Fatal-Aware Agentic RL）三大核心组件。与传统视觉语言模型（VLM）的单次前向推理不同，OpenSearch-VL 智能体以闭环方式运行：先检查图像、裁剪或增强感兴趣区域，再发起网络和图像搜索、访问检索到的页面，最终基于收集到的证据撰写答案。在七个知识密集型多模态基准（SimpleVQA、VDR、MMSearch、LiveVQA、BrowseComp-VL、FVQA、InfoSeek）上，OpenSearch-VL 平均得分比对应的智能体基线高出超过 10 个点，在 30B/32B 规模上匹配强商业系统的准确率。全部数据、代码和模型检查点均已开源。

## 动机与问题

### 多模态深度搜索的核心挑战

现有的视觉语言模型在回答知识密集型视觉问题时面临根本性限制：

1. **单次推理的局限**：传统 VLM 尝试在单次前向传播中完成所有推理，无法动态获取外部知识来补充模型内部知识的不足。对于需要领域专业知识或实时信息的问题，这种模式注定失败。

2. **视觉理解的瓶颈**：图像中的关键信息可能存在于小区域、低分辨率或需要特定视角校正的部分，单次端到端处理难以充分利用这些信息。

3. **多模态搜索的复杂性**：与纯文本搜索不同，多模态搜索需要同时处理视觉内容理解、文本搜索查询生成和跨模态证据整合。

4. **RL 训练中的级联失败**：在多轮工具调用的 RL 训练中，一次工具调用失败（如搜索超时、网页加载错误）会导致后续所有步骤的轨迹变得无意义，但标准 RL 算法会错误地惩罚整个轨迹，包括失败前的合理推理。

### 开放性目标

该项目旨在提供一个完全可复现的开放方案，包括端到端训练所需的数据集、代码和模型检查点。

## 方法

### 数据策管流水线

#### Wikipedia 超链接图数据合成

基于 Wikipedia 的超链接图结构，合成以图像为锚点的多跳视觉问答数据：

- 利用 Wikipedia 文章之间的超链接关系构建知识图谱
- 识别包含图像的文章节点，将图像与相关的多跳知识路径关联
- 基于多跳路径自动生成需要跨页面信息整合的视觉问答对

#### 两个开放数据集

1. **SearchVL-SFT-36k**：用于监督微调的 36,000 条训练样本
   - 来源于 7 个工具使用数据集：FVQA、Palace、WebQA、LiveVQA、WikiArt、Wiki-zh、Wiki-en
   - 每条轨迹平均包含 6.3 轮工具调用交互
   
2. **SearchVL-RL-8k**：用于强化学习的 8,000 条训练样本

### 统一的视觉-检索工具环境

提供 10 种工具，在 SFT 数据生成、RL rollout 和推理阶段使用一致的接口：

| 工具类别 | 工具名称 | 功能描述 |
|---------|---------|---------|
| 视觉处理 | `crop` | 裁剪图像感兴趣区域 |
| 视觉处理 | `super_resolution` | 图像超分辨率增强 |
| 视觉处理 | `sharpen` | 图像锐化处理 |
| 视觉处理 | `perspective_correct` | 视角校正 |
| 内容分析 | `layout_parsing` | 版面布局解析 |
| 搜索检索 | `text_search` | 文本搜索 |
| 搜索检索 | `image_search` | 图像搜索 |
| 搜索检索 | `web_search` | 网页搜索 |
| 网页浏览 | `visit` | 访问网页并提取内容 |
| 代码执行 | `python_interpreter` | Python 代码执行器 |

这一设计使得智能体既能修复不完美的视觉输入（通过裁剪、增强、校正），又能通过一致的接口获取外部知识。

### 致命感知的 GRPO 算法（Fatal-Aware GRPO）

这是本文最核心的算法贡献，专门解决多轮工具调用中的级联失败问题：

#### 问题定义

在长 rollout 中，工具调用可能在某一步发生"致命失败"（fatal failure），例如搜索引擎超时、网页无法加载等。失败后的所有后续 token 都是基于错误状态生成的，不应参与策略梯度计算。

#### 解决方案

1. **致命步骤掩码**（Fatal Masking）：检测 rollout 中的致命失败步骤，将致命步骤之后的所有 token 从策略梯度计算中掩码掉。

2. **单侧优势截断**（One-Sided Advantage Clamping）：保留致命失败之前合理推理步骤的正向信用分配，而不是像传统方法那样惩罚整个轨迹。具体来说，只截断负优势值，保留正优势值，从而鼓励模型在失败前展现出的良好推理行为。

### 训练流程

- 在三个 Qwen3-VL 变体（8B、30B-A3B、32B）上进行实验
- 使用 DeepSpeed ZeRO-3 进行全参数微调（LLM + 视觉塔 + 投影器）
- 通过 Ray 框架实现分布式训练
- 评测协议：Pass@1 + GPT-4o 评判，与 VDR-Bench 一致

## 关键创新

1. **致命感知的 GRPO**：首次在多模态智能体 RL 训练中提出处理级联工具失败的系统性方法。通过致命掩码和单侧优势截断的结合，既避免了对失败后无意义 token 的梯度更新，又保留了失败前合理推理的正向激励。

2. **统一的多功能工具环境**：将视觉处理（裁剪、增强、校正）和信息检索（搜索、浏览）工具统一在同一个交互环境中，使智能体能够在推理过程中灵活组合不同类型的工具。

3. **Wikipedia 超链接图驱动的数据合成**：利用 Wikipedia 的天然超链接结构自动生成多跳视觉问答数据，既保证了知识链条的真实性，又实现了大规模自动化生产。

4. **完全开放的训练方案**：从数据到代码到模型检查点全部开源，为多模态深度搜索智能体的研究提供了可复现的基础设施。

## 实验设置

### 七大评测基准

1. **SimpleVQA**：简单视觉问答
2. **VDR**：视觉深度研究基准
3. **MMSearch**：多模态搜索评测
4. **LiveVQA**：实时视觉问答
5. **BrowseComp-VL**：视觉浏览理解评测
6. **FVQA**：基于事实的视觉问答
7. **InfoSeek**：信息检索式视觉问答

### 模型规模

- OpenSearch-VL-8B
- OpenSearch-VL-30B-A3B（MoE 架构，30B 总参数，3B 激活参数）
- OpenSearch-VL-32B

## 实验结果

### 核心性能指标

OpenSearch-VL-8B 是最强的开源 8B 智能体，平均得分比 SenseNova-MARS-8B 高 +3.9。

OpenSearch-VL-30B-A3B 相比 Qwen3-VL 智能体基线提升 +13.8 平均分：
- VDR: +13.3
- MMSearch: +24.5
- FVQA: +10.2
- InfoSeek: +16.2

OpenSearch-VL-32B 在多数基准上超越 Gemini-2.5-Pro 和 Claude-4-Sonnet 的直接推理基线。

### 消融实验：Fatal-Aware GRPO 各组件贡献

| 方法 | 平均分 | 提升 |
|------|-------|------|
| SFT 基线 | 64.6 | - |
| Vanilla Search-Augmented GRPO | 67.6 | +3.0 |
| Hard Masking（Vision-DeepResearch 方法） | 67.7 | +3.1 |
| Fatal Masking（仅致命掩码） | 69.1 | +4.5 |
| Full Method（致命掩码 + 单侧优势截断） | **71.8** | **+7.2** |

完整方法比 vanilla GRPO 高出 +4.2，并在每个基准上都达到最佳分数，证明了两个组件（致命掩码和单侧优势截断）的协同效果。

## 局限性

1. **工具环境的真实性差距**：虽然提供了丰富的工具集，但本地搜索环境与真实互联网搜索之间仍可能存在分布差异，尤其是在检索结果的新鲜度和多样性方面。

2. **致命失败的检测准确性**：当前的致命失败检测机制可能无法捕捉所有类型的工具失败（如部分失败、超时后返回不完整结果等灰色地带）。

3. **计算成本**：全参数微调 32B 规模的 VLM（包括视觉塔和投影器）需要大量 GPU 资源，限制了社区复现的可及性。

4. **评测基准偏差**：七个基准均聚焦于知识密集型视觉问答，可能未覆盖多模态搜索的全部应用场景（如视觉推理、创意任务等）。

5. **单一视觉编码器架构**：仅在 Qwen3-VL 系列上验证，对其他 VLM 架构的适用性尚不明确。

## 核心要点

1. 多模态深度搜索是一个与纯文本深度搜索本质不同的问题，它需要同时处理视觉理解、视觉增强和跨模态检索，OpenSearch-VL 的统一工具环境为此提供了一个优雅的解决方案。

2. Fatal-Aware GRPO 解决了多轮智能体 RL 训练中一个长期被忽视但至关重要的问题——级联工具失败。其致命掩码 + 单侧优势截断的组合在消融实验中展现出了每个组件的独立贡献和协同增益。

3. 10+ 点的平均提升和在 32B 规模上超越 Gemini-2.5-Pro 的表现，证明了开源多模态搜索智能体已经达到了与商业系统竞争的水平。

4. Wikipedia 超链接图是多跳视觉问答数据的天然富矿，这种数据合成范式可以推广到其他知识图谱结构化的数据源。

5. 完全开放的数据-代码-模型方案为社区提供了可复现的强基线，有望加速多模态搜索智能体领域的研究进展。
