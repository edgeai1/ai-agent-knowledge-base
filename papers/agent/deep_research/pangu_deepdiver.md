# DeepDiver：通过开放网络强化学习实现自适应搜索强度扩展

## 摘要

DeepDiver（来自华为盘古团队）提出了搜索强度扩展（SIS）——一种涌现能力，使得经过 RL 训练的模型能够学会针对更难的问题自主升级其搜索频率和深度，而不是满足于过度自信、证据不足的回答。在新颖的 WebPuzzle 基准（24k 训练样本，275 测试样本）上使用 7k 实例进行训练后，DeepDiver 使 Qwen2.5-7B 和 Pangu-7B 在真实网络信息搜索任务上匹配了 671B 参数的 DeepSeek-R1，其中 DeepDiver-Pangu-7B 在 WebPuzzle 上达到了 38.1 的准确率。

## 动机与问题

先前基于 RL 的搜索智能体（R1-Searcher、Search-R1）存在三个关键局限：

1. **静态搜索模式**：模型学习固定数量的搜索轮次，不考虑问题难度；不会对更难的问题自适应地增加搜索努力
2. **仅维基百科语料库**：训练和评估使用维基百科段落，而非带有噪声、不一致性和更大规模的实时开放网络
3. **过度自信陷阱**：在搜索不充分的情况下，模型倾向于从参数化知识中"猜测"，而不是投入更多搜索努力来寻找证据

DeepDiver 的洞察：**有效的深度研究要求模型学习何时应该更密集地搜索，而不仅仅是如何搜索。** 这种自适应搜索行为应该从 RL 中自然涌现，类似于 DeepSeek-R1 在推理中展现的"顿悟时刻"。

## 方法

### WebPuzzle 基准

一个专门为评估实时网络信息搜索而设计的新基准：

```
WebPuzzle 基准：
  - 24,000 个训练样本
  - 275 个测试样本
  - 查询类型：维基百科领域 + 开放领域
  - 来源：实时互联网（非静态语料库）
  - 难度：需要多步网络探索
  - 语言：包含英文和中文查询
```

WebPuzzle 的问题被设计为需要真正的网络探索——它们无法仅从参数化知识中回答，迫使模型发展有效的搜索策略。

### 训练流程

```
+------------------------------------------------------------------+
|  DeepDiver 训练流程                                                |
|                                                                  |
|  阶段 1：冷启动 SFT                                               |
|  +------------------------------------------------------------+  |
|  |  - 在搜索增强的轨迹上进行监督微调                               |  |
|  |  - 教授基本的搜索调用格式                                      |  |
|  |  - 为 RL 探索提供初始策略                                      |  |
|  +------------------------------------------------------------+  |
|                          |                                       |
|                          v                                       |
|  阶段 2：带开放网络搜索的 RL 训练                                   |
|  +------------------------------------------------------------+  |
|  |  - 在 7k 个 WebPuzzle 实例上训练                              |  |
|  |  - 模型与实时网络搜索引擎交互                                   |  |
|  |  - 基于结果的奖励（答案正确性）                                  |  |
|  |  - 搜索强度扩展自然涌现                                        |  |
|  +------------------------------------------------------------+  |
+------------------------------------------------------------------+
```

### 搜索强度扩展 (SIS) -- 核心概念

SIS 不是一个设计的机制，而是在 RL 训练过程中产生的**涌现行为**：

```
RL 之前（冷启动 SFT 模型）：
  - 平均搜索轮次：每个问题 1.75 次
  - 不考虑难度的固定模式
  - 对困难问题快速放弃

RL 之后（DeepDiver）：
  - 平均搜索轮次：每个问题 2.51 次（Qwen2.5-7B）
  - 自适应：困难问题更多轮次
  - 当初始结果不充分时坚持搜索

关键发现：搜索强度与性能强相关
  - 随着搜索引擎调用次数增加 --> 训练奖励增加
  - 模型学会在有回报的地方"投入"更多搜索努力
```

### SIS 在两个维度上体现

```
1. 频率扩展：
   简单问题：搜索 1 次 --> 回答
   困难问题：搜索 3-5 次 --> 回答
   模型学习根据问题难度自适应调整搜索频率

2. 深度扩展：
   浅层：宽泛的初始查询
   深层：针对特定方面的精细化后续查询
   模型学习将复杂的信息需求分解为逐步更具体的子查询
```

### 与先前方法的比较

```
                    搜索模式
方法                简单问题   困难问题   自适应？
-------             ------    ------    ---------
标准 RAG             1 次调用  1 次调用   否
R1-Searcher         ~2 次调用 ~2 次调用  部分
Search-R1           ~2 次调用 ~2 次调用  部分
DeepDiver (SIS)     ~1 次调用 ~4 次调用  是（涌现）
```

### 开放网络 vs. 维基百科语料库搜索

DeepDiver 的一个显著特点是使用实时网络搜索进行训练，而非静态维基百科语料库：

```
维基百科语料库搜索（先前工作）：       开放网络搜索（DeepDiver）：
- 干净、结构化的段落               - 嘈杂、异构的内容
- 完整覆盖维基百科主题             - 覆盖所有主题的广泛范围
- 无时效性问题                     - 实时、最新的信息
- 可预测的检索质量                 - 可变的检索质量
- ~2100 万段落                     - 整个索引网络
```

这迫使模型开发更鲁棒的搜索策略，以处理真实世界的噪声和不一致性。

## 关键创新

1. **搜索强度扩展 (SIS)**：首次识别和表征这种涌现的 RL 行为。模型学会根据问题难度自适应调整搜索努力——这是先前工作中不存在的全新能力。

2. **WebPuzzle 基准**：新颖的 24k 样本基准，用于评估真实网络信息搜索，弥补了基于维基百科的评估与真实世界搜索之间的差距。

3. **开放网络 RL 训练**：首次展示基于 RL 的搜索智能体在实时网络搜索引擎（而非静态语料库）上训练，产生更鲁棒的智能体。

4. **小模型与大模型持平**：7B 模型在网络搜索任务上匹配 671B 的 DeepSeek-R1，证明了搜索增强可以弥合巨大的能力差距。

5. **SFT 到 RL 课程**：冷启动 SFT 阶段提供了比从指令微调模型直接进行纯 RL 更好的初始化，使 RL 探索更有效。

## 实验设置

**基础模型：**
- Qwen2.5-7B-Instruct
- Pangu-7B-Reasoner（华为的专有推理模型）

**训练：**
- 在搜索增强轨迹上进行 SFT 冷启动
- 在 7k 个 WebPuzzle 实例上进行 RL 训练
- 训练期间使用开放网络搜索引擎进行检索

**评估基准：**
- WebPuzzle（275 个测试样本，主要基准）
- WebPuzzle-en（英文子集）
- HotpotQA（300 个采样样本）
- 2WikiMultiHopQA
- Bamboogle（125 个样本）
- FRAMES（300 个采样样本）

**基线方法：**
- 冷启动 SFT 模型（RL 之前）
- R1 蒸馏模型（从更强模型蒸馏）
- R1-Searcher
- DeepResearcher
- DeepSeek-R1（671B，上限参考）

## 结果

**WebPuzzle 性能：**

| 模型 | WebPuzzle 准确率 |
|--------------------------|--------------------|
| 冷启动 Qwen-7B          | ~27.6              |
| R1 蒸馏 Qwen-7B         | 29.8               |
| DeepDiver-Qwen-7B       | **37.6**           |
| DeepDiver-Pangu-7B      | **38.1**           |
| DeepSeek-R1 (671B)      | ~38.x              |

**关键结果**：DeepDiver-Qwen-7B 比冷启动模型提升了 **10 个百分点**，DeepDiver-Pangu-7B 达到 38.1，与 DeepSeek-R1（671B 参数）相当。

**WebPuzzle-en 比较：**
- DeepDiver 比 DeepResearcher 高出 **11.1 个百分点**

**跨基准结果：**
- R1 蒸馏的 Pangu-7B 在 Bamboogle 上比冷启动下降了 5.7 个百分点（蒸馏在某些任务上有害）
- DeepDiver-Pangu-7B 在所有基准上均有提升（无回退）

**搜索强度统计：**
- DeepDiver-Qwen-7B：平均 2.51 轮搜索（vs. SFT 基线的 1.75）
- 训练过程中搜索频率与奖励之间存在明显的正相关
- 模型学会将更多搜索努力分配给有帮助的问题

## 局限性

1. **WebPuzzle 特定优化**：7k 训练实例来自 WebPuzzle；尚不清楚 SIS 如何迁移到根本不同的信息搜索任务（如科学文献搜索、代码搜索）
2. **依赖实时网络**：训练需要实时搜索 API，这引入了不可重复性（网络内容随时间变化）和成本（API 调用）
3. **专有基础模型**：Pangu-7B-Reasoner 不公开可用，限制了最佳结果的可复现性；仅 Qwen 的结果可复现
4. **SFT 冷启动要求**：与 Search-R1 直接从指令微调模型训练不同，DeepDiver 需要 SFT 预热阶段，增加了复杂性
5. **评估规模**：WebPuzzle 测试集较小（275 个样本）；更大规模的评估将增强对结果的信心
6. **未分析失败模式**：当 SIS 失败时（模型过度搜索但仍然得到错误答案），论文未深入分析原因
7. **延迟**：更多搜索轮次意味着更高的推理延迟；未分析准确性-延迟权衡

## 核心要点

- 搜索强度扩展是一种真正涌现的 RL 行为——模型学会对更难的问题搜索更多，而无需被明确告知这样做
- 这种自适应搜索强度与性能增益强相关，表明它是有效深度研究的关键机制（而不仅仅是副作用）
- 开放网络 RL 训练产生比维基百科语料库训练更鲁棒的智能体，代价是可复现性和 API 费用
- 7B 模型在配备自适应搜索后可以在信息搜索任务上匹配 671B 模型——这是搜索作为能力均衡器的最有力展示之一
- SFT 到 RL 课程（冷启动后 RL）比单独的 SFT 或直接从指令微调基座进行 RL 效果更好
- WebPuzzle 作为真实网络信息搜索的基准填补了重要空白，补充了先前工作使用的基于维基百科的基准
- 被 NeurIPS 2025 接收，验证了 SIS 发现的重要性
