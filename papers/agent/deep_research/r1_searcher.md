# R1-Searcher：通过强化学习激发大语言模型的搜索能力

## 摘要

R1-Searcher 提出了一个两阶段的基于结果的强化学习框架，教导大语言模型在推理过程中自主调用外部搜索/检索系统，无需过程奖励或知识蒸馏。第一阶段使用检索奖励来教导模型正确的检索调用格式；第二阶段添加答案奖励，训练模型利用检索到的信息生成准确答案。该方法在多跳问答基准上超越了强大的 RAG 基线，甚至超过了闭源的 GPT-4o-mini。

## 动机与问题

大型推理模型（LRM）如 DeepSeek-R1 已经证明了强化学习可以增强复杂推理能力。然而，这些模型完全依赖内部参数化知识，存在以下问题：

1. **不完整性**：无法覆盖所有事实知识，尤其是最新或小众信息
2. **时效性**：参数化知识在训练截止日期后会变得过时
3. **幻觉倾向**：当内部知识不足时，模型会捏造答案

现有的检索增强方法（RAG）存在以下不足：
- 固定的检索流程无法适应推理需求
- 无法对复杂问题执行迭代式多步检索
- 依赖有监督蒸馏数据或手工设计的过程奖励

核心问题：**能否使用纯强化学习（无蒸馏、无过程奖励）来教导大语言模型在其思维链推理过程中何时以及如何进行搜索？**

## 方法

### 整体架构

```
+------------------------------------------------------------------+
|  两阶段基于结果的强化学习训练流程                                    |
|                                                                  |
|  第一阶段：检索奖励训练                                             |
|  +------------------------------------------------------------+  |
|  |  目标：学习正确的检索调用格式                                  |  |
|  |  奖励 = format_correct(retrieval_call) ? 0.5 : 0.0          |  |
|  |  - 无答案准确性要求                                          |  |
|  |  - 模型探索何时/如何调用检索器                                 |  |
|  +------------------------------------------------------------+  |
|                          |                                       |
|                          v                                       |
|  第二阶段：答案奖励训练                                             |
|  +------------------------------------------------------------+  |
|  |  目标：学习利用检索信息生成正确答案                              |  |
|  |  奖励 = 检索奖励 + 答案准确性奖励                               |  |
|  |  - 结合检索格式奖励与答案奖励                                  |  |
|  |  - 模型学习有效利用搜索结果                                    |  |
|  +------------------------------------------------------------+  |
+------------------------------------------------------------------+
```

### 推理与搜索交替进行

在推理过程中，模型生成的思维链将推理token与搜索调用token交替排列：

```
<think> reasoning about the question... </think>
<search> query terms for retrieval </search>
<result> [retrieved passages injected here by external system] </result>
<think> continued reasoning incorporating retrieved info... </think>
<search> refined follow-up query </search>
<result> [more retrieved passages] </result>
<think> final reasoning leading to answer </think>
<answer> final answer </answer>
```

### 奖励设计

**第一阶段 -- 检索奖励 (R_ret)：**

```
R_ret = 0.5  如果模型产生了有效的 <search>...</search> 调用
R_ret = 0.0  否则
```

目的：快速教导模型正确的调用格式，而不将检索学习与答案准确性混淆。这种解耦加速了学习过程。

**第二阶段 -- 组合奖励 (R_total)：**

```
R_total = R_ret + R_ans
其中 R_ans = accuracy_score(predicted_answer, ground_truth)
```

答案奖励使用覆盖精确匹配 (ACC_R) 或 LLM 评判 (ACC_L) 评分。

### 强化学习算法

本文使用组相对策略优化（GRPO），沿用 DeepSeek-R1 的方法。对于每个问题，采样多个rollout，并基于结果（最终答案正确性 + 检索调用）计算奖励。

```
GRPO 更新：
  1. 对问题 q 采样 G 个 rollout {o_1, ..., o_G}
  2. 计算奖励 {r_1, ..., r_G}
  3. 归一化奖励：advantage_i = (r_i - mean(r)) / std(r)
  4. 更新策略以增加高奖励 rollout 的概率
```

### 检索系统

使用标准的段落检索流程：
- 语料库：维基百科段落（分块）
- 检索器：稠密检索（如 Contriever 或类似模型）
- 每个搜索查询返回 Top-k 个段落

## 关键创新

1. **两阶段解耦训练**：将检索格式学习（第一阶段）与面向答案的训练（第二阶段）分离，防止模型同时被两个学习目标淹没。这种课程式分解至关重要。

2. **无蒸馏的纯强化学习**：与需要从更强模型蒸馏或手工设计过程奖励的先前工作（如 Search-o1）不同，R1-Searcher 仅使用基于结果的奖励，更加简单且可扩展。

3. **自主搜索时机**：模型通过探索学习何时调用搜索——无需指定何时搜索。这种涌现行为是一个关键发现。

4. **检索奖励引导**：对有效检索调用给予 0.5 分的奖励，提供了一种学习信号，加速了搜索行为的发现，而不会导致过度检索的偏向。

## 实验设置

**基础模型：**
- LLaMA-3.1-8B-Instruct
- Qwen-2.5-7B-Base

**基准测试（4个多跳问答数据集）：**
- HotpotQA（域内）-- 500个开发样本
- 2WikiMultiHopQA（域内）-- 500个开发样本
- Musique（域外）-- 500个开发样本
- Bamboogle（域外）-- 完整集合

**基线方法：**
- 直接提示（无检索）
- 标准 RAG（先检索后阅读）
- 迭代式 RAG（IRCoT, ReAct）
- Search-o1（蒙特卡洛树搜索 + 检索）
- GPT-4o-mini（闭源）

**评估指标：**
- ACC_R：覆盖精确匹配准确率
- ACC_L：LLM 评判准确率（GPT-4o-mini 作为评判者）

## 结果

**关键性能提升（ACC_L / LLM 评判指标）：**

| 基准测试 | 相对于基线的提升 |
|-----------------|---------------------------|
| HotpotQA        | +48.2%                    |
| 2WikiMultiHopQA | +21.7%                    |
| Bamboogle       | +4.0%                     |

**跨模型比较亮点：**
- R1-Searcher (LLaMA-3.1-8B) 在多跳基准上超越了 GPT-4o-mini
- R1-Searcher (8B) 在 Bamboogle 上比 Search-o1 (32B) 高出 +11.4%
- 在域内和域外基准上均有一致的提升

**消融实验发现：**
- 移除第一阶段（检索奖励）会显著降低性能——模型难以同时学习检索格式和答案准确性
- 单阶段训练收敛更慢，最终性能更低
- 两阶段方法提供了明确的课程效果

## 局限性

1. **仅维基百科检索语料库**：仅在维基百科段落上进行评估；尚不清楚该方法在开放网络或异构知识源上的表现如何
2. **聚焦多跳问答**：基准测试仅限于多跳事实问答；未在长文本生成、摘要或开放式研究任务上进行评估
3. **固定检索系统**：检索后端未被联合优化——仅训练了大语言模型策略；联合训练检索器可能带来进一步提升
4. **计算成本**：每个问题需要多个rollout的强化学习训练成本高昂，尤其是交替检索增加了延迟
5. **未与过程奖励方法比较**：论文认为基于结果的强化学习已经足够，但未直接与过程奖励方法进行比较

## 核心要点

- 纯基于结果的强化学习（无蒸馏或过程奖励）足以教导大语言模型在推理过程中进行自主搜索行为
- 两阶段课程（先格式后准确性）对于稳定学习至关重要
- 小型开源模型（8B）在配备学习到的搜索能力后，可以在检索密集型任务上超越 GPT-4o-mini
- 检索奖励引导技巧（0.5 分奖励）是一种简单但有效的方式，可以在强化学习中启动搜索行为的探索
- 这项工作是 2025 年初出现的"搜索增强推理"范式的基础性工作，与 Search-R1 及相关同期工作并行发展
