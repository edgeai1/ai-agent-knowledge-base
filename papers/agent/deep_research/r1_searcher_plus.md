# R1-Searcher++: Incentivizing the Dynamic Knowledge Acquisition of LLMs via Reinforcement Learning

## 简要总结

R1-Searcher++ 是由中国人民大学高瓴人工智能学院提出的新型框架，旨在训练大语言模型自适应地利用内部知识和外部知识源。该框架采用两阶段训练策略：首先通过 SFT 冷启动进行初步格式学习，然后通过强化学习（RL）实现动态知识获取。RL 阶段使用基于结果的监督来鼓励探索，引入了内部知识利用的奖励机制，并集成了记忆化机制以持续同化检索到的知识。实验表明，R1-Searcher++ 在多个多跳推理基准上大幅超越了此前的 RAG 和推理方法，同时显著减少了不必要的检索次数。

## 动机与问题

### 核心问题

1. **静态知识的幻觉问题**：LLM 虽然强大，但由于知识截止日期的限制，容易在需要最新或罕见知识的问题上产生幻觉
2. **现有 RAG 方法的不足**：
   - **成本高昂**：无差别地为每个问题调用外部搜索引擎，即使模型内部已有足够知识
   - **泛化能力差**：现有方法通常在特定数据集上训练和评估，难以迁移到新领域
   - **忽略内部知识**：大多数方法未能有效利用 LLM 自身已有的知识，导致冗余检索
3. **内外知识的动态平衡**：如何让模型智能地判断何时使用内部知识、何时需要外部检索，是一个尚未解决的核心挑战

### 与 R1-Searcher 的关系

R1-Searcher++ 是 R1-Searcher（arXiv:2503.05592）的升级版。原版 R1-Searcher 首次提出通过 RL 激励 LLM 的搜索能力，但未能有效处理内部知识与外部检索的动态平衡问题。R1-Searcher++ 在此基础上引入了内部知识奖励和记忆化机制。

## 方法

### 整体框架

R1-Searcher++ 采用两阶段训练策略，结合三个核心技术组件：

#### 阶段一：SFT 冷启动

- **目的**：让模型学习基本的检索增强推理格式
- **过程**：使用标注数据对模型进行监督微调，建立"思考-搜索-整合"的基本交互模式
- **格式学习**：模型学习在推理过程中适时插入搜索查询，并将检索结果整合到推理链中

#### 阶段二：RL 动态知识获取

RL 阶段包含三个关键技术组件：

**组件 1：基于结果的监督（Outcome Supervision）**
- 使用最终答案的正确性作为奖励信号
- 鼓励模型通过探索发现最优的知识获取策略
- 不对中间步骤施加硬性约束，给予模型充分的探索自由

**组件 2：内部知识利用奖励（Group Reward）**
- 设计了专门的奖励机制来激励模型利用内部知识
- 当模型能够在不进行外部检索的情况下正确回答问题时，给予额外奖励
- 该机制使模型学会在内部知识充足时避免不必要的检索

**组件 3：记忆化机制（Memorization Mechanism）**
- **外部知识内化**：通过一个重写模型，将检索到的文档和问题转化为纯推理路径
- **内化训练**：将这些推理路径作为训练数据，使用标准负对数似然（NLL）损失训练主模型
- **持续积累**：随着训练的推进，模型不断将外部知识内化为自身参数，减少对外部检索的依赖

### 知识获取流程

模型在推理时动态决策：
1. 首先评估当前问题是否在内部知识范围内
2. 若内部知识充足，直接推理回答（获得内部知识奖励）
3. 若需要外部信息，生成搜索查询并整合检索结果
4. 检索到的知识通过记忆化机制逐步内化

## 关键创新

1. **内外知识自适应切换**：首次系统性地解决了 LLM 在内部知识和外部检索之间的动态平衡问题
2. **内部知识利用奖励**：通过专门的 Group Reward 机制，激励模型在内部知识充足时避免冗余检索
3. **记忆化机制**：将外部检索到的知识转化为内部推理格式，实现知识的持续同化和积累
4. **SFT + RL 协同训练**：SFT 建立格式基础，RL 优化决策策略，两者形成有效协同
5. **检索效率提升**：在提升答案质量的同时大幅减少检索次数，实现了"少查多对"

## 实验结果

### 模型配置

实验基于以下骨干模型进行：
- **Qwen2.5-7B**：主要实验模型
- **Llama-3.1-8B-Instruct**：用于验证方法的跨模型泛化性

### 评估基准

使用四个多跳推理数据集：
- **域内（In-domain）**：HotpotQA、2WikiMultiHopQA（训练集的部分数据用于训练）
- **域外（Out-of-domain）**：Musique、Bamboogle（用于评估泛化能力）

### 评估指标

- **Cover Exact Match (ACC_R)**：评估预测答案是否包含标准答案
- **LLM-as-Judge (ACC_L)**：使用 GPT-4o-mini 评估预测的正确性

### 主要结果

- R1-Searcher++ 在所有评测基准上均超越了此前最优的 RAG 和推理方法
- 相较于最佳 vanilla RL 基线，LasJ 指标提升高达 4.3%
- 平均检索次数相比 vanilla RL 方法减少 42.9%
- 使用 Llama-3.1-8B-Instruct 时：HotpotQA 提升 48.2%、2WikiMultiHopQA 提升 21.7%、Bamboogle 提升 4.0%

### 消融实验结果

五种消融变体的性能变化：

| 消融变体 | 影响 |
|---------|------|
| w/o Stage-1（去除 SFT 冷启动） | 性能显著下降，证明格式学习的必要性 |
| w/o Stage-2（去除 RL 训练） | 性能下降，证明 RL 优化的重要性 |
| w/o Group Reward（去除组奖励） | 检索次数增加，内部知识利用率下降 |
| w/o Memorization（去除记忆化） | 性能下降，知识内化能力丧失 |
| w/o Both（同时去除两者） | 性能降幅最大，证明两个组件的协同效应 |

所有消融变体的性能均低于完整方法，证实了每个组件的不可或缺性。

## 局限性

1. **搜索引擎依赖**：外部检索仍依赖传统搜索引擎，检索质量受搜索引擎能力限制
2. **记忆化效率**：记忆化机制需要额外的重写模型，增加了训练流程的复杂性和计算开销
3. **知识冲突处理**：当内部知识与外部检索结果存在冲突时，模型的处理策略有待进一步研究
4. **评估局限**：主要在多跳问答任务上评估，对其他类型的知识密集型任务（如事实验证、摘要生成）覆盖不足
5. **模型规模限制**：实验主要在 7B-14B 规模的模型上进行，更大规模模型上的表现需要验证
6. **领域特定知识**：记忆化机制对领域特定知识的内化效果可能因领域而异

## 核心要点

- R1-Searcher++ 的核心贡献在于将"何时搜索"的决策问题形式化为 RL 优化问题，而非简单规则
- 内部知识利用奖励（Group Reward）是一个简洁而有效的创新：用额外奖励激励模型"少查但查对"
- 记忆化机制实现了"外部知识 -> 内部知识"的持续转化，使模型随训练推进变得越来越"博学"
- 检索次数减少 42.9% 而性能仍然提升，这一结果有力证明了大量现有检索是冗余的
- SFT 冷启动对于 RL 训练的成功至关重要，直接从零开始 RL 训练会导致格式混乱和性能崩溃
- 该工作与 Search-R1、AutoAgent-Search 等同期工作共同推动了"RL 驱动的检索增强推理"这一新方向
- R1-Searcher++ 来自人民大学 RUCAIBox 团队，该团队在推荐系统和 LLM 领域有深厚积累
- 记忆化机制的核心思路是通过"重写模型"将检索文档转化为纯推理路径，再用 NLL 损失训练主模型
- 域外基准（Musique、Bamboogle）上的表现证明了方法的泛化能力，而非仅在域内过拟合
- "内外知识自适应"这一研究方向具有广泛的实际应用价值，因为现实中的大量查询是模型已知的
- 与传统 RAG 方法（如 REALM、REPLUG）的根本区别在于：R1-Searcher++ 将检索决策纳入 RL 优化，而非依赖固定规则或阈值
