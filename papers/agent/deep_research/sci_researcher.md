# SciResearcher: Scaling Deep Research Agents for Frontier Scientific Reasoning

## 简要总结

SciResearcher 是一个面向前沿科学推理的全自动智能体框架，旨在解决当前深度研究智能体在前沿科学领域面临的核心瓶颈：领域知识分散在稀疏且异构的学术来源中，问题求解需要远超事实记忆的复杂计算与推理能力。该框架通过自动化的数据构建流水线，合成基于学术证据的多样化概念性和计算性任务，并通过监督微调（SFT）和智能体强化学习（Agentic RL）训练出 SciResearcher-8B 模型。该模型在 HLE-Bio/Chem-Gold 基准上达到 19.46% 的准确率，确立了同参数规模下的新最优性能，在 SuperGPQA-Hard-Biology 和 TRQA-Literature 基准上分别取得 13-15% 的绝对提升。

## 动机与问题

### 核心挑战

现有深度研究智能体主要聚焦于通用信息检索任务，其能力通常通过知识图谱构建或迭代网页浏览来培养。然而，前沿科学推理与通用搜索存在本质差异：

1. **知识稀疏性与异构性**：前沿科学的领域知识分散在大量学术论文、数据库和专业资源中，不像通用知识可以在主流网页上轻松获取。
2. **计算推理深度**：前沿科学问题往往需要多步骤的数学推导、实验设计分析和复杂的逻辑链条，远非简单的事实召回可以应对。
3. **长视野规划能力**：解决一个科学问题可能需要智能体执行多轮信息获取、工具调用和推理整合，形成长视野的任务规划链条。
4. **数据构建困难**：高质量前沿科学训练数据的获取成本极高，传统手工标注方法难以规模化。

### 研究目标

构建一个能够自动化合成高质量科学推理训练数据的框架，并通过后训练方法赋予小规模语言模型强大的前沿科学推理能力。

## 方法

### 全自动数据构建流水线

SciResearcher 的核心创新在于其全自动化的前沿科学数据构建框架，该流水线包含以下关键步骤：

1. **学术证据采集**：从学术论文和科学数据库中系统性地提取领域知识，构建覆盖多学科领域的知识基础。
2. **任务合成**：基于采集到的学术证据，自动合成两类任务：
   - **概念性任务**：考察对科学概念的深层理解、跨领域关联和理论推演能力
   - **计算性任务**：需要数学推导、数据分析、公式应用等定量推理的问题
3. **能力维度设计**：任务设计覆盖三个核心能力维度：
   - **信息获取**（Information Acquisition）：从学术来源中精准检索相关知识
   - **工具集成推理**（Tool-Integrated Reasoning）：调用计算工具、代码执行器等辅助推理
   - **长视野规划**（Long-Horizon Planning）：制定多步骤解题策略并逐步执行

### 两阶段后训练

1. **监督微调阶段（SFT）**：
   - 使用高质量的教师模型生成轨迹数据
   - 轨迹数据经过两阶段处理：七条过滤规则移除低质量轨迹，四个清洗步骤标准化保留的数据
   - 通过 SFT 为模型建立基本的科学推理和工具调用能力

2. **智能体强化学习阶段（Agentic RL）**：
   - 在 SFT 的基础上进一步通过强化学习优化策略
   - RL 不仅弥合了与教师模型的差距，还超越了教师模型的性能
   - 通过奖励信号引导智能体学习更优的搜索策略和推理路径

### 数据过滤与清洗

轨迹数据的质量控制是整个流水线的关键环节，采用两阶段处理机制：

#### 过滤阶段（7 条规则）

对教师模型生成的原始轨迹应用七条过滤规则，系统性地移除低质量样本：
- 移除推理链条中断或逻辑不连贯的轨迹
- 过滤工具调用失败率过高的轨迹
- 排除最终答案与参考答案不一致的样本
- 去除过短（缺乏推理深度）或过长（包含冗余循环）的轨迹

#### 清洗阶段（4 个步骤）

对通过过滤的轨迹进行四步标准化处理：
- 统一工具调用格式和响应格式
- 规范化推理过程中的术语表达
- 修正轨迹中的格式异常和标记错误
- 对齐不同来源的数据格式以确保训练一致性

### 基座模型与训练细节

SciResearcher-8B 基于 8B 参数规模的基座模型构建，通过上述两阶段后训练流程获得强大的科学推理能力。SFT 阶段使用高质量教师模型（如 Tongyi DeepResearch）生成的轨迹作为监督信号，RL 阶段则通过基于奖励的策略优化进一步释放模型的推理潜力。

## 关键创新

1. **前沿科学数据的全自动合成**：首次提出针对前沿科学领域的全自动化数据构建框架，突破了传统手工标注的规模瓶颈，使得高质量科学推理数据的大规模生产成为可能。

2. **学术证据锚定**：所有合成任务均以真实学术文献为锚点，确保任务的科学严谨性和知识的准确性，避免了合成数据常见的"脱离实际"问题。

3. **多维能力协同训练**：将信息获取、工具集成推理和长视野规划三种能力统一在一个训练框架中，使模型能够在解题过程中灵活调用不同能力。

4. **小模型超越大模型**：通过精心设计的数据和训练流程，8B 参数规模的模型在前沿科学推理任务上超越了多个更大规模的专有智能体。

## 实验设置

### 评测基准

1. **HLE-Bio/Chem-Gold**：高难度人类水平评测基准中的生物学和化学黄金子集，专门考察前沿科学推理能力。
2. **SuperGPQA-Hard-Biology**：SuperGPQA 基准的困难生物学子集，涵盖 285 个研究生级别学科的知识评测，采用人类-LLM 协同过滤机制消除平凡或模糊的题目。
3. **TRQA-Literature**：Total Recall Question Answering 基准的文献子集，基于 Wikidata-Wikipedia 构建的深度研究评测框架，考察全面的信息检索和整合能力。

### 对比基线

与多个更大规模的专有深度研究智能体进行对比，包括当前主流的商业级研究智能体系统。

### 评测基准详解

#### HLE（Humanity's Last Exam）

HLE 是一个极高难度的评测基准，旨在考察 AI 在最前沿知识领域的推理能力。其 Bio/Chem-Gold 子集聚焦于生物学和化学中最具挑战性的问题，这些问题通常需要研究生级别以上的专业知识和多步骤推理能力。19.46% 的准确率虽然看起来不高，但考虑到该基准的极端难度，这一成绩具有重要意义。

#### SuperGPQA

SuperGPQA 横跨 285 个研究生级别学科，通过人类-LLM 协同过滤机制确保问题质量。其 Hard 子集进一步聚焦于最困难的问题，Hard-Biology 子集则专门考察前沿生物学领域的深度推理能力。

#### TRQA（Total Recall Question Answering）

TRQA 受 TREC Total Recall Tracks 启发，基于 Wikidata-Wikipedia 构建，要求智能体在大规模文献库中实现全面的信息检索和整合。Literature 子集考察文献综合分析能力，是衡量深度研究智能体信息整合能力的重要基准。

## 实验结果

### 核心性能指标

| 基准 | SciResearcher-8B 性能 | 提升幅度 |
|------|----------------------|---------|
| HLE-Bio/Chem-Gold | 19.46% | 同规模最优 |
| SuperGPQA-Hard-Biology | +13% 绝对提升 | 显著超越基线 |
| TRQA-Literature | +15% 绝对提升 | 显著超越基线 |

### 关键发现

1. **参数效率**：8B 参数的 SciResearcher 在前沿科学推理任务上超越了多个更大规模的专有智能体，证明了高质量训练数据和合理训练策略的重要性远大于单纯的模型规模。
2. **RL 的关键作用**：强化学习阶段不仅弥合了与 SFT 教师模型之间的性能差距，还实现了超越，表明 RL 对于科学推理能力的发展至关重要。
3. **数据质量优于数量**：经过严格过滤和清洗的训练轨迹比原始大规模数据产生了更好的效果。
4. **跨学科泛化**：在生物学、化学和文学等不同学科上均展现出强劲的性能提升，说明框架具有良好的跨领域泛化能力。

## 局限性

1. **领域覆盖范围**：虽然在生物、化学和文学领域表现出色，但对物理学、工程学等其他前沿科学领域的覆盖和评测有待进一步验证。
2. **实验科学局限**：该框架主要适用于可以通过文献检索和计算推理解决的科学问题，对于需要实验室实验验证的前沿问题仍存在根本性限制。
3. **评测基准局限**：现有基准可能无法完全反映真实科学研究中的复杂性，如跨学科整合、创新假设生成等更高层次的能力。
4. **模型规模天花板**：虽然 8B 模型表现出色，但在最困难的问题上仍可能受限于模型容量，更大规模模型的潜力有待探索。
5. **数据时效性**：学术文献持续更新，训练数据的时效性可能影响模型在最新前沿问题上的表现。

## 核心要点

1. SciResearcher 证明了前沿科学推理不仅仅是"更难的问答"，它需要专门设计的数据构建流水线来捕捉科学问题的独特特征：知识稀疏性、计算复杂性和长视野依赖性。
2. 全自动数据合成范式可能成为未来科学智能体训练的主流方法，因为它同时解决了数据规模和数据质量的双重挑战。
3. 小模型在高质量数据和合理训练策略加持下可以在特定领域超越大模型，这为资源受限场景下的科学智能体部署提供了可行路径。
4. SFT + Agentic RL 的两阶段训练范式被进一步验证为构建强大科学推理智能体的有效策略。
5. 该工作为"AI for Science"领域提供了一个重要的基础设施组件——可扩展的科学推理训练数据生成框架。

## 与相关工作的对比

### 与通用深度研究智能体的区别

| 维度 | 通用深度研究智能体 | SciResearcher |
|------|------------------|---------------|
| 知识来源 | 通用网页 | 学术论文和科学数据库 |
| 任务类型 | 事实检索为主 | 概念+计算混合任务 |
| 推理深度 | 中等 | 深度（多步数学推导） |
| 数据构建 | 知识图谱/网页浏览 | 全自动学术证据合成 |
| 目标领域 | 通用 | 前沿科学 |

### 与其他科学推理系统的对比

SciResearcher 与 SciAgent 等多智能体科学推理系统不同之处在于：它不仅关注推理过程本身，还关注训练数据的自动化构建，形成了"数据构建-模型训练-能力评估"的完整闭环。

### 技术路线演进

SciResearcher 代表了深度研究智能体从"通用搜索"向"领域科学推理"演进的一个重要分支。该路线强调：
1. 领域特异性的训练数据比通用数据更有效
2. 学术证据锚定可以显著提升科学推理的可靠性
3. SFT + RL 两阶段训练在科学领域同样有效，且 RL 阶段的增益可能比通用领域更为显著
