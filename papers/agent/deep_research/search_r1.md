# Search-R1：通过强化学习训练大语言模型进行推理并利用搜索引擎

## 摘要

Search-R1 是一个强化学习训练框架，使大语言模型能够将多轮推理与实时搜索引擎调用交替进行。其核心技术贡献是"检索token掩码"——将外部检索的token从强化学习损失计算中排除以稳定训练。使用简单的基于结果的奖励（答案正确性），该框架在七个问答基准上实现了 +26%（Qwen2.5-7B）、+21%（Qwen2.5-3B）和 +10%（LLaMA3.2-3B）的平均相对提升，超越了强基线方法。

## 动机与问题

DeepSeek-R1 的出现表明，强化学习可以通过思维链教导大语言模型进行推理。一个自然的延伸是：强化学习能否也教导大语言模型在推理过程中动态查询外部知识源？这解决了三个根本问题：

1. **知识局限**：大语言模型将知识存储在参数中，这本质上是不完整的，且可能过时
2. **幻觉**：在没有真实来源访问的情况下，大语言模型会生成看似合理但事实不正确的信息
3. **静态检索**：现有的 RAG 流程在生成前进行一次检索；无法根据中间推理步骤自适应地优化查询

**为何在强化学习中这很困难？**
- 标准强化学习的rollout仅包含模型生成的token，但搜索增强的rollout混合了模型token和外部检索文本
- 检索文本不是由策略生成的，不应接收梯度更新
- 在混合序列上进行朴素的强化学习训练会导致训练不稳定和发散

## 方法

### 框架概述

```
+--------------------------------------------------------------------+
|  Search-R1 训练循环（基于 veRL / vLLM 基础设施构建）                   |
|                                                                    |
|  对于每个训练问题 q：                                                 |
|    1. 从策略大语言模型采样 G 个 rollout                                |
|    2. 每个 rollout 交替进行：                                         |
|       <think>...</think>（模型生成的推理）                             |
|       <search>query</search>（模型生成的搜索查询）                     |
|       <result>...</result>（外部检索——已掩码）                         |
|       ...（对多轮重复）                                               |
|       <answer>final_answer</answer>                                |
|    3. 计算结果奖励：r = correctness(answer, gold)                    |
|    4. 对损失应用检索token掩码                                         |
|    5. 通过 GRPO 更新策略                                              |
+--------------------------------------------------------------------+
```

### 多轮交替推理与搜索

模型以结构化格式在推理和搜索之间交替：

```
第1轮：
  <think> 我需要找到 X 发生的时间... </think>
  <search> when did X happen </search>
  <result> [检索到：X 发生在 1995 年，根据...] </result>

第2轮：
  <think> 现在我知道 X 是 1995 年。我需要找到当时的总统是谁... </think>
  <search> US president 1995 </search>
  <result> [检索到：Bill Clinton 从 1993 年到 2001 年担任总统...] </result>

最终：
  <think> X 发生在 1995 年，当时 Clinton 是总统。 </think>
  <answer> Bill Clinton </answer>
```

### 检索token掩码（核心创新）

关键洞察：检索到的token是由搜索引擎注入的，不是由策略模型生成的。将它们包含在强化学习损失中会：
- 将外部文本质量归因于策略，导致噪声梯度
- 产生分布不匹配（检索文本 != 模型的分布）
- 导致训练不稳定或崩溃

**解决方案**：将 `<result>...</result>` 标签内的所有token从损失计算中掩码：

```
Loss_masked = sum over t in T_model [ -advantage * log pi(a_t | s_t) ]
  其中 T_model = <result>...</result> 范围外的所有token位置

<result>...</result> 内的token被排除在以下计算之外：
  - 策略梯度计算
  - KL 散度惩罚
  - 价值函数估计
```

### 奖励函数

使用答案正确性的简单基于结果的奖励：

```
r(rollout) = EM(predicted_answer, gold_answer)
  其中 EM = 精确匹配（或用于部分评分变体的 F1）
```

无过程奖励，无检索质量奖励——仅最终答案正确性。这种奖励的简洁性是有意为之的：它让模型通过探索发现自己的搜索策略。

### 强化学习算法：GRPO

```
GRPO（组相对策略优化）：
  输入：问题 q，策略 pi_theta，奖励函数 r
  1. 采样 rollout：{o_1, ..., o_G} ~ pi_theta(. | q)
  2. 计算奖励：{r_1, ..., r_G}
  3. 计算优势：A_i = (r_i - mean(r)) / std(r)
  4. 策略梯度：
     L(theta) = -1/G * sum_i [ A_i * sum_t log pi_theta(o_i^t | o_i^{<t}, q) ]
     （仅计算非掩码token）
  5. KL 惩罚：D_KL(pi_theta || pi_ref) 加入损失
```

PPO 也进行了评估：GRPO 收敛更快但可能出现不稳定性；PPO 提供更稳定的优化但收敛更慢。

### 基础设施

基于 veRL（Volcano Engine RL）框架和 vLLM 构建，用于高效推理。这一点值得注意，因为多轮搜索交互需要精心的工程设计：
- 每次搜索调用都会暂停生成、查询检索系统并注入结果
- 框架处理具有不同搜索轮次数量的可变长度rollout

## 关键创新

1. **检索token掩码**：对在强化学习训练中混合模型生成token和外部来源token这一根本问题的原则性解决方案。这是主要的技术贡献，对训练稳定性至关重要。

2. **无 SFT 预热的端到端强化学习**：与 R1-Searcher（使用两阶段）不同，Search-R1 直接从指令微调的基座模型开始进行强化学习训练，证明正确的掩码策略无需课程分解即可满足需求。

3. **基于 veRL 的可扩展框架**：实际的工程贡献——该框架处理了大规模多轮搜索增强强化学习的复杂性。

4. **跨模型通用性**：在三个不同的基座模型（Qwen2.5-7B、Qwen2.5-3B、LLaMA3.2-3B）上进行了验证，表明该方法不局限于特定模型。

## 实验设置

**基础模型：**
- Qwen2.5-7B-Instruct
- Qwen2.5-3B-Instruct
- LLaMA-3.2-3B-Instruct

**训练数据：**
- 域内：NQ（Natural Questions）、HotpotQA 训练集

**评估基准（7个数据集）：**
- 域内：NQ、HotpotQA
- 域外：TriviaQA、PopQA、2WikiMultiHopQA、Musique、Bamboogle

**基线方法：**
- 无检索（直接提示）
- 标准 RAG（单轮先检索后阅读）
- 迭代式 RAG 方法
- R1-Searcher（同期工作）

**检索后端：**
- 基于维基百科语料库的稠密检索
- 每个搜索查询返回 Top-k 段落

## 结果

**相对于基线的平均相对提升：**

| 基础模型 | 平均相对提升 |
|-----------------|--------------------------|
| Qwen2.5-7B     | +26%                     |
| Qwen2.5-3B     | +21%                     |
| LLaMA-3.2-3B   | +10%                     |

**泛化能力：** 提升在域内（NQ、HotpotQA）和域外（TriviaQA、PopQA、2WikiMultiHopQA、Musique、Bamboogle）基准上均成立，表明学习到的搜索行为可以迁移到未见过的问题分布。

**强化学习算法比较：**
- GRPO：收敛更快，但在长时间训练后可能出现不稳定性
- PPO：更慢但更稳定的收敛轨迹

**检索token掩码消融实验：**
- 无掩码：训练变得不稳定，性能显著下降
- 有掩码：训练曲线稳定，各基准上持续改进
- 这一消融实验证实了掩码不仅有帮助，而且是必不可少的

**搜索轮次数量：**
- 模型学会了平均使用 1-3 个搜索轮次，多跳问题使用更多轮次，单跳问题使用更少轮次
- 这种自适应搜索行为在没有明确指令的情况下从强化学习中自然涌现

## 局限性

1. **仅维基百科语料库**：与 R1-Searcher 类似，评估仅限于基于维基百科的检索；开放网络搜索结果可能引入额外挑战（噪声、矛盾信息、格式不一致）
2. **聚焦事实问答**：所有七个基准都是事实型问答任务；该方法未在长文本研究、报告生成或综合分析上进行评估
3. **固定检索器**：检索系统未与大语言模型策略联合训练；模型必须处理检索器返回的任何结果
4. **计算开销**：多轮搜索在训练（rollout生成更慢）和推理过程中都增加了延迟
5. **简单奖励**：仅基于结果的奖励可能无法捕捉检索质量、推理连贯性或来源归因等细微差别
6. **未与蒸馏方法比较**：未与从更强教师模型蒸馏搜索行为的模型进行比较

## 核心要点

- 检索token掩码是使用搜索增强推理进行稳定强化学习训练的关键技术洞察——没有它，训练会失败
- 简单的结果奖励（答案正确性）足以让模型通过探索学习复杂的多轮搜索策略
- 该方法可跨模型系列（Qwen、LLaMA）和模型规模（3B、7B）泛化
- Search-R1 建立了基础框架（基于 veRL），许多后续深度研究智能体论文在此基础上构建
- 发表于 COLM 2025，使其成为强化学习用于搜索增强推理范式中最早经过同行评审的论文之一
- GitHub 上的开源代码库成为社区构建搜索增强强化学习智能体的广泛使用的起点
