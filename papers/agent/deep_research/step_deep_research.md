# Step-DeepResearch：低成本端到端深度研究智能体技术报告

## 简要总结

Step-DeepResearch 是由阶跃星辰（StepFun）开发的端到端深度研究智能体模型。该模型仅使用 32B 参数，在 Scale AI Research Rubrics 基准上达到 61.4% 的得分，与 OpenAI Deep Research 和 Gemini Deep Research 等闭源系统性能相当，但单次调用成本不到 0.50 元人民币——不到顶级商业系统（Gemini≈6.65元、OpenAI≈5.32元）的十分之一。核心方法论包括：基于原子能力的数据合成策略、从代理式中期训练到监督微调再到强化学习的渐进式训练流水线、检查清单式评判器奖励设计，以及面向中文场景的 ADR-Bench 基准。

## 动机与问题

### 现有深度研究系统的局限

1. **基准缺陷**：BrowseComp 等现有基准往往无法满足开放式研究的真实需求，仅关注封闭式问答而忽视长篇报告生成的质量评估
2. **成本壁垒**：顶级闭源深度研究系统部署成本极高，72B+ 参数的模型推理开销限制了实际应用的可扩展性
3. **能力碎片化**：现有训练方法通常将深度研究视为整体任务，缺乏对构成研究过程的原子能力的系统性分解和强化
4. **中文研究空白**：缺少专门针对中文深度研究场景的高质量评估基准

### 核心研究问题

> 中等规模模型（32B）能否通过精细化的训练策略达到专家级深度研究能力，同时保持显著的成本优势？

## 方法

### 一、基于原子能力的数据合成策略

Step-DeepResearch 将深度研究分解为四类核心可训练原子能力，并为每类设计专门的数据合成方法：

#### 1. 规划与任务分解
- **逆向工程合成**：从高质量文档（研究报告、综述论文）出发，通过后见引导逆向生成复杂规划数据
- 训练模型学习将开放式查询分解为结构化子任务序列

#### 2. 深度搜索与信息检索
- **图结构合成**：利用知识图谱和多文档拓扑图构建多跳推理问题
- 多文档拓扑遍历（topology walking）确保训练数据覆盖真实场景中的复杂检索路径

#### 3. 反思、验证与交叉验证
- **错误-反思循环**：构建包含 Extract → Plan → Verify → Replan → Report 的多智能体验证工作流
- 注入跨源交叉验证能力，使模型能够对比多个信息源以检测矛盾和不一致

#### 4. 报告生成
- 中期训练阶段注入领域写作风格
- 监督微调阶段优化指令遵循和格式控制

**设计理念**：将训练目标从"预测下一个token"重塑为"决定下一个原子动作"，这在实证中改善了复杂环境下的鲁棒性并增强了跨任务泛化能力。

### 二、渐进式训练流水线

#### 阶段一：代理式中期训练（Agentic Mid-Training）

分为两个子阶段：

**32K 上下文阶段**：
- 注入基础能力但不引入显式工具调用
- 训练数据量达 1500 亿 token
- 在 FRAMES 基准上取得 +10.88% 的增益
- 目标：建立信息理解和推理的基础能力

**128K 上下文阶段**：
- 扩展到工具调用、检索规划和多工具协调
- 使模型具备在超长上下文中维持研究一致性的能力

#### 阶段二：监督微调（SFT）

将原子能力组合为端到端研究轨迹：
- **轨迹效率优化**：强调最短正确路径
- **受控噪声注入**：增强模型对真实检索噪声的鲁棒性
- **N-gram 去重**：防止生成冗余内容
- **严格引用强制**：训练模型在报告中准确标注信息来源

#### 阶段三：强化学习（RL）

- 使用 PPO 算法进行在线策略优化，在真实工具环境中交互
- **两步逆向合成**：自动生成任务和对应的评分准则
- **严格二值奖励映射**：将三级评判（满足/部分满足/不满足）转化为二值信号（正面准则 1/0，负面准则 0/1），消除中间状态的噪声
- **优势估计**：使用 GAE（$\gamma=1, \lambda=1$）进行长序列信用分配
- 施加工具调用预算和 token 限制以控制训练成本

### 三、检查清单式评判器（Checklist-style Judger）

奖励系统的核心机制：

```
评判流程：
1. 为每个研究任务生成多维度检查清单（正面准则 + 负面准则）
2. 准则要求满足：原子性、可验证性、无歧义性、独立性、与任务对齐
3. 评判器对每条准则进行二值评判
4. 加权汇总获得最终奖励信号

训练流程：
Phase 1: SFT - 使用强模型标注作为监督信号
Phase 2: RL - 使用可验证奖励进一步优化评判能力
```

### 四、系统架构

**单智能体 ReAct 循环**：Reasoning → Action → Observation 交替执行

**工具集设计**：
- 权威索引：600+ 权威网站的策划索引
- 知识密集检索：2000万+ 段落级知识库
- 权威感知排序：基于启发式规则的权威性排序
- Token 高效补丁编辑：在本地编辑中减少 >70% token 消耗
- 隐式上下文管理：基于摘要感知的两层策略
- 有状态 TODO 管理：目标对齐的任务追踪
- Tmux 集成沙箱：安全的代码执行环境
- 感知优化浏览器：针对研究场景优化的网页浏览

**上下文管理**：
- 一般任务使用摘要与折叠策略
- 深度研究任务使用保留引用的变体，在压缩过程中维护引用完整性

### 五、ADR-Bench（应用驱动深度研究基准）

**覆盖范围**：法律、IT、教育、金融、科学、社会、艺术、医疗、政治共 9 个领域

**数据规模**：
- 通用领域：70 个查询（7 个领域各 10 个，来自真实用户场景）
- 专业领域：40 个查询（法律和金融各 20 个，由领域专家撰写）
- 总计：110 个查询，双轨评估

**评估框架**：
- 通用领域：盲法比较评估（五级评定：左优/右优/均好/均一般/均差），覆盖信息完整性、内容深度、需求契合度、可读性四个维度
- 专业领域：基于专家策划准则的自动化评估
- Elo 风格评分协议，将自动化指标与人类感知有用性关联

## 关键创新

1. **原子能力分解训练**：首次系统性地将深度研究分解为可独立训练的原子能力（规划、搜索、反思、生成），而非将其视为不可分割的整体任务
2. **三阶段渐进式训练**：从代理式中期训练到 SFT 再到 RL 的完整优化路径，训练目标从 token 预测渐进到原子动作决策
3. **检查清单式评判器**：通过二值化奖励映射和多维度准则评估，为开放式研究报告提供稳健的自动化评判
4. **ADR-Bench**：首个面向中文深度研究场景的应用驱动基准，填补了现有评估体系在中文和开放式研究方面的空白
5. **极致成本效率**：32B 参数实现与 72B+ 闭源系统可比的性能，单次调用成本降低一个数量级

## 实验结果

### Scale AI Research Rubrics 基准

| 模型 | 得分 |
|------|------|
| **Step-DeepResearch (32B)** | **61.4%** |
| OpenAI Deep Research | 可比水平 |
| Gemini Deep Research | 可比水平 |

### ADR-Bench 表现

- 在所有评估维度上获得最高 Elo 评分
- 超越 Kimi-Researcher、MiniMax Agent 等商业系统
- 在通用领域和专业领域均表现优异

### FRAMES 基准

- 代理式中期训练阶段取得 +10.88% 增益

### 成本对比

| 系统 | 单次调用成本（RMB） |
|------|-------------------|
| Step-DeepResearch | < 0.50 |
| OpenAI Deep Research | ≈ 5.32 |
| Gemini Deep Research | ≈ 6.65 |

Step-DeepResearch 实现了成本-性能帕累托前沿上的最优位置。

## 局限性

1. **评分准则生成挑战**：模型自动生成的评分准则在可评估性方面仍有不足，需要大量人工修订
2. **评估成本高昂**：Elo 对战消耗的资源是普通比较评估的 3 倍
3. **领域专家稀缺**：难以找到愿意投入时间进行深度评估的领域专家
4. **开放式评估困难**：在开放式长篇内容评估中准确性维度的验证极为困难，比较评估不得不省略准确性维度
5. **领域粒度权衡**：ADR-Bench 中 6-12 个领域为最优范围，过少导致覆盖不足，过多引起领域重叠
6. **工具环境约束**：RL 训练中必须施加工具调用预算和 token 限制以控制成本，这可能限制某些复杂查询的探索深度

## 核心要点

- Step-DeepResearch 证明了中等规模模型（32B）通过精细化的原子能力训练和渐进式优化，可以在深度研究任务上达到与顶级闭源系统可比的性能，同时成本降低超过 10 倍
- 原子能力分解的训练范式将"预测下一个 token"转变为"决定下一个原子动作"，是深度研究智能体训练方法论的重要创新
- 检查清单式评判器为开放式研究报告的自动化评估提供了可行方案，但评分准则的自动生成质量仍是瓶颈
- ADR-Bench 填补了中文深度研究评估基准的空白，其双轨评估框架（盲法比较 + 准则自动评估）为后续工作提供了参考范式
- 该工作的实际意义在于大幅降低了高质量深度研究服务的部署门槛，使中等规模计算资源的团队也能提供专家级研究能力
