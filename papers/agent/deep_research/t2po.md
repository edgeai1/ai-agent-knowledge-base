# T²PO：基于不确定性引导的探索控制，实现稳定的多轮智能体强化学习

## 简要总结

T²PO（Token-level 和 Turn-level 渐进优化）是一个**不确定性感知框架**，通过在 token 级和 turn 级两个粒度上显式控制探索，解决多轮智能体 RL 训练中的不稳定性问题。在 token 级，T²PO 监控不确定性动态并在边际变化低于阈值时触发思考终止干预；在 turn 级，识别探索进展微乎其微的交互轮次并动态重采样。在 WebShop 上成功率达到 81.64%（4B）/ 82.42%（8B），在 ALFWorld 上比先前 SOTA 提高 8-12 个百分点，同时显著降低训练方差。

## 背景与问题

### 研究背景

多轮强化学习（Multi-turn RL）是训练 LLM 智能体的基础范式——让智能体通过与环境的迭代交互来推理、行动和自我进化。近年来 GRPO、GiGPO、DAPO 等方法取得了显著进展，但一个核心问题始终未解决：**训练不稳定性**。

### 问题的具体表现

论文通过实验（Figure 1）展示了这一问题的严重性：在不同环境初始化随机种子下运行 SOTA 基线方法，可以观察到：
- **成功率突然崩塌**：训练初期性能上升，但随后急剧下降
- **KL 散度爆炸**：策略与参考模型的距离急剧增大
- **梯度范数飙升**：梯度不稳定，训练无法收敛

这种现象被称为 **training collapse**（训练崩溃）。

### 现有方法的不足

先前工作试图通过以下方式缓解：
1. **细粒度信用分配**（如过程奖励模型）—— 改善了奖励信号但未消除根因
2. **轨迹过滤**（如 SimpleTIR 过滤空轮、rStar2-Agent 仅保留高质量轨迹）—— 通过启发式数据选择，但不直接调控推理过程
3. **内部奖励**（如 SEED-GRPO 的熵奖励、DeepConf 的置信度过滤）—— 依赖单一尺度信号或静态奖励塑形

**核心局限**：这些方法要么在粗粒度的轨迹层面操作，要么通过间接的奖励塑形隐式控制，在多轮设置的复杂性下对超参数高度敏感，仍然频繁导致训练崩溃。

### 核心洞察：问题的根源是「低效探索」

论文的关键洞察来自对代表性训练轨迹的分析。作者发现不稳定性的根本原因是**低效探索（insufficient exploration）**——探索-利用权衡的系统性违反。他们将这种失败模式命名为**「犹豫」（hesitation）**：

**Token 级犹豫**：推理型 LLM 受"顿悟时刻"（aha moment）启发，倾向于在生成动作前产生冗长的 CoT。虽然推理提升了决策质量，但过长的内部序列带来两个问题：
- 信息增益在某个点后迅速饱和，但采样噪声持续累积
- 计算开销增大，策略梯度方差被放大

**Turn 级犹豫**：一旦智能体对环境的感知稳定下来，它可能在多个轮次中重复产生语义相似但失败的推理轨迹，导致：
- 冗余交互，探索效率降低
- 恢复空间有限——在有限的交互预算内几乎无法脱困

> **"Hesitation is defeat"**（犹豫即失败）—— 论文将此作为核心命题，智能体一旦陷入犹豫模式，在有限预算内几乎无法恢复。

### Idea 的发现过程

作者的思路演进：
1. **观察**：分析 SOTA 基线的训练轨迹，发现成功率崩塌伴随着 KL 散度和梯度范数的异常
2. **诊断**：进一步分析发现，崩塌的根因不是奖励稀疏或信用分配，而是策略在生成大量低信息量 token/turn 时浪费了探索预算
3. **洞察**：既然问题是低效探索，那么解决方案不应是增加更多奖励信号，而是**直接控制探索本身**
4. **方案**：利用模型内在的不确定性信号（而非外部奖励）来检测并消除犹豫行为

## 方法

### 整体框架

T²PO 在两个粒度上显式控制探索：

```
┌────────────────────────────────────────────────────────────┐
│                     T²PO 框架                              │
│                                                            │
│  Token 级：思考干预（TTI）                                  │
│  ┌──────────────────────────────────────┐                  │
│  │ 监控不确定性信号 M_t 的边际变化       │                  │
│  │ 当 1/(N+1)·Σ Δ_{t-i} < ε 时          │                  │
│  │ → 强制终止思考，注入 </think><action> │                  │
│  └──────────────────────────────────────┘                  │
│                                                            │
│  Turn 级：动态采样（TDS）                                   │
│  ┌──────────────────────────────────────┐                  │
│  │ 聚合 token 级信号为 turn 级信号 Φ^k    │                  │
│  │ 监控连续轮次的变化 Γ^k = |Φ^k - Φ^{k-1}| │              │
│  │ 当 Γ^k < η 时 → 丢弃并重新生成该轮    │                  │
│  └──────────────────────────────────────┘                  │
│                                                            │
│  策略更新：分层优势估计 + 裁剪目标 + KL 惩罚               │
└────────────────────────────────────────────────────────────┘
```

### 4.1 自校准不确定性信号（Self-calibrated Uncertainty Signal）

#### 为什么不能只用熵或只用置信度？

论文指出，常用的两个不确定性度量各有盲区：

**Shannon 熵 H_t**：
```
H_t = -Σ_{i=1}^{V} p_t^{(i)} log p_t^{(i)}
```
- 优点：反映分布的整体平滑程度
- 盲区：当分布接近均匀或高度集中时，辨别力有限。对于大词汇表（如 Qwen3 的 152K），两个截然不同的预测之间的熵差可能极小（仅 log 2）

**Token 置信度 C_t**：
```
C_t = -(1/j) Σ_{i=1}^{j} log p_t^{(i)}    (top-j tokens 的负平均对数概率)
```
- 优点：反映 top-k token 的集中程度
- 盲区：仅依赖 argmax token 的概率，完全忽略剩余概率质量的分布方式。如论文 Figure 3 所示，非常不同的 token 分布可能产生相同的置信度

#### 融合方案

两者互补：H_t 覆盖平滑分布，C_t 覆盖尖锐分布。先分别归一化到 [0,1]：

```
H̄_t = (H_t - H_min) / (H_max - H_min)
C̄_t = (C_t - C_min) / (C_max - C_min)
```

然后构造自校准信号：

```
M_t = α · H̄_t + (1-α) · (1 - C̄_t),    α ∈ [0, 1]
```

**M_t 的值越高，表示模型越确定**（低熵 + 高置信度）。当 M_t 的变化趋于平坦时，表示继续生成 token 不再带来新的信息——即犹豫的信号。

关键优势：M_t 的等高线是非退化的分段线性形状（而非 C_t 的 max 算子导致的退化），既保持了 top-1 驱动的层级化，又保留了熵对尾部分散的敏感性。

### 4.2 Token 级思考干预（TTI）

#### 动机

推理型 LLM 在生成动作前产生详细的 CoT（`<think>...</think>`）。论文分析发现（Figure 4a）：
- 不确定性信号 M_t 在生成过程中先下降后趋于平坦
- 最不确定的 token（词云 Figure 4b）恰好是与任务相关的关键词（产品名称、属性描述等）
- **关键发现：在信息增益饱和之后生成的 token 只会降低探索效率**

#### 何时停止？

监控 M_t 的时间变化率：
```
Δ_t^k = |M_t^k - M_{t-1}^k|
```

使用滑动窗口平均来平滑局部波动，定义**非犹豫事件**：当滑动窗口均值低于容忍阈值 ε 时触发：

```
1/(N+1) · Σ_{i=0}^{N} Δ_{t-i}^k < ε        (公式 4)
```

第一次满足此条件的时间步记为 t*。

**为什么不在峰值处截断？** 论文分析了高不确定性 token 的两种类型：
1. 连接词/话语标记——与推理转换相关的"顿悟时刻"
2. 任务特定的语义 token（产品名、属性）——完成任务所必需的信息

直接在峰值截断会过早移除关键语义内容。滑动窗口平滑确保只有在**持续的高不确定性**被检测到时才触发。

#### 如何停止？

在 t* + 1 步，通过**覆写 logits** 强制生成 `</think>` token：

```
z_{t*+1}(v) = { +∞,  若 v = 151668 (</think> 的 token id)
              { -∞,  若 v ≠ 151668
```

这使得 p(y_{t*+1} = </think> | y_{≤t*}) = 1。

然后注入确定性 token 队列 Q = [</think>, \n, <action>]，不从模型分布采样，确保推理和执行阶段的边界清晰。

#### 约束条件

1. **一次性激活**：每次生成只触发一次 TTI，用二元指示器 I_stop 控制
2. **全局思考预算**：设定最大解码长度 L_max，即使 TTI 未触发也在达到上限时强制终止

**定义 4.1（TTI 规则）**：TTI 被触发当且仅当：
```
1/(N+1) · Σ_{i=0}^{N} Δ_{t-i}^k < ε    或    t ≥ L_max        (公式 6)
```

### 4.3 Turn 级动态采样（TDS）

#### 动机

智能体交互跨越多个轮次。在 turn 级，一旦环境感知稳定下来，模型可能在连续轮次中重复产生语义相似但无效的推理轨迹。这导致：
- 交互多样性降低
- 探索效率下降
- 有效探索预算被浪费

#### Turn 级控制信号

将单轮内所有 token 的 M_t 聚合为 turn 级信号：
```
Φ^k = (Π_{t=1}^{T} M_t)^{1/T}    （几何平均）
```

监控连续轮次之间的变化：
```
Γ^k = |Φ^k - Φ^{k-1}|
```

**Γ^k 的含义**：
- **大值**：模型内部信心和不确定性结构在轮次间发生了显著变化，表明正在演化信念或进入新的问题求解状态
- **小值**：智能体在重复生成相似的低信息量推理内容

#### 何时重采样？

当 Γ^k < η（turn 级阈值）时，判定当前轮次的探索进展不足。

#### 如何重采样？

丢弃当前轮次的生成结果，在相同状态下重新 rollout。重复直到：
- Γ^k ≥ η（获得足够的探索多样性），或
- 达到最大重采样预算 B_max

**定义 4.2（TDS 规则）**：
```
a^k_new = { Re-generate(a^k),  若 Γ^k = |Φ^k - Φ^{k-1}| < η
          { a^k,                否则                            (公式 7)
```

### 4.4 策略更新

#### 记忆上下文窗口

多轮交互产生极长的序列。T²PO 采用记忆窗口，仅包含最近 P 个轮次的交互历史（而非完整轨迹），降低计算和内存开销。

#### 分层优势估计

结合组级和 turn 级的优势信号：

**组级优势**（GRPO 风格）：
```
A(τ_i^k) = [R(τ_i^k) - mean({R(τ_i^k)}_{i=1}^G)] / F_norm({R(τ_i^k)}_{i=1}^G)
```
捕获完整交互轨迹之间的全局性能差异。

**Turn 级优势**（GiGPO 风格）：
```
A^turn(a_i^k)
```
在更细的 turn 粒度上计算。

**融合**：
```
A'(a_i^k) = A(τ_i^k) + ω · A^turn(a_i^k)
```

#### 策略目标

带裁剪的 PPO 风格目标 + KL 惩罚：

```
J(θ) = E[min(ρ_θ · A', clip(ρ_θ, 1±c) · A')] - β · D_KL(π_θ || π_ref)    (公式 8)
```

其中 ρ_θ = π_θ(a_i^k | s_i^k) / π_{θ_old}(a_i^k | s_i^k)。

## 关键创新

1. **首次将训练崩溃归因于低效探索**：之前的工作将不稳定性归因于奖励稀疏或信用分配问题，T²PO 首次系统性地将其归因于探索-利用权衡的违反（犹豫），并提供了实证支持

2. **自校准不确定性信号**：融合熵和置信度的互补优势，克服了各自的盲区，提供了真正自适应的监控信号

3. **Token 级思考干预**：不是简单地限制推理长度（hard budget）或惩罚长输出（lengthy reward），而是通过监控信息增益的饱和点来自适应终止，保留了有价值的推理同时消除冗余

4. **Turn 级动态采样**：不依赖外部准确率信号（在多轮设置中难以定义），而是利用内在不确定性信号检测冗余轮次，实现动态重采样

5. **即插即用**：T²PO 不引入额外模型参数或环境特定启发式，可直接集成到 GRPO、GiGPO、GSPO 等现有策略更新方案中

| 环境 | 类型 | 规模 | 任务描述 |

- 训练框架：veRL
- 格式惩罚：统一的格式合规惩罚

### 消融实验（Table 3）

| 配置 | Task Score | Success Rate |
|------|-----------|-------------|
| 无 RFT 冷启动 | 79.28 | 61.32 |
| 无 TTI（token 级干预） | 81.28 | 73.27 |
| 无 TDS（turn 级采样） | 72.40 | 63.67 |
| **完整 T²PO** | **93.84** | **81.64** |

关键发现：
- **TDS 贡献最大**：移除 TDS 后成功率暴跌 18 个百分点（81.64 → 63.67），说明 turn 级冗余是最严重的问题
- **TTI 也很重要**：移除后成功率降 8 个百分点（81.64 → 73.27），说明 token 级犹豫也显著浪费探索预算
- **RFT 冷启动必要**：无 RFT 时性能大幅下降，结构化拒绝微调对稳定早期训练至关重要

### 与其他思考控制方法对比（Table 4）

| 方法 | Task Score | Success Rate |
|------|-----------|-------------|
| Lengthy reward（惩罚长输出） | 77.96 | 65.87 |
| Short CoT cold-start（短 CoT 蒸馏） | 82.39 | 71.29 |
| Void turn filtering（过滤空轮） | 85.17 | 76.20 |
| Hard thinking budget（固定长度上限） | 84.96 | 79.21 |
| **T²PO (TTI + TDS)** | **93.84** | **81.64** |

分析：
1. **Lengthy reward**：全局惩罚长输出，无法区分有价值的推理和冗余推理，效果有限
2. **Short CoT cold-start**：改善了早期稳定性，但 RL 训练过程中不主动调控推理，后期仍然漂移
3. **Hard thinking budget**：静态长度限制无法适应不同任务的推理复杂度，简单任务浪费、困难任务截断
4. **Void turn filtering**：仅移除无效轮次，但不能解决语义相似的冗余有效轮次问题

T²PO 的优势在于：**自适应地根据信息增益动态决定何时停止和重采样**，而非依赖全局静态规则。

### 探索效率分析（Figure 7）

- **训练稳定性**：基线方法初期性能提升后崩溃；T²PO 全程单调递增，无崩溃
- **Token 级效率**：T²PO 在相同 token 预算下产生更高比例的成功轨迹。基线方法在推理边界之后浪费大量 token 在冗余继续上
- **Turn 级效率**：T²PO 减少约 25% 的交互轮次，同时达到更高成功率。基线智能体频繁进入跨轮次的重复推理循环

## 为什么方法有效？

1. **直接干预 vs. 间接奖励塑形**：T²PO 不通过修改奖励函数来间接引导行为，而是在生成过程中直接控制何时停止思考和何时重新探索。这避免了奖励塑形引入的超参敏感性和收敛偏差。

2. **利用内在信号**：不确定性是模型自身的内在属性，不需要额外的价值网络、过程奖励模型或环境特定的启发式。这使方法通用且即插即用。

3. **双粒度互补**：Token 级控制消除单轮内的冗余推理，Turn 级控制消除跨轮次的重复探索。两者协同作用，全面恢复了探索-利用的平衡。

4. **信息论基础**：自校准信号 M_t 融合了信息论（熵）和统计学（置信度）的互补视角，提供了比任一单一指标更鲁棒的不确定性度量。

---

## 贡献总结

1. **诊断**：首次将多轮智能体 RL 的训练崩溃系统性归因于低效探索（犹豫），提供了实证分析支持
2. **信号设计**：提出融合熵和置信度的自校准不确定性信号，克服了各自的盲区
3. **Token 级控制**：基于信息增益饱和的自适应思考终止机制（TTI）
4. **Turn 级控制**：基于不确定性变化的动态重采样机制（TDS）
5. **实证验证**：在三个多样化环境（网页、具身、搜索）上取得全面 SOTA，同时显著提升训练稳定性

---

## 局限性

1. **信号校准的超参数**：α（熵-置信度权重）、ε（TTI 阈值）、η（TDS 阈值）、N（滑动窗口大小）需要调节，虽然论文展示了一定的鲁棒性，但最优设置可能因任务而异
2. **重采样开销**：TDS 在检测到冗余轮次时进行重新 rollout，增加了计算成本（虽然通过减少无效轮次部分抵消）
3. **依赖 CoT 格式**：TTI 依赖 `<think>...</think>` 的结构化格式，对于不使用显式 CoT 的模型不直接适用
4. **仅验证了中等规模模型**：实验仅在 4B 和 8B 模型上验证，更大模型（70B+）上的表现有待考察

---

## 核心要点

- **核心问题**：多轮智能体 RL 训练不稳定的根因是低效探索——智能体在 token 级过度思考、在 turn 级重复无效推理
- **核心方案**：通过模型内在的不确定性信号在两个粒度上显式控制探索，而非通过奖励塑形间接引导
- **核心结果**：WebShop 成功率 81.64%/82.42%，ALFWorld 提升 8-12 点，Search QA 全面 SOTA，训练全程无崩溃
- **方法论意义**：证明了训练效率和训练有效性可以同时优化——关键在于识别并消除低效探索，而非仅增加更多训练信号
- **实践价值**：即插即用，可直接集成到 GRPO/GiGPO/GSPO 等现有方案中，不引入额外模型参数
