# 从网络搜索到智能体深度研究：用推理智能体激励搜索

## 摘要

这篇综述论文（23位作者，2025年6月）追踪了信息搜索系统从传统网络搜索引擎，经过 RAG 增强的大语言模型，到新兴的"智能体深度研究"范式的演化过程。它提供了全面的方法分类体系，识别了四个核心流程阶段（规划、问题开发、网络探索、报告生成），引入了深度研究的测试时扩展定律，并认为智能体深度研究将成为信息搜索的主导未来范式。

## 动机与问题

信息检索领域正在经历根本性的转变。三个并行的发展推动了这一变革：

1. **大语言模型推理能力**：DeepSeek-R1 和 o1 等模型证明，扩展的思维链推理可以显著提升问题解决能力
2. **工具增强**：大语言模型现在可以在生成过程中调用外部工具（搜索引擎、浏览器、API），打破了封闭世界假设
3. **智能体架构**：多步骤、自主智能体框架使大语言模型能够规划、执行、观察和迭代——模拟人类研究过程

**本综述填补的空白**：虽然个别论文（R1-Searcher、Search-R1、DeepDiver 等）提出了特定系统，但缺乏一个统一框架来理解：
- 这些系统之间以及与先前范式的关系
- 核心架构组件是什么
- 支配其有效性的理论原则
- 该领域的发展方向

## 方法（综述框架）

### 演化分类体系：信息搜索的四个范式

```
范式 1：传统网络搜索
+------------------------------------------+
| 用户查询 --> 搜索引擎 --> 排序后的         |
| 文档列表 --> 用户阅读和综合                 |
| - 关键词匹配，PageRank                    |
| - 用户完成所有推理                         |
| - 无自然语言理解                           |
+------------------------------------------+
            |
            v（大语言模型出现）
范式 2：大语言模型作为聊天机器人
+------------------------------------------+
| 用户查询 --> 大语言模型 --> 基于参数化     |
| 知识的自然语言回答                         |
| - 对话式界面                              |
| - 无外部知识访问                           |
| - 在事实查询上易产生幻觉                    |
+------------------------------------------+
            |
            v（引入 RAG）
范式 3：RAG 增强的大语言模型
+------------------------------------------+
| 用户查询 --> 检索文档 --> 大语言模型       |
| 阅读文档 + 生成答案                        |
| - 单次检索                                |
| - 固定检索流程                             |
| - 有限的搜索推理能力                        |
+------------------------------------------+
            |
            v（RL + 智能体）
范式 4：智能体深度研究
+------------------------------------------+
| 用户查询 --> 智能体规划 --> 迭代式         |
| 搜索 + 推理 --> 综合报告                   |
| - 多轮、自适应搜索                         |
| - 关于搜索的自主推理                        |
| - 自我验证和优化                           |
| - 完整研究流程自动化                        |
+------------------------------------------+
```

### 深度研究的测试时扩展定律

一项新的理论贡献——形式化推理时计算与研究质量之间的关系：

```
测试时扩展 (TTS) 假说：
  Research_Quality ~ f(Compute_inference)

  其中 Compute_inference 包括：
  - 生成的推理 token 数量
  - 发出的搜索查询数量
  - 处理的检索文档数量
  - 优化迭代次数

  关键洞察：与训练时扩展（更多参数/数据）不同，
  TTS 允许固定模型通过在推理时"更努力地思考"来提升。

  在现有工作中的体现：
  - DeepSeek-R1：更多推理 token --> 更好的数学性能
  - Search-R1：更多搜索轮次 --> 更好的问答准确率
  - DeepDiver SIS：自适应搜索强度 --> 更好的网络搜索
  - PokeeResearch RTS：多线程 --> 更好的研究质量
```

## 关键创新（本综述的）

1. **统一的演化框架**：映射了从网络搜索到智能体深度研究的完整轨迹，展示每个范式如何解决先前范式的局限

2. **四阶段流程分解**：将规划、问题开发、网络探索和报告生成确定为规范阶段，提供了分析任何深度研究系统的视角

3. **搜索的测试时扩展定律**：将 TTS 概念从纯推理扩展到搜索增强推理，形式化了多个近期系统（SIS、RTS 等）背后的原则

4. **全面的分类体系**：将训练方法、搜索集成模式和评估方法组织成结构化分类

5. **社区资源汇集**：在配套的 GitHub 仓库（Awesome-Deep-Research）中整理了行业产品、研究论文、基准测试和开源实现

## 实验设置

作为综述，本文不进行新实验。相反，它综合了领域内的发现，分析的系统包括：

**涵盖的系统：**
- R1-Searcher、Search-R1、ReSearch、AutoAgent-Search
- DeepDiver、DeepResearcher、PokeeResearch
- Search-o1、MindSearch、WebThinker
- 商业产品：Google Deep Research、OpenAI Deep Research、Perplexity

**讨论的基准测试：**
- 事实问答：NQ、TriviaQA、PopQA
- 多跳问答：HotpotQA、2WikiMultiHopQA、Musique、Bamboogle
- 复杂研究：GAIA、BrowseComp、HLE、FRAMES
- 开放网络：WebPuzzle
- 报告质量：FanOutQA、ALCE

## 结果（综合发现）

本综述从文献中提炼出以下实证模式：

**1. RL 在搜索智能体上始终优于 SFT：**
- R1-Searcher：在 HotpotQA 上比 SFT 基线提高 +48.2%
- Search-R1：平均提升 +26%（Qwen2.5-7B）
- DeepDiver：在 WebPuzzle 上比冷启动 SFT 提高 +10 个百分点

**2. 小型 RL 训练模型匹配更大模型：**
- DeepDiver 7B 在 WebPuzzle 上匹配 DeepSeek-R1 671B
- R1-Searcher 8B 在多跳问答上超越 GPT-4o-mini
- Search-R1 7B 与更大的 RAG 系统具有竞争力

**3. 自适应搜索 > 固定搜索：**
- DeepDiver SIS：平均 2.51 轮 vs 1.75 固定轮次，且性能增益相关
- PokeeResearch RTS：GAIA +4.4，BrowseComp +3.0，来自多线程综合

**4. 开放网络 > 维基百科语料库（就泛化而言）：**
- DeepDiver（开放网络训练）比维基百科训练的基线高出 11+ 个百分点

**5. 测试时扩展适用于搜索：**
- 更多搜索轮次、线程和推理 token 始终提升性能
- 最终会出现收益递减，但最优计算量因任务而异

## 局限性

1. **新近性偏差**：作为 2025 年 6 月的综述，它捕捉的是一个快速发展的领域；发表后已有大量重要工作出现
2. **英语为主**：大多数被调研的系统和基准聚焦英语；多语言深度研究的覆盖有限
3. **无新实验**：综述贡献框架和分析，而非实证证据；所声称的测试时扩展定律是假说，而非经过证明的定理
4. **商业系统覆盖**：对专有系统（Google、OpenAI）的分析受限于缺乏技术细节
5. **评估差距**：综述注意到但未解决超越事实准确性的深度研究质量评估这一根本挑战
6. **安全与对齐**：对自主网页浏览智能体风险（错误信息、对抗性网络内容、隐私）的讨论有限

## 核心要点

- 智能体深度研究代表了从静态检索到动态、推理驱动的信息搜索的真正范式转换——这不是渐进式改进，而是 AI 系统与信息交互方式的质的变化

- 四阶段流程（规划 -> 开发问题 -> 探索网络 -> 生成报告）为理解任何深度研究系统提供了有用的分析框架

- 测试时扩展延伸到搜索：分配更多推理时计算（更多搜索、更多推理、更多线程）始终提升研究质量，类似于更多推理 token 提升数学/代码性能

- RL 是搜索智能体的主导训练范式，基于结果的奖励尽管简单但出人意料地有效

- 该领域正在趋向一个共同架构：指令微调大语言模型 + RL 训练 + 多轮搜索 + 基于结果的奖励，具体细节有所不同

- 关键开放挑战：超越事实问答的评估、自主网络智能体的安全性、推理时扩展的计算成本、多语言能力

- 配套的 Awesome-Deep-Research GitHub 仓库作为跟踪该领域快速发展的活跃资源
