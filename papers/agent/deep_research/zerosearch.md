# ZeroSearch：无需真实搜索即可激发大语言模型的搜索能力

## 简要总结

ZeroSearch 由阿里巴巴 NLP 团队提出，解决了使用强化学习增强 LLM 搜索能力时的两个核心瓶颈：真实搜索引擎返回文档质量不可预测，以及搜索 API 调用成本过高。核心方案是用 LLM 自身作为"模拟检索模块"替代真实搜索引擎——通过轻量级监督微调将 LLM 转化为能够生成有用和带噪文档的检索模拟器，配合课程式质量递降策略逐步提高训练难度。使用 14B 模拟引擎在 4 块 A100 上训练成本仅 $70.80，相比真实搜索 API 的 $586.70 降低 88%。7B 模拟引擎达到与 Google Search 相当的性能，14B 模拟引擎甚至超越真实搜索引擎。

## 动机与问题

### 当前方法的两大瓶颈

1. **文档质量不可控**：真实搜索引擎返回的文档质量不可预测——有时高度相关，有时完全无关或带有误导性。这种不确定性使 RL 训练的奖励信号充满噪声，影响策略学习的效率和稳定性
2. **API 成本过高**：RL 训练需要大量 rollout 与搜索引擎交互。以 Google Search API 为例，训练约 64K 查询需花费 $586.70，对学术和中小规模团队而言不可承受

### 关键洞察

> LLM 本身蕴含了大量的世界知识，可以被转化为搜索引擎的"模拟器"——生成与真实搜索结果分布相近的文档，同时可精确控制文档质量以实现课程式训练。

### 与先前工作的关系

- **Search-R1** 等方法在 RL 训练中使用真实搜索引擎，受限于成本和质量波动
- ZeroSearch 的模拟检索方法与 Search-R1 的检索 token 掩码技术正交互补——前者解决训练数据源问题，后者解决梯度归因问题

## 方法

### 一、检索模拟模块的构建

#### 步骤一：交互轨迹收集

```
使用策略模型与真实搜索引擎的交互轨迹：
1. 对训练集中的每个问题 q，策略模型生成搜索查询
2. 查询真实搜索引擎获取文档
3. 策略模型基于文档生成答案
4. 根据答案正确性标记轨迹：
   - 正面轨迹（答案正确）→ 提取 (query, useful_docs) 对
   - 负面轨迹（答案错误）→ 提取 (query, noisy_docs) 对
```

#### 步骤二：轻量级监督微调

```
训练检索模拟模块 M_sim：
- 输入：查询 query + 质量控制提示
- 输出：模拟文档集

关键设计——通过简单关键词切换控制文档质量：
- 提示包含 "useful" → 生成高质量相关文档
- 提示包含 "noisy" → 生成低质量或误导性文档
- 同时包含问题的 ground-truth 答案以扩大知识覆盖范围

超参数：learning rate = 1e-6
```

#### 步骤三：替代真实搜索引擎

训练完成后，RL 训练中的所有搜索请求由模拟模块 $M_{sim}$ 处理，完全消除对外部搜索 API 的依赖。

### 二、课程式质量递降策略

这是 ZeroSearch 训练效果的关键——通过渐进式提高文档噪声来锻炼模型的推理能力：

#### 噪声概率调度函数

$$p_i = p_s + \frac{b^{i/m} - 1}{b - 1} \times (p_e - p_s)$$

其中：
- $p_s$：初始噪声概率（较低，从简单开始）
- $p_e$：最终噪声概率（较高，最终阶段较困难）
- $b$：指数基数，控制递降速度
- $i$：当前训练步
- $m$：总训练步数

#### 课程学习的直觉

```
训练阶段 1（早期）：
  - 噪声概率低 → 大部分文档高质量
  - 模型学习：基本输出格式 + 搜索查询生成 + 从好文档中提取答案

训练阶段 2（中期）：
  - 噪声概率中等 → 混合质量文档
  - 模型学习：区分有用和无用文档 + 在噪声中推理

训练阶段 3（后期）：
  - 噪声概率高 → 大部分文档带噪声或误导性
  - 模型学习：在高噪声环境中保持正确推理 + 生成更精准的搜索查询
```

**实验验证**：easy-to-hard（由易到难）方向显著优于 hard-to-easy（由难到易）方向，后者性能低 1%-5%。

### 三、强化学习训练

#### 损失掩码机制

```
关键创新：对模拟检索返回的文档 token 不计算梯度

Loss = sum over t in T_policy [ RL_loss(a_t | s_t) ]
其中 T_policy = 策略模型生成的 token 位置（排除模拟文档 token）

原因：模拟文档不是策略模型的输出，对其计算梯度会导致：
- 分布不匹配
- 训练不稳定
- 将文档质量错误归因于策略
```

消融实验表明，移除损失掩码导致各基准性能下降约 5%。

#### 兼容的 RL 算法

| 算法 | 特点 |
|------|------|
| **GRPO** | 每个 prompt 采样 5 个响应；训练更稳定（推荐） |
| **PPO** | 包含独立的价值模型（lr=1e-5）；收敛更快但可能不稳定 |
| **Reinforce++** | 同样受支持，表现可比 |

### 四、模拟引擎规模效应

| 模拟引擎规模 | 效果 |
|-------------|------|
| 3B (SFT-3B) | 有效但性能较低，证明基本可行性 |
| 7B (SFT-7B) | 达到与 Google Search 相当的性能 |
| 14B (SFT-14B) | **超越**真实搜索引擎性能 |

更大的模拟模型能更好地区分相关/不相关文档，从而实现更有效的课程学习。

## 关键创新

1. **模拟检索替代真实搜索**：用 LLM 自身的知识生成模拟文档替代真实搜索 API 调用，从根本上消除了 RL 训练对外部搜索服务的依赖
2. **课程式质量递降**：通过指数调度函数渐进增加文档噪声比例，实现从易到难的课程学习，比固定噪声比例或反向课程显著更优
3. **质量可控的文档生成**：仅通过提示词中的简单关键词（"useful" vs "noisy"）即可切换生成文档的质量等级，设计极为简洁高效
4. **成本降低 88%**：将训练成本从 $586.70 降至 $70.80（14B 模拟引擎），使搜索增强 RL 训练对学术团队可行
5. **模拟超越现实**：14B 模拟引擎训练出的策略模型性能竟然超过使用真实搜索引擎训练的模型，这一反直觉结果归因于文档质量的可控性消除了真实搜索的随机噪声

## 实验结果

### 主要结果（Qwen-2.5-7B-Instruct）

| 基准 | ZeroSearch | Search-R1 (真实搜索) | 提升 |
|------|-----------|---------------------|------|
| NQ | 43.24% | - | - |
| TriviaQA | 61.81% | - | - |
| PopQA | 51.52% | - | - |
| HotpotQA | 29.21% | - | - |
| 7 数据集平均 | 40.54% | 基线 | 一致超越 |

ZeroSearch 在域内和域外数据集上均一致性地超越 Search-R1（使用真实搜索引擎的基线）。

### 跨模型泛化

| 模型 | 基座版本平均 | 指令版本平均 |
|------|------------|------------|
| Qwen-2.5-3B | 33.97% | 33.02% |
| LLaMA-3.2-3B | 35.26% | 31.19% |
| Qwen-2.5-7B | - | 40.54% |

- 基座模型和指令微调模型均可使用，最终性能可比
- 指令微调模型起点更高但最终差距不大

### 成本对比

| 训练方式 | 硬件需求 | 成本 (~64K 查询/12 小时) | 相对成本 |
|---------|---------|------------------------|---------|
| Google Search API | - | $586.70 | 100% |
| SFT-7B 模拟引擎 | 2× A100 | $35.40 | 6.0% |
| SFT-14B 模拟引擎 | 4× A100 | $70.80 | 12.1% |

成本节省：84%-94%

### 消融研究结果

| 消融项 | 影响 |
|--------|------|
| 移除损失掩码 | 各基准下降约 5% |
| 反向课程（由难到易） | 低于正向课程 1%-5% |
| GRPO vs PPO | GRPO 更稳定；PPO 收敛更快 |
| 基座 vs 指令模型 | 最终性能可比 |

## 局限性

1. **GPU 基础设施需求**：虽然消除了 API 成本，但仍需 GPU 服务器部署模拟引擎，对于完全无 GPU 资源的团队仍有门槛
2. **GPU 利用率不均**：策略更新期间 GPU 利用率下降，存在计算资源浪费。论文建议通过跨训练任务共享模拟服务器来缓解
3. **知识局限**：模拟引擎的知识上限受限于训练数据的知识截止日期，无法生成截止日期之后的新信息
4. **模拟保真度**：模拟文档虽然统计上与真实搜索结果相近，但可能在某些领域或查询类型上存在系统性偏差
5. **规模依赖性**：模拟引擎的质量与参数规模正相关（3B < 7B < 14B），小规模模拟引擎可能不足以完全替代真实搜索

## 核心要点

- ZeroSearch 提出了一个违反直觉但极具实用价值的发现：用 LLM 生成的模拟文档训练出的搜索增强模型，可以**等于甚至超越**使用真实搜索引擎训练的模型
- 这一结果的根本原因在于：真实搜索引擎的文档质量存在不可控的随机性，而模拟引擎提供了可精确调控的质量渐变，使课程学习更加有效
- 88% 的成本降低（$586.70 → $70.80）使搜索增强 RL 训练从大公司的专属技术变为学术团队可及的方法
- 课程式质量递降策略的有效性验证了一个重要的训练原则：让模型先在简单环境中学习基础技能，再逐步面对更困难的场景，比直接在困难环境中训练更优
- 损失掩码对训练稳定性至关重要——这与 Search-R1 的检索 token 掩码理念一致，进一步确认了 RL 训练中区分策略 token 和外部 token 的重要性
- 模拟引擎的规模效应（3B < 7B < 14B）为实践者提供了明确的选择指导：7B 是性价比最优的选择，14B 是追求极致性能的选择
