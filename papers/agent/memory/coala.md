## TL;DR

Proposes a unifying cognitive architecture framework for language agents, drawing from cognitive science to systematize agent design.

## Method

Defines agent components:
- **Working Memory**: current context, active state
- **Long-term Memory**:
  - Episodic (past experiences)
  - Semantic (facts, knowledge)
  - Procedural (skills, code)
- **Decision-making**: planning, action selection
- **Action modules**: tool use, code execution, communication

## Why It Matters

Provides a theoretical backbone for understanding and designing agent architectures. Helps practitioners reason about what memory types their agent needs and how to structure the decision-making process.
