# 基于大语言模型的深度搜索智能体综述：范式、优化、评估与挑战

## 简要总结

本文是首篇系统性分析 LLM 驱动搜索智能体的综合综述，被 ACL 2026 主会议收录。作者来自上海交通大学和中南大学。综述从架构范式、优化方法、应用领域和评估体系四个维度全面分析和分类现有工作，区别于先前仅关注子方向（如深度研究或推理增强检索）的综述。核心论点是 LLM 搜索智能体标志着从手动搜索结果整合到自主、上下文感知的多轮信息获取的范式转变。论文提供了涵盖搜索结构分类、优化策略对比、评估框架演进等方面的完整管线分析，并指出信息融合、不完美检索、多模态扩展等六大未来研究方向。配套资源库 Awesome-Search-Agent-Papers 托管于 GitHub。

## 动机与问题

### 研究背景

LLM 搜索智能体的兴起代表了信息检索领域的关键转变：
- 从传统网页搜索（用户手动输入关键词、浏览结果、整合信息）
- 到 LLM 增强搜索（LLM 辅助查询改写、结果摘要）
- 再到搜索智能体（自主理解用户意图、动态规划多轮检索、跨源信息整合）

### 现有综述的不足

先前综述存在的局限性：
1. **Xu & Peng (2025)** 和 **Huang et al. (2025b)**：仅聚焦于深度研究（Deep Research）子方向
2. **Liang et al. (2025)** 和 **Gao et al. (2025)**：仅关注推理-RAG 集成
3. 这些综述都未能覆盖完整的管线——从搜索结构到优化到评估

### 本综述的定位

> 首篇覆盖完整管线（搜索结构 + 优化 + 应用 + 评估）的搜索智能体系统性综述，为该领域提供全景式知识地图。

## 方法

本综述采用系统性文献分析方法，围绕四个核心维度展开：

### 一、搜索架构范式分类

#### 并行结构（Parallel Structures）

**分解式（Decomposition-based）**：
- 将复杂查询分解为多个子查询，并行执行
- 遵循 规划 → 执行 → 验证 的范式
- 适用于可独立回答的多面查询

**多样化式（Diversification-based）**：
- 为同一查询生成多个可能的变体
- 并行搜索不同变体以增加召回率
- 适用于查询意图不明确的场景

#### 顺序结构（Sequential Structures）

**反思驱动（Reflection-driven）**：
- 循环机制：搜索 → 生成答案 → 反思质量 → 再搜索
- 根据反思结果动态调整搜索策略
- 典型代表：Self-RAG、CRAG

**主动性驱动（Proactivity-driven）**：
- 智能体根据上下文动态决定何时搜索、搜索什么
- 不依赖预定义的搜索时机
- 典型代表：Search-R1、R1-Searcher

#### 混合结构（Hybrid Structures）

**树结构（Tree-based）**：
- 多个后继节点并行扩展
- 通过投票或奖励模型聚合结果
- 适用于需要多路径探索的复杂问题

**图结构（Graph-based）**：
- 允许回溯和任意节点连接
- 支持动态问题分解
- 最高灵活性但计算开销最大

#### 结构选择的经验规律

- **无调优设置**：混合结构更受青睐，因其通过结构多样性弥补模型的局限
- **微调设置**：顺序结构配合模型内化的灵活性表现更优

### 二、优化方法分类

#### 无调优方法（Tuning-Free）

**单智能体架构**：
- 结构化工作流（如 ReAct 循环）
- 提示工程优化搜索策略
- 工具选择和调用优化

**多智能体系统**：
- 角色分工：规划器、搜索器、生成器、浏览器、评估器、记忆器
- 固定或集中式执行编排
- 代表：MindSearch、Co-Search

**测试时扩展**：
- 推理中心扩展：增加思维链长度和深度
- 搜索中心扩展：增加搜索轮次和覆盖范围

#### 调优方法（Tuning-Based）

**监督微调（SFT）**：
- 教师模型蒸馏：从强模型的搜索轨迹中学习
- 自我改进：通过拒绝采样筛选高质量轨迹
- RL 预热：为后续 RL 训练提供良好初始化

**强化学习（RL）**：
- 算法：PPO、GRPO、Reinforce++
- 多目标奖励函数设计：
  - 答案正确性
  - 搜索效率（步数、token 消耗）
  - 检索多样性
  - 证据质量

**混合方法（SFT → RL）**：
- SFT 建立基础能力 → RL 进一步优化
- 迭代循环：交替进行 SFT 和 RL
- 对比学习集成

#### 优化方法的发展趋势

> 近期趋势表明向动态性和上下文适应性转变——无调优设置中混合结构更受青睐，而微调模型在顺序结构中内化灵活性。混合 SFT→RL 优化正成为主流策略。

### 三、评估框架演进

#### 数据集分类

**封闭式问答（Closed-ended QA）**：
- 多跳推理：HotpotQA、IIRC、MuSiQue、2WikiMultihopQA
- 挑战性长尾问题：BrowseComp、GAIA
- 事实核查：FEVER、HoVer
- 领域特定：金融、医疗、代码、生物、化学

**开放式任务（Open-ended）**：
- 深度研究报告生成
- 需要综合性信息综合的任务
- Scale AI Research Rubrics、ADR-Bench

#### 评估指标演进

```
演进路径：
规则评判器 → LLM 评判器 → 智能体评判器 → 人类评估

第一代：规则评判器
  - Exact Match (EM)
  - F1 Score
  - 局限：仅适用于有标准答案的封闭式问题

第二代：LLM-as-Judge
  - 利用 LLM 评估语义等价性
  - 支持开放式回答的评估
  - 局限：可能引入模型偏见

第三代：Agent-as-Judge
  - 整体性评估智能体的多步过程
  - 评估检索质量、推理链路、最终答案
  - 指标：Precision、Recall、NDCG、MAP

第四代：人类评估
  - 金标准但规模受限
  - 通常用于验证自动化指标的有效性
```

### 五、核心组件的功能抽象

论文将搜索智能体的核心功能管线抽象为五个组件：

```
1. Planning:   π₀ = Plan(q, C)         // 初始策略制定
2. Reflection: π_{t+1} = Reflect(o_t, h_t, π_t)  // 基于观察更新计划
3. Action:     a_{t+1} = Act(π_{t+1})   // 执行检索/浏览操作
4. Selection:  E = Select(q, O)         // 从观察中提取相关证据
5. Generation: ŷ_q = Generate(q, E)     // 基于证据生成最终回答
```

## 关键创新

1. **首篇全管线综述**：区别于先前仅关注子方向的综述，覆盖从架构到优化到评估的完整管线
2. **三层搜索结构分类**：提出并行/顺序/混合的层次化分类体系，每类下进一步细分具体范式
3. **优化方法系统对比**：在无调优和有调优两大框架下，系统性地对比单智能体/多智能体/测试时扩展/SFT/RL/混合方法
4. **评估演进图谱**：清晰描绘了从规则评判到 LLM 评判再到智能体评判的评估方法演进路径
5. **六大未来方向**：基于全面分析提出信息融合、不完美检索、多模态、定制化 RL、鲁棒基础设施、自我演化等六个前沿方向
6. **功能抽象管线**：将搜索智能体抽象为 Plan-Reflect-Act-Select-Generate 五步功能管线，提供统一的分析框架

## 实验结果

作为综述论文，本文不包含原创实验，但通过系统性文献分析得出以下关键发现：

### 搜索结构比较

- 混合结构在无调优设置中表现最佳，但计算开销最大
- 顺序结构配合微调最为高效，因模型可内化搜索灵活性
- 并行结构适用于可分解的多面查询

### 优化策略对比

- 混合 SFT→RL 成为主流训练策略
- GRPO 在搜索增强 RL 中比 PPO 更稳定
- 多目标奖励函数（正确性 + 效率 + 多样性）优于单一正确性奖励

### 评估方法对比

- 封闭式 QA 的 EM/F1 指标已成熟但适用范围有限
- 开放式任务的评估仍是开放问题
- Agent-as-Judge 是最有前景的自动化评估范式

## 局限性

1. **商业系统覆盖不足**：OpenAI、Google、Perplexity 等商业系统的技术细节不公开，综述只能基于公开信息进行分析
2. **快速演进领域**：搜索智能体领域发展极快，综述的时效性存在固有挑战
3. **定量对比困难**：不同论文使用不同基准、不同设置，难以进行严格的横向性能对比
4. **应用深度有限**：虽然覆盖了多个应用领域，但对每个领域的分析深度受限于篇幅
5. **实践指导不足**：综述侧重于方法分类和趋势分析，对实践者如何选择具体方案的指导性有待加强

## 核心要点

- 本综述为 LLM 搜索智能体领域提供了迄今最全面的知识地图，其 并行/顺序/混合 的搜索结构三分法和 无调优/有调优 的优化方法二分法为后续研究提供了清晰的分类框架
- **搜索结构选择的经验法则**：无调优用混合结构以广度弥补模型局限，有调优用顺序结构让模型内化灵活性——这一洞察对系统设计有直接指导价值
- **混合 SFT→RL 已成主流**：在优化策略方面，先通过 SFT 建立基础能力再通过 RL 优化的两阶段方法被越来越多的工作采用，包括 Step-DeepResearch、Search-R1 等
- **评估方法的快速演进**表明该领域对"什么是好的搜索智能体"的认知正在深化——从仅看最终答案（EM/F1）到评估整个搜索过程（Agent-as-Judge）
- **六大未来方向**中，信息融合（如何整合多源异质信息）和不完美检索（如何在噪声检索中保持鲁棒推理）可能是短期内最具影响力的研究课题
- 多模态搜索（跨文本、图像、视频的搜索与推理）和自我演化（智能体自主改进搜索策略）代表了该领域的长期愿景
