# 基于强化学习的智能体搜索综合综述：基础、角色、优化、评估与应用

## 简要总结

本综述是首篇全面系统地梳理基于强化学习（RL）的智能体搜索（agentic search）领域的综述论文。论文沿三个互补维度组织这一新兴领域：（i）RL 用于什么（功能角色 functional roles）、（ii）RL 如何使用（优化策略 optimization strategies）、（iii）RL 应用在哪里（优化范围 scope of optimization）。综述涵盖了从检索增强生成（RAG）的基础概念到最新的 RL 驱动的自适应搜索系统（如 Search-R1、ZeroSearch、AgentGym-RL），全面分析了 RL 在搜索各决策点的介入方式，包括何时检索（retrieval control）、如何构造查询（query optimization）、如何将证据整合到推理中（reasoning-retrieval integration）以及使用哪些工具和知识源（tool and knowledge integration）。

## 动机与问题

### LLM 信息获取的根本性限制

大语言模型（LLM）虽然彻底改变了信息获取的方式，但存在三个根本性限制：

1. **静态知识**：LLM 的参数化知识在训练截止日期后即冻结，无法获取实时信息
2. **事实幻觉（Factual Hallucination）**：LLM 倾向于生成表面合理但事实错误的内容
3. **缺乏实时和领域特定信息获取能力**：无法动态检索最新数据或特定领域的专业信息

### 传统 RAG 的瓶颈

检索增强生成（Retrieval-Augmented Generation, RAG）通过将 LLM 输出锚定于外部证据来缓解上述问题。然而，传统 RAG 管线存在显著局限：

- **单轮交互**：传统 RAG 通常执行"检索一次 -> 生成"的单轮流程，缺乏迭代优化能力
- **启发式方法**：检索策略（何时检索、检索什么）依赖硬编码的启发式规则
- **缺乏自适应控制**：无法根据任务复杂度和中间结果动态调整检索和推理策略
- **检索与推理分离**：检索和推理被视为独立的管线阶段，缺乏深层整合

### 智能体搜索的兴起

智能体搜索（Agentic Search）通过赋予 LLM 规划（Plan）、检索（Retrieve）和反思（Reflect）的能力来解决上述限制，使 LLM 能够与搜索环境进行多步交互。在这一范式中，RL 提供了一种强大的机制来实现自适应和自我改进的搜索行为。

### 综述的必要性

随着 DeepSeek-R1、Search-R1 等里程碑式工作的出现，RL 驱动的智能体搜索已经从理论探索进入实用阶段。然而，该领域的快速发展使得研究者难以把握全局，需要一篇全面的综述来梳理现有工作、建立分类体系、指明未来方向。

## 方法

### 三维分类框架

综述提出了一个三维分类框架来组织整个领域：

**维度一：功能角色（What RL is for）**

RL 在智能体搜索中扮演五大功能角色：

1. **检索控制（Retrieval Control）**：RL 决定何时触发检索操作
   - 自适应检索触发：根据模型的置信度或任务复杂度动态决定是否需要外部检索
   - 多轮检索决策：在长程推理中多次决定是否需要补充信息

2. **查询优化（Query Optimization）**：RL 优化如何构造搜索查询
   - 查询改写和扩展
   - 基于中间结果的查询迭代优化
   - 多角度查询生成

3. **推理-检索整合（Reasoning-Retrieval Integration）**：RL 优化如何将检索到的证据整合到推理过程中
   - 推理与检索的交错执行（interleaved reasoning and retrieval）
   - 证据的选择性整合
   - 基于推理状态的检索策略调整

4. **工具与知识源整合（Tool and Knowledge Integration）**：RL 决定使用哪些工具和知识源
   - 多工具选择和协调
   - 不同知识源（网络搜索、数据库、知识图谱）的动态切换
   - 工具使用策略的优化

5. **多智能体协调**：RL 优化多个搜索智能体之间的协调和分工

**维度二：优化策略（How RL is used）**

RL 在智能体搜索中的优化策略包括：

1. **冷启动初始化（Cold-Start Initialization）**
   - 监督微调（SFT）阶段为 RL 训练提供基线能力
   - 使模型具备基本的任务理解和工具使用能力
   - 减轻长程环境中稀疏奖励导致的早期不稳定性

2. **RL 微调（RL Fine-tuning）**
   - 策略 LLM 接收复杂查询
   - 在模拟或真实搜索环境中生成交错的推理和工具使用动作
   - 通过奖励信号优化搜索策略

3. **纯 RL 训练方法**
   - ZeroSearch：用潜在空间检索模拟替代 SFT，实现无外部监督的纯 RL 训练
   - AgentGym-RL：采用课程式的视野扩展（curriculum-based horizon scaling）来稳定纯 RL 训练

**维度三：优化范围（Where RL is applied）**

RL 的优化可以针对搜索流程的不同阶段和组件进行：
- 查询生成阶段
- 检索执行阶段
- 证据评估阶段
- 推理整合阶段
- 端到端全流程优化

### 训练环境与奖励设计

**训练环境**：
- 模拟搜索环境：使用预构建的文档集合模拟搜索引擎响应
- 真实搜索环境：直接与实际搜索引擎交互
- 混合环境：结合模拟和真实环境的优势

**奖励函数设计**：
- 基于最终答案质量的稀疏奖励
- 基于中间检索质量的密集奖励
- 基于推理链质量的过程奖励
- 多目标奖励：综合考虑答案质量、检索效率、推理一致性

### 代表性方法

**Search-R1**：
- 典型的 RL 训练管线示例
- 两阶段训练：冷启动初始化 + RL 微调
- 在搜索环境中学习自适应的检索和推理策略

**ZeroSearch**：
- 用潜在空间检索模拟替代 SFT
- 实现完全基于 RL 的训练，无需外部监督数据
- 降低了对标注数据的依赖

**AgentGym-RL**：
- 采用课程式视野扩展策略
- 逐步增加任务复杂度和搜索视野
- 稳定纯 RL 训练过程中的学习

## 关键创新

1. **首个全面分类框架**：建立了三维分类框架（功能角色/优化策略/优化范围），为快速发展的领域提供了结构化的研究地图
2. **五大功能角色识别**：系统性地识别和分类了 RL 在智能体搜索中的五大功能角色，揭示了 RL 的介入点远超基本检索
3. **从单轮到多步的范式转变分析**：深入分析了从传统 RAG 到 RL 驱动的智能体搜索的范式转变，阐明了每个转变的技术动因
4. **训练范式的系统比较**：对 SFT+RL、纯 RL、课程学习等不同训练范式进行了系统比较
5. **实用方法论指导**：为研究者和实践者提供了选择 RL 策略、设计奖励函数和构建训练环境的指导

## 实验结果

### 评估基准覆盖

综述系统性地梳理了用于评估 RL 驱动的智能体搜索的基准和数据集：

**问答数据集**：
- Natural Questions (NQ)
- TriviaQA
- HotpotQA（多跳推理）
- 其他多步推理 QA 数据集

**事实验证基准**：
- FEVER
- EX-FEVER
- FEVEROUS
- FactBench
- RealFactBench

**信息检索基准**：
- BEIR（跨领域检索基准）
- AmbigQA（歧义查询处理）

**深度研究基准**：
- RAG-Gym
- 以及其他专门评估深度研究智能体的基准

### 关键趋势与发现

1. **RL 带来一致性提升**：在几乎所有评估基准上，RL 优化的系统相比纯 SFT 基线都获得了显著的性能提升
2. **自适应检索的价值**：学习何时检索（而非总是检索）是性能提升的关键因素之一
3. **推理-检索交错的优势**：将推理和检索过程交错执行，相比管线式分离执行，能更好地利用证据
4. **训练效率的权衡**：纯 RL 方法（如 ZeroSearch）虽然降低了数据依赖，但训练稳定性仍是挑战
5. **规模化效应**：RL 优化的效果随模型规模和训练数据的增加而增强

### 应用领域

综述识别了 RL 驱动的智能体搜索在以下领域的应用：
- **开放域问答**：最成熟的应用场景
- **事实验证与幻觉检测**：利用 RL 优化的检索来验证 LLM 生成的事实
- **多步推理任务**：需要多轮检索和推理的复杂问题
- **领域特定搜索**：医学、法律、科学等专业领域的信息检索
- **深度研究系统**：面向长篇研究报告生成的综合系统

## 局限性

1. **领域快速演变**：该领域发展极快（尤其是 2025 年以来），综述可能遗漏了发表后的重要工作
2. **实验可比性困难**：不同论文使用不同的评估设置、基线和超参数，直接比较结论需谨慎
3. **训练成本分析不足**：综述未充分讨论不同 RL 方法的计算成本和训练效率的定量比较
4. **真实世界部署案例有限**：大多数工作仅在学术基准上评估，真实世界的部署挑战（延迟、成本、可靠性）讨论不足
5. **多模态搜索覆盖不足**：主要聚焦于文本搜索，多模态（图像、视频、结构化数据）搜索场景覆盖有限
6. **安全与对齐考虑**：RL 优化可能导致搜索行为的不可预测性，安全和对齐方面的讨论较少
7. **长程搜索的挑战**：RL 在需要数十步搜索的极长视野任务中的表现仍不明确
8. **奖励工程的困难**：设计有效的奖励函数仍然是一个依赖领域知识的工程挑战

## 核心要点

- 本综述是首篇全面梳理 RL 驱动的智能体搜索领域的综述，提出了三维分类框架：功能角色（What）、优化策略（How）、优化范围（Where）
- RL 在四个关键决策点介入搜索过程：检索控制（何时检索）、查询优化（如何构造查询）、推理-检索整合（如何整合证据）、工具与知识源整合（使用哪些资源）
- 典型训练管线（如 Search-R1）包含冷启动 SFT + RL 微调两阶段，ZeroSearch 和 AgentGym-RL 探索了纯 RL 训练路线
- 从传统单轮 RAG 到 RL 驱动的多步智能体搜索，核心范式转变在于引入了自适应控制和自我改进能力
- 评估覆盖问答（NQ、TriviaQA、HotpotQA）、事实验证（FEVER 系列）、信息检索（BEIR）和深度研究等多类基准
- 自适应检索触发（学习何时检索）和推理-检索交错执行是性能提升的两个关键机制
- 官方资源库 https://github.com/ventr1c/Awesome-RL-based-Agentic-Search-Papers 持续更新该领域的代表性工作
- 未来方向包括多模态搜索、长程搜索稳定性、训练效率提升和真实世界部署验证
